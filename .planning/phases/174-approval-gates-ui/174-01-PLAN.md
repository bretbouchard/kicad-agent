# Gate Types (from Obdurate Runtime)

```swift
enum GateType: String, Codable {
    case ercWarning           // ERC detected warnings
    case drcFailure           // DRC failed
    case opConfirmation       // Destructive operation (delete component, clear board)
    case phaseTransition      // Moving between GSD phases
    case verificationFailure  // Post-op verification failed
    case escalation           // T1/T2/T3/T4 escalation triggered
}
```

# Gate Context Structure

```swift
struct GateContext: Codable {
    var id: UUID
    var type: GateType
    var intent: String           // What the user requested
    var operation: String        // Op that will execute
    var verificationResult: VerificationResult?
    var requirementId: String?  // Linked requirement (if applicable)
    var timestamp: Date
}
```

# Four-State Resolution (from Obdurate Runtime)

When user approves/rejects gate, Decision event created with resolution state:
- `.implemented` → Approved, executed
- `.addedAsPhase` → Deferred to current phase (retry later)
- `.supersededByAlternative` → Alternative approach taken (with evidence)
- `.deferredToNamedTarget` → Deferred to future milestone (with trigger)

# Weightless Approval Pattern

- One tap to approve (primary action, prominent)
- "Show me" → drill into detail (secondary action, less prominent)
- Reject → requires confirmation (prevent accidental rejects)
- Clear context: intent, op, verification result, requirement linkage visible inline

## STRIDE Threat Register

|-----------|----------|-----------|-------------|-----------------|
| T-174-01 | Spoofing | Gate context | mitigate | Validate gate context structure (all required fields present). Reject malformed gates. |
| T-174-02 | Tampering | Decision events | mitigate | All Decision events go through SwiftData validation. Tampered events rejected. |
| T-174-03 | Information Disclosure | Verification errors | accept | Errors may contain file paths — local-only, no external exposure. |
| T-174-04 | Denial of Service | Gate spam | mitigate | Cap pending gates at 10 per conversation. Auto-dismiss oldest after 10 minutes. |

## Adversarial Scenario

**Scenario:** A malicious tool injects a fake gate event with crafted HTML in the intent field (e.g., `<img src=x onerror='alert("XSS")'>`). When the gate is rendered, the script executes.

**Mitigation:** Sanitize all gate context fields before rendering. Use `AttributedString` with safe HTML rendering (strip scripts, events). Validate gate structure: `intent`, `operation`, `verificationResult` must match expected types. Reject gates with unexpected fields.
</threat_model>
