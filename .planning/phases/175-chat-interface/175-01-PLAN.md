# Message Streaming (CHAT-02)

LLM responses stream token-by-token:
- Provider returns `AsyncThrowingStream<String, Error>`
- UI appends tokens to message text in real-time
- Stream failures show partial response + "Retry" button
- ESC cancels stream (`.task.cancel()`)

# Image Attachments (CHAT-06)

User can attach images to messages:
- Drag-and-drop images to chat input
- "Attach" button → `PhotosPicker` (iOS 18+)
- Images >10MB auto-compressed (max dimension 2048px)
- Unsupported formats show inline error (`.alert("Unsupported image format. Please use PNG, JPG, or HEIC.")`)

# Cost Tracking (CHAT-07)

Per assistant message:
- Token count (input + output)
- Model badge (FoundationModels, Gemma 4 12B, etc.)
- Cost estimate (USD, based on provider pricing)
- Cumulative session cost

# Conversation History (CHAT-05)

Scroll, search, copy:
- Lazy virtualization for 10K messages (performance)
- Full-text search across conversations
- Copy message text to clipboard
- Edit message → fork conversation (Phase 173)

# Streaming Implementation Pattern

```swift
// Provider returns AsyncThrowingStream
func streamCompletion(messages: [Message]) async throws -> AsyncThrowingStream<String, Error> {
    let provider = providerRouter.selectProvider(for: .conversation)
    return try await provider.streamCompletion(messages: messages)
}

// UI consumes stream
Task {
    do {
        for try await token in stream {
            messageText.append(token)
        }
    } catch {
        streamFailed = true
    }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User → Chat Input | User-provided text/images |
| LLM → Streaming Response | Untrusted LLM output |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-175-01 | Spoofing | LLM streaming response | mitigate | Sanitize LLM output (strip HTML, scripts). Render as plain text or safe Markdown. |
| T-175-02 | Tampering | Image attachments | mitigate | Validate image magic bytes (PNG: `\\x89PNG`, JPG: `\\xFF\\xD8\\xFF`). Reject non-images. |
| T-175-03 | Information Disclosure | Chat history | accept | Chats stored locally — no external exposure. CloudKit sync is encrypted. |
| T-175-04 | Denial of Service | Unbounded message list | mitigate | Lazy virtualization + cap at 10K messages. Archive older messages to separate store. |

## Adversarial Scenario

**Scenario:** A malicious LLM response includes a crafted Markdown link with JavaScript injection (e.g., `[click me](javascript:alert('XSS'))`). When the message is rendered, tapping the link executes the script.

**Mitigation:** Sanitize Markdown rendering. Use `AttributedString` with safe Markdown parsing (strip `javascript:`, `data:` URLs). Only allow `https://`, `http://` links. Tap → opens in `NSWorkspace.openURL(_:)` (which validates URL before opening).
</threat_model>
