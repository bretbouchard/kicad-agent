    Function `run_strategy_with_orchestrator(strategy, pcb_path, project_dir, strategy_name) -> StrategyEvalResult`:
    - Construct `RoutingOrchestrator(strategy=strategy)`.
    - Call `result = orch.route_board(pcb_path, project_dir=project_dir)`.
    - Compute metrics from `result.per_net`:
      - `total_nets = len(result.per_net)`
      - `routed_nets = result.total_routed`
      - `completion_pct = routed_nets / total_nets if total_nets > 0 else 0.0`
      - `via_count = sum(nr.via_count for nr in result.per_net.values())`
      - `total_trace_length_mm = sum(nr.route_length_mm for nr in result.per_net.values())`
    - For AI strategy: detect fallback by checking if any net's notes contain "ai_fallback:". Set `parse_success` and `validation_passed` accordingly (True if no fallback detected).
    - Run DRC: `drc_result = run_drc(pcb_path)`. Populate drc_pass, drc_unconnected.
    - Return StrategyEvalResult.

    Function `run_drc(pcb_path) -> tuple[bool, int]`:
    - Run `kicad-cli pcb drc <pcb_path>` via subprocess.
    - Parse output for "unconnected" count.
    - Return `(drc_pass, unconnected_count)`.
    - On subprocess error or kicad-cli missing: return `(False, -1)` (sentinel).
    - Wrap in try/except — DRC is best-effort, eval must not crash if kicad-cli is unavailable.

    - Construct `KiCadVisionPipeline(config)`.
    - Return `AiRoutingStrategy(pipeline, pcb_path)`.
    - This is the only function that loads the real model — keep it isolated so unit tests don't trigger it.
