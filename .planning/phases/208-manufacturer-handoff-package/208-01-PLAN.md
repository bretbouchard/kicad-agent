# Phase 208 Plan 01 — Manufacturer Handoff Package

**Goal:** One call (`build_handoff_export`) produces a complete zip bundle with all manufacturing artifacts + readme + manifest, with a pre-handoff validation gate that blocks incomplete bundles (no zip on DRC/ERC failure). Generalizes the hard-coded JLCPCB BOM into a profile-driven formatter. Embeds DRC/ERC results in the manifest as proof of manufacturability.

**Phase source dir:** `.planning/phases/208-manufacturer-handoff-package/`

## Threat Model

| # | Attack surface | Threat | Mitigation (task) |
|---|----------------|--------|-------------------|
| TM-1 | `project_dir` on `BuildHandoffExportOp` | Path traversal via `..` segments writing zip outside project | Task 5: reject `".." in Path(op.project_dir).parts` before any disk write (mirror `build.py:66-71`) |
| TM-2 | `zipfile.write(filepath, arcname)` | Zip-slip via crafted `arcname` containing `../` | Task 3: arcname is always `artifact_file.name` (basename only) — never a path. Assert in tests no arcname contains a path separator |
| TM-3 | `vendor` field on `BuildHandoffExportOp` | Profile-name injection / file-path escape (`vendor="../../etc/passwd"`) | Task 5: `pattern=r"^[a-z0-9_]+$"` on the Pydantic field (same as `DrcVendorOp.vendor`); `load_profile` resolution rejects unknown keys |
| TM-4 | readme.md generation | XSS / markdown injection via board name/title/company from `title_block` (user-controlled PCB content) | Task 4: readme is plain markdown written via `atomic_write`; never rendered as HTML by this codebase. Document that the readme is data, not a trusted executable context |
| TM-5 | BOM column injection | CSV formula injection via component `Value`/`Reference` fields (=cmd, @cmd) reaching vendor CSVs | Task 2: prefix any cell starting with `=`, `+`, `-`, `@`, `\t`, `\r` with a single quote `'` when writing vendor BOM CSVs (defensive; recorders in the wild) |
| TM-6 | Export file paths | Export wrappers write into `build_dir`; symlink-following could escape | Task 3: `build_dir` created with `mkdir(parents=True, exist_ok=False)` under resolved `project_dir`; wrappers receive absolute `build_dir` paths |

## Must Haves (from phase goal)

1. `build_handoff_export` op exists and is registered (HANDOFF-01)
2. One call produces: gerbers, drill, BOM, pick-and-place, STEP (optional), netlist, schematic PDF (if sch), PCB PDF (HANDOFF-02)
3. Everything bundled into a single `handoff.zip` with manifest.json + readme.md, streamed (no memory blow-up on large STEP) (HANDOFF-03, Pitfall 7)
4. readme.md contains board name, rev, date, company, layer count, dimensions, surface finish, copper weight, mask/silk color, impedance requirements, DRC/ERC results (HANDOFF-04)
5. BOM formatting is profile-driven via `ManufacturerProfile` output fields — handoff orchestrator NEVER calls `export_jlcpcb_bom` directly (HANDOFF-05, Pitfall 3)
6. Pre-handoff validation gate: DRC fail / ERC fail / vendor DRC fail → NO zip created, clear error (HANDOFF-06, Pitfall 5)
7. STEP/render inclusion configurable via `include_step` / `include_render` (HANDOFF-07)
8. `build_handoff_export(vendor="jlcpcb")` produces JLCPCB-formatted BOM/CPL via the profile (HANDOFF-08)
9. `manifest.json` includes `drc_passed`, `erc_passed`, `vendor_drc_passed`, `drc_violation_count`, `erc_violation_count` (HANDOFF-09)
