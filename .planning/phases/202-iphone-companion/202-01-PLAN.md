<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| iPhone → Mac (encrypted TCP) | Untrusted network (LAN/WAN), needs encryption |
| iPhone → CloudKit | Syncs conversation state via iCloud (encrypted by Apple) |
| iPhone → Push notifications | APNs channel, needs auth |
| iPhone → FoundationModels | Local inference, no network |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-202-01 | S | Encrypted TCP channel | mitigate | TLS 1.3 with certificate pinning (hash from QR pair), no plaintext fallback |
| T-202-02 | T | Bonjour discovery spoofing | mitigate | Verify pairing token from iCloud Keychain, QR code signed by Mac |
| T-202-03 | I | Offline queue injection | mitigate | Queue only accepts user-generated messages (no remote push), SwiftData validation |
| T-202-04 | D | Push notification spoofing | mitigate | Verify push payload signature (Mac signs with shared secret), validate thread ID |
| T-202-05 | E | FoundationModels prompt injection | mitigate | Sanitize FoundationModels prompts (only read operations, no generation), validate user input |
| T-202-06 | R | Cost tracking data leak | mitigate | Cost data is read-only from SwiftData (synced via CloudKit encryption), no sensitive data in renders |

## Adversarial Scenario

Attacker on same LAN spoofs Bonjour _kicadagent._tcp service, presents fake Mac daemon, attempts to intercept messages or inject malicious operations.

Mitigation:
1. Certificate pinning: iPhone only connects to Mac with cert hash from QR pair (unforgeable without private key)
2. Pairing token verification: iCloud Keychain token verified via CloudKit (shared secret, attacker cannot forge)
3. Operation validation: Mac daemon validates all operations (Obdurate Runtime), iPhone is thin client only
4. No plaintext fallback: TLS mandatory, certificate validation enforced
5. Read-only mode: If TLS handshake fails, iPhone shows "Pairing expired", prompts re-pair via QR

Defense in depth: Certificate pinning + token verification + operation validation + TLS-only.
</threat_model>
