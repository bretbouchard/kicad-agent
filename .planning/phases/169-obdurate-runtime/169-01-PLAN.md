<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Swift app → Obdurate Runtime | Untrusted state transitions cross here — machine validates all guards |
| Obdurate Runtime → OperationExecutor | Trusted internal boundary — executor assumes pre-validated ops |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-169-01 | Spoofing | State transition validation | mitigate | validate_transition() checks all guards, reject invalid transitions with logged error |
| T-169-02 | Tampering | Op journal integrity | mitigate | JSONL append-only with fsync, entries signed with UUID, query validates JSON structure |
| T-169-03 | Repudiation | Audit trail | mitigate | Every op logged with timestamp, actor, operation_id, requirement_id — fsync prevents loss |
| T-169-04 | Information disclosure | Requirement linkage | accept | Requirement IDs are public metadata — no secrets exposed |
| T-169-05 | Denial of Service | Escalation loop | mitigate | EscalationLadder caps at T4 (human input required), reset() clears counts on success |
| T-169-06 | Elevation of Privilege | State machine bypass | mitigate | Hard guards enforced in Python (review→execute needs approval), no bypass possible |

## Adversarial Scenario

**Scenario:** Malicious Swift app attempts to bypass state machine by sending execute phase op while in questioning phase.

**Mitigation:**
- GSDStateMachine.validate_transition() checks current_phase before allowing transition
- Invalid transition raises StateMachineError with logged error
- OpJournal logs attempted bypass (operation_id, op_type, from_phase, to_phase, result="rejected")
- EscalationLadder records failure (task="state_transition_bypass", severity=critical)

**Response:** State machine rejects transition, returns error to Swift app. Op continues in current phase until valid transition.
</threat_model>
