     - Check if artifact exists in cache (file-based: `{cache_dir}/{hash}.json`)

   - Cache dir: `{PROJECT_ROOT}/.cache/generative/`

## Adversarial Scenario

**Name:** "Cache poisoning attack via artifact hash collision"

**Scenario:** Attacker with file system access poisons cache:
- Creates malicious `.kicad_sch` file with backdoor (hidden net connecting to external pad)
- Computes artifact hash of malicious file
- Replaces legitimate cache entry with malicious file (same hash)
- Pipeline returns malicious file on cache hit (user gets compromised circuit)

**Mitigation:**
1. Validate cache file schema on load (KiCad files must parse with kiutils)
2. Hash verification: On cache hit, re-hash file and compare to cache key (detect tampering)
3. ERC/DRC gates still run on cache hit (Phase 182 validation catches malicious circuits)
4. Cache entries signed with HMAC (optional, for defense-in-depth)

Per GENOUT-03: Generation is deterministic. Cache poisoning breaks determinism (different content for same hash). Hash verification on cache load prevents this.
</threat_model>
