# Plan 233-01: Swift Schematic SVG Renderer

<objective>
Replace the kicad-cli-dependent SVG export with a Swift-native renderer.
The macOS app should be able to render any schematic entirely on-device,
without shelling out to kicad-cli. Required for App Store sandbox
compatibility (gap F1 in GAP-ANALYSIS-CURRENT.md).

## Approach

1. Audit existing `macos-app/Sources/KiCadAgent/Views/InlineRendering/SwiftSVGRenderer.swift`
2. Confirm coverage: symbols, wires, labels, power symbols, hierarchy
3. Wire into `SchematicPreviewView` for live preview
4. Add regression tests for representative schematics
5. Verify no kicad-cli subprocess calls in renderer path

## Status

SwiftSVGRenderer.swift exists from Phase 238 sweep. Audit + wire-up
work is small.
