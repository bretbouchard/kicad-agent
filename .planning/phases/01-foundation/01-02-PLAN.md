**UUID Extractor** (`src/kicad_agent/parser/uuid_extractor.py`):

Create a structured UUID extraction system:

```python
@dataclass(frozen=True)
class UUIDEntry:
    """A single extracted UUID with its structural context."""
    uuid_value: str
    parent_type: str       # e.g., 'footprint', 'pad', 'graphical', 'zone', 'via', 'segment'
    parent_index: int      # Sequential index within the parent_type
    line_number: int       # Line number in original file

@dataclass
class UUIDMap:
    """Map of all UUIDs extracted from a KiCad file, keyed by structural position."""
    entries: list[UUIDEntry]  # Ordered by appearance in file
    source_file_type: str     # 'pcb' or 'footprint'
```

`extract_uuids(content: str, file_type: str) -> UUIDMap`:
- Use regex to find all `(uuid "...")` tokens in the raw content.
- For each UUID, determine the parent type by scanning backwards from the UUID match to find the enclosing S-expression type (e.g., `(footprint`, `(pad`, `(gr_line`, `(zone`, `(via`, `(segment`).
- Record the sequential index of the parent type (0th footprint, 1st footprint, etc.).
- Return a UUIDMap with all entries in order of appearance.

`extract_uuids_from_file(path: Path, file_type: str) -> UUIDMap`:
- Read file, delegate to `extract_uuids`.

**UUID Reinjector** (`src/kicad_agent/serializer/uuid_reinjector.py`):

`reinject_uuids(serialized_content: str, uuid_map: UUIDMap) -> str`:
- For PCB and footprint files, kiutils serialization produces output without UUIDs.
- Walk the serialized content, matching structural positions to UUIDMap entries.
- Insert `(uuid "value")` tokens at the correct positions within parent S-expressions.
- The matching strategy: for each footprint/pad/graphical/zone/via/segment in the serialized output (in order), insert the corresponding UUID from the UUIDMap (matched by parent_type and parent_index).
- Return the modified content string.

Create `src/kicad_agent/validation/roundtrip.py`:

```python
@dataclass(frozen=True)
class RoundTripResult:
    is_stable: bool          # True if pass1 == pass2 (byte-identical after normalization)
    original_path: Path
    pass1_path: Path         # First parse -> serialize output
    pass2_path: Path         # Second parse -> serialize output
    file_type: str
    uuid_preserved: Optional[bool] = None  # True if UUID count matches (PCB/footprint only)
    error: Optional[str] = None
```

`round_trip_stable(path: Path, tmp_dir: Path) -> bool`:
- Determine file type from `path.suffix`.
- **First pass**: Parse with the appropriate `parse_*` function. For PCB/footprint, also call `extract_uuids` on raw_content. Serialize with the appropriate `serialize_*` function (pass UUIDMap for PCB/footprint). Output to `tmp_dir / "pass1" / path.name`.
- **Second pass**: Parse the pass1 output. For PCB/footprint, extract UUIDs from pass1's raw_content. Serialize pass1. Output to `tmp_dir / "pass2" / path.name`.
- Compare pass1 and pass2 byte-for-byte: `pass1_path.read_text() == pass2_path.read_text()`.
- Return True if identical.

`round_trip_compare(path: Path, tmp_dir: Path) -> RoundTripResult`:
- Same two-pass logic as `round_trip_stable`, but return a `RoundTripResult` with all details.
- For PCB and footprint files: compare UUID count in original vs pass2 output. Set `uuid_preserved` accordingly.
- If any step raises an exception, return `RoundTripResult(is_stable=False, ..., error=str(e))`.

The function dispatches based on file suffix:
- `.kicad_sch` -> `parse_schematic` / `serialize_schematic`
- `.kicad_pcb` -> `parse_pcb` / `serialize_pcb` (with UUID extraction)
- `.kicad_sym` -> `parse_symbol_lib` / `serialize_symbol_lib`
- `.kicad_mod` -> `parse_footprint` / `serialize_footprint` (with UUID extraction)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| raw_content -> UUID extractor | UUID extraction uses regex on raw text. Malformed S-expressions could produce incorrect UUID maps. |
| UUID map -> reinjector | Incorrect UUID mapping could inject UUIDs at wrong positions, corrupting the file. |
