<objective>
Harden silent failure modes across the execution pipeline. Batch executor stops and rolls back on individual op failure (no more partial mutation corruption). Batch tracks cumulative IR state by re-parsing after each mutation using both native and kiutils parse paths. Batch Phase 1 pre-check runs gate for ALL file types (not just .kicad_sch). Transaction cleanup logs failures loudly. Lock file creation raises LockError on write failure only (read failure remains soft warning). Repair and undo modules log failures instead of silently continuing.

Purpose: Silent failure modes mask data corruption. When a batch op fails but execution continues, the file is serialized with partial mutations -- an unrecoverable state. When lock file creation silently fails, concurrent processes can corrupt the same file without any warning. These are safety-critical fixes.

Output: Stop-and-rollback batch execution, cumulative IR tracking with both parse paths, Phase 1 gate for all file types, loud failure logging across all execution paths.
</objective>
