<objective>
Extend PreAnalysisGate into a universal pre-flight gate covering ALL execution paths (schematic, PCB, cross-file). Add PCB-specific checks (swap_footprint pad count, remove_net connectivity, move_footprint overlap), cross-file checks (lib_id validation, ERC prerequisite), and expanded schematic checks (swap_symbol pin compatibility, regenerate_wiring force requirement). Wire the gate into execute_pcb() and execute_cross_file(). Extract PCB and cross-file checks into separate modules to keep pre_analysis.py under the 800-line limit.

Purpose: Currently only execute_schematic() runs pre-flight checks. PCB and cross-file operations bypass validation entirely, allowing silent corruption. This plan closes the largest validation gap in the execution pipeline.

Output: Universal gate with file-type dispatch (extension-first routing), PCB/cross-file/schematic checks wired into all three execution paths, extracted modules for PCB and cross-file checks, comprehensive test coverage.
