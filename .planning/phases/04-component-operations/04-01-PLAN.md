<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM JSON -> Operation schema | Untrusted input from LLM crosses into validated Pydantic models |
| Operation handler -> kiutils objects | Validated intent mutates in-memory KiCad structures |
| Transaction -> filesystem | In-memory mutations are written to disk with rollback protection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | S (Spoofing) | add_component UUID | mitigate | Generate UUID v4 via Python uuid module -- never accept UUID from LLM input |
| T-04-02 | T (Tampering) | add_component library_id | mitigate | Validate library_id contains single colon; reject malformed input |
| T-04-03 | T (Tampering) | remove_component reference | mitigate | Reference must match existing component exactly -- no wildcard or pattern deletion |
| T-04-04 | I (Information disclosure) | Transaction snapshot files | accept | Snapshots use 0o600 permissions (Phase 2 H-03), temp directory, auto-cleanup |
| T-04-05 | D (Denial of service) | add_component position values | accept | PositionSpec uses float with no range constraint; KiCad handles out-of-bounds gracefully |
| T-04-06 | E (Elevation) | OperationExecutor dispatch | mitigate | Dispatch uses exact op_type matching; unknown op_type raises ValueError |
</threat_model>
