## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM JSON -> MoveComponentOp | Untrusted coordinate values from LLM cross into Position mutation |
| LLM JSON -> ModifyPropertyOp | Untrusted property name/value from LLM crosses into component metadata |
| Property mutation -> serialized file | Property changes must survive serialize/parse cycle |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-11 | T (Tampering) | modify_property Reference | mitigate | Reference changes propagate to symbol_instances -- no orphaned instances |
| T-04-12 | T (Tampering) | modify_property value | accept | Property values are strings with max_length=1024 from schema validation |
| T-04-13 | I (Information disclosure) | property values in mutation log | accept | Mutation log is in-memory only, never persisted to disk |
| T-04-14 | D (Denial of service) | move_component coordinates | accept | Precision rounding prevents floating-point explosion; out-of-bounds is KiCad's concern |
| T-04-15 | T (Tampering) | move_component position | mitigate | Precision rounding (4/6 decimals) prevents accumulated floating-point drift |
</threat_model>
