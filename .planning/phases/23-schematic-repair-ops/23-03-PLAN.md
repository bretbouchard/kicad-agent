## Objective

Implement three schematic mutation operations: `add_no_connect` (ERC-driven no-connect placement), `add_power_flag` (PWR_FLAG for power_pin_not_driven), and `snap_to_grid` (grid-snapping for off-grid wire endpoints). All three modify schematics to fix ERC violations.

Purpose: These are the core repair operations that fix the most common ERC violations. Without them, the LLM cannot autonomously repair schematics.

Output: Three new functions in `repair.py` (extending the existing module), two new Pydantic op models, handler registrations, and comprehensive tests.

1. **`snap_to_grid(ir: SchematicIR, grid_mm: float = 0.01) -> dict[str, Any]`**:
   - Import `_is_on_grid` from `kicad_agent.validation.grid_check`.
   - Call `ir.get_wire_endpoints()` to get all wire endpoint data.
   - Access `ir.schematic.graphicalItems` to get wire objects.
   - For each wire, iterate over its `points` list.
   - For each point, if `_is_on_grid(point.X, grid_mm)` is False or `_is_on_grid(point.Y, grid_mm)` is False:
     - Snap: `point.X = round(point.X / grid_mm) * grid_mm`, same for Y.
     - Record mutation: `ir._record_mutation("snap_to_grid", {"wire_index": wire_info["wire_index"]})`.
   - IMPORTANT: Group endpoints by proximity first. Collect all unique endpoint positions (rounded to 0.01mm for grouping). For each group at the same position, snap ALL endpoints in the group to the SAME grid point. This prevents connectivity breaks where two wires meet at an off-grid point and get snapped to different grid points.
   - Return `{"snapped_count": count, "grid_mm": grid_mm}`.

   Implementation strategy for connectivity preservation:
   ```python
   # First pass: collect all endpoint positions and their wire/point references
   endpoint_groups: dict[tuple[float, float], list[tuple[int, int]]] = {}
   # key = rounded position, value = list of (wire_index, point_index)
   for wire_info in wire_endpoints:
       wi = wire_info["wire_index"]
       wire = sch.graphicalItems[wi]
       for pi, point in enumerate(wire.points):
           key = (round(point.X, 2), round(point.Y, 2))
           endpoint_groups.setdefault(key, []).append((wi, pi))

   # Second pass: for each group, compute shared snap target and apply
   for key, refs in endpoint_groups.items():
       x, y = key
       if not _is_on_grid(x, grid_mm) or not _is_on_grid(y, grid_mm):
           snap_x = round(x / grid_mm) * grid_mm
           snap_y = round(y / grid_mm) * grid_mm
           for wi, pi in refs:
               wire = sch.graphicalItems[wi]
               wire.points[pi].X = snap_x
               wire.points[pi].Y = snap_y
           snapped_count += 1
           ir._record_mutation("snap_to_grid", {"group_at": [x, y], "snapped_to": [snap_x, snap_y]})
   ```

2. **`add_power_flags(ir: SchematicIR, sch_path: Path) -> dict[str, Any]`**:

## Threat Model

| Boundary | Description |
|----------|-------------|
| ERC positions -> mutation | External data from kicad-cli drives mutation targets |
| Grid snap -> connectivity | Incorrect snapping breaks wire connections |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-07 | T | Malicious ERC positions | accept | Positions come from kicad-cli (trusted tool), validated by IR methods |
| T-23-08 | D | Grid snap breaking connectivity | mitigate | Grouped endpoint snapping ensures co-located endpoints snap to same target |
| T-23-09 | T | PWR_FLAG placement without lib definition | mitigate | Check lib_symbols for power:PWR_FLAG existence; add if missing |
| T-23-10 | D | Excessive off-grid endpoints | accept | Bounded by file size; mutation tracking provides audit trail |
