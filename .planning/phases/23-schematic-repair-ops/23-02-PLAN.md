## Objective

Implement the `convert_kicad6_to_10` operation: a multi-pass format conversion that transforms KiCad 5/6 schematic files into valid KiCad 10 format. Uses section-based reassembly (NOT line-by-line editing) to avoid the paren-depth tracking bugs that caused 4 failed conversion attempts in real sessions.

Purpose: This is the highest-complexity operation in the phase. KiCad 6 format files fail to load in KiCad 10 due to multiple format differences. This operation automates the conversion so the LLM can process legacy schematics.

Output: One new module (`format_convert.py`), one new Pydantic op model, one handler registration, and a KiCad 6 test fixture.

STEP 2: Create `src/kicad_agent/ops/format_convert.py`.

Use section-based reassembly (NOT line-by-line mode tracking). The converter operates on raw text content, NOT via kiutils parsing (kiutils may fail on malformed KiCad 6 files).

Structure:

```python
"""KiCad 6 to KiCad 10 format converter -- multi-pass section-based reassembly.

Handles all known KiCad 5/6 format artifacts:
  - Single-line header -> multi-line KiCad 10 header
  - Unquoted UUIDs -> quoted UUIDs
  - Tab characters -> spaces
  - ;; comments -> removed
  - (net (code ...)) elements -> removed
  - (schematic_objects ...) wrapper -> unwrapped
  - (pins ...) wrapper inside sheets -> unwrapped
  - (at X Y) without rotation -> (at X Y 0)
  - Malformed stroke format -> corrected
  - (fields_autoplaced) -> removed
"""
```

Public API:

```python
def convert_kicad6_to_10(content: str) -> str:
    """Convert KiCad 5/6 format schematic content to KiCad 10 format.

    Uses section-based reassembly: splits content into major sections,
    applies conversion passes to each section independently, then
    reassembles. This avoids the paren-depth tracking bugs that cause
    line-by-line converters to lose hierarchical labels.

    Args:
        content: Raw text content of a KiCad 5/6 schematic file.

    Returns:
        Converted content compatible with KiCad 10 format.
    """
```

Internal helper functions (each handles one artifact):

1. `_fix_header(content: str) -> str`: Replace `(kicad_sch (version 20211123) (generator eeschema)` with `(kicad_sch\n  (version 20250114)\n  (generator "kicad-agent")\n  (generator_version "10.0.1")\n  (uuid ...)` format. Extract existing UUID from content if present, generate new one if not. Quote the UUID.

2. `_quote_uuids(content: str) -> str`: Find bare UUID patterns `[0-9a-f]{8}-[0-9a-f]{4}-...` that are NOT already inside double quotes. Replace `(uuid abc-def-...)` with `(uuid "abc-def-...")`. Use regex with lookahead/lookbehind to avoid double-quoting.

3. `_remove_semicolon_comments(content: str) -> str`: Remove lines starting with `;;` and strip ` ;; ...` inline comments. Preserve line count (replace with blank lines) to maintain line positions for debugging.

4. `_replace_tabs(content: str) -> str`: Replace all tab characters with 2 spaces (KiCad standard indent).

5. `_remove_net_elements(content: str) -> str`: Remove `(net (code ...) ...)` S-expressions. Must track paren depth to find the matching close paren for each `(net (code`.

6. `_unwrap_schematic_objects(content: str) -> str`: Remove `(schematic_objects` wrapper while keeping the children. Find `(schematic_objects\n`, track to matching `)`, extract inner content, replace wrapper with children.

7. `_unwrap_pins_wrapper(content: str) -> str`: Remove `(pins\n ... )` wrapper inside sheet elements. Extract pin children and place them as direct children of the sheet element.

8. `_fix_missing_rotation(content: str) -> str`: Find `(at X Y)` with exactly 2 numeric values and no third value, append ` 0` rotation: `(at X Y 0)`. Use the same regex pattern as format_check.py (`\(at\s+[-\d.]+\s+[-\d.]+\s*\)`).

9. `_fix_stroke_format(content: str) -> str`: Fix malformed `(stroke (width X)) (type Y))` to correct `(stroke (width X) (type Y))`. Detect the split pattern and merge.

10. `_remove_fields_autoplaced(content: str) -> str`: Remove `(fields_autoplaced)` elements.

11. `_remove_nets_section(content: str) -> str`: Remove any top-level `(nets ...)` section.

12. `_fix_sheet_instances(content: str) -> str`: Ensure sheet_instances has correct format for KiCad 10.

13. `_add_embedded_fonts(content: str) -> str`: Ensure `(embedded_fonts)` element exists before the closing paren.

The main `convert_kicad6_to_10` function calls these in order:

```python
def convert_kicad6_to_10(content: str) -> str:
    # Phase 1: Pre-processing (text cleanup)
    content = _remove_semicolon_comments(content)
    content = _replace_tabs(content)

    # Phase 2: Header fix
    content = _fix_header(content)

    # Phase 3: UUID quoting
    content = _quote_uuids(content)

    # Phase 4: Legacy element removal
    content = _remove_net_elements(content)
    content = _unwrap_schematic_objects(content)
    content = _unwrap_pins_wrapper(content)
    content = _remove_fields_autoplaced(content)
    content = _remove_nets_section(content)

    # Phase 5: Format fixes
    content = _fix_missing_rotation(content)
    content = _fix_stroke_format(content)
    content = _fix_sheet_instances(content)

    # Phase 6: Final cleanup
    content = _add_embedded_fonts(content)

    return content
```
