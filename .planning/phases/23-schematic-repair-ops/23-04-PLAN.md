## Objective

Implement the `rebuild_root_sheet` operation: generates a root schematic file from sub-sheet hierarchical labels. Reads all sub-sheets referenced by the current root, extracts hierarchical labels, classifies them as input/output/bidirectional, positions them on LEFT (inputs) or RIGHT (outputs) of the root sheet, generates net labels at pin base positions, and writes the resulting root sheet.

Purpose: This is the most architecturally complex operation. The real-world use case is rebuilding the root sheet of a hierarchical design after sub-sheets have been modified (pins added, renamed, or deleted). The LLM needs this to keep hierarchical schematics in sync.

Output: One new module (`root_sheet.py`), one new Pydantic op model, handler registration, and comprehensive tests.

2. **`rebuild_root_sheet(root_sch_path: Path) -> list[RootSheetResult]`**:

   This function reads the root schematic, discovers sub-sheets, extracts hierarchical labels from each sub-sheet, and rebuilds the root sheet with proper sheet pins and net labels.

   Implementation steps:

   a. Parse the root schematic: `parse_result = parse_schematic(root_sch_path)`, `ir = SchematicIR(_parse_result=parse_result)`, `sch = ir.schematic`.

   b. For each `sheet` in `sch.sheets`:
      - Get `sheet.fileName.value` (the sub-sheet file path).
      - Resolve the sub-sheet path: `sub_path = root_sch_path.parent / sheet_file_name`.
      - If sub-sheet does not exist, skip with a warning log.
      - Parse the sub-sheet: `sub_result = parse_schematic(sub_path)`, `sub_ir = SchematicIR(_parse_result=sub_result)`.
      - Extract hierarchical labels from sub-sheet: `[label.text for label in sub_ir.schematic.hierarchicalLabels if label.text]`.
      - For each hierarchical label, determine its direction from `label.shape` (or `label.connectionType`):
        - "input" -> LEFT side
        - "output" -> RIGHT side
        - "bidirectional" -> LEFT side (default)
        - "passive" -> LEFT side (default)
        - Others -> LEFT side

   c. Position pins on the sheet:
      - Use the existing sheet's position (`sheet.position.X`, `sheet.position.Y`) and size (`sheet.size.width`, `sheet.size.height`).
      - LEFT pins: evenly spaced along the left edge of the sheet, starting at Y = sheet.position.Y + 5.08 (first pin offset), spacing 2.54mm.
      - RIGHT pins: evenly spaced along the right edge, starting at same Y offset.
      - Pin X position: LEFT pins at sheet.position.X, RIGHT pins at sheet.position.X + sheet.size.width.

   d. Generate net labels at pin base positions:
      - For each placed pin, create a local net label at the pin's base position with the same text as the pin name.
      - The label connects the pin to the net in the root sheet.

   e. Clear the existing sheet pins and replace with the new ones derived from sub-sheet hierarchical labels. This is the key mutation -- the sheet's pin list is rebuilt from the actual sub-sheet content.

   f. Record mutation via `ir._record_mutation("rebuild_root_sheet", {...})`.

   g. Build and return `RootSheetResult` for this sheet.

   Return a list of `RootSheetResult`, one per processed sub-sheet.

   IMPORTANT: This function modifies the IR in place (clears and rebuilds sheet pins). The caller (executor handler) is responsible for serializing the result.
