# Phase 248: Reconcile volta-12b-v2 naming (Storage vs app)

**Status:** Planned (2026-07-14)
**Bead:** volta-him
**Priority:** P1
**Effort:** Small (~2-4 hours)
**Plans:** TBD via /gsd-plan-phase 248

## Goal

Eliminate the naming divergence between the canonical Volta v2 adapter on `/Volumes/Storage` (`kicad-vision-v2`) and what the macOS app/HuggingFace expects (`volta-pcb-adapter-v2`). Fix the broken symlink at `volta/volta-12b-v2`. Document canonical paths.

## Problem Statement

Discovered 2026-07-14 during Phase 246 (Volta v2 eval harness) cleanup:

- **Storage drive** (`/Volumes/Storage/models/volta/adapters/`) contains:
  - `kicad-vision-v2` (PEFT, likely the canonical)
  - `kicad-vision-v2-peft`
  - `kicad-vision-v2-mlx`
  - 7 other older variants
- **HuggingFace** has: `bretbouchard/volta-pcb-adapter-v2` (referenced in `ModelDownloader.swift:36`)
- **macOS app** expects adapter at: `~/Library/Application Support/VoltaPCB/models/volta-pcb-adapter-v2/` (per `ModelDownloader.swift:107-109`)
- **Repo symlink** `volta/volta-12b-v2` is **broken** (empty target, created 2026-07-14)
- **Phase 246 eval harness** expects a resolvable path for the canonical adapter

This naming drift is technical debt from earlier phase planning (Vision model → Volta rename happened mid-stream).

## Requirements

- **REQ-248-01:** Verify `kicad-vision-v2` on Storage is byte-equivalent (or PEFT-config-equivalent) to `volta-pcb-adapter-v2` on HuggingFace
- **REQ-248-02:** Fix the broken symlink at `volta/volta-12b-v2` to point to the canonical location
- **REQ-248-03:** Document canonical adapter paths in `.claude/CLAUDE.md` (Storage, HF, app sandbox)
- **REQ-248-04:** Phase 246 eval harness can resolve the canonical adapter without errors
- **REQ-248-05:** All existing tests still pass (no regression)

## Key Work

1. **Verification step (req 01):**
   - Read `adapter_config.json` from both sources — compare `base_model_name_or_path`, `r`, `alpha`, `target_modules`
   - Compute SHA256 of `adapter_model.safetensors` from both sources
   - If equal → alias/symlink fix is safe
   - If different → must investigate further (could be v2 vs v2.1 divergence)
2. **Symlink fix (req 02):**
   - Either:
     - `ln -s /Volumes/Storage/models/volta/adapters/kicad-vision-v2 volta/volta-12b-v2` (if equivalent)
     - Or create a `volta/volta-12b-v2/README.md` documenting the actual path
3. **Documentation (req 03):**
   - Add `## Canonical Adapter Paths` section to `.claude/CLAUDE.md`
4. **Test integration (req 04-05):**
   - Re-run Phase 246 eval harness `verify_hf_availability.py` script
   - Run `swift test` to confirm no regressions

## Out of Scope (deferred to Phase 249)

- Project-wide rename `volta` → `Volta` (separate phase, much larger blast radius)
- Renaming `/Volumes/Storage/.../kicad-vision-v2/` directory itself
- Backfilling the HF repo name (would break all existing users)

## Success Criteria

1. `kicad-vision-v2` (Storage) and `volta-pcb-adapter-v2` (HF) verified equivalent via SHA256 or PEFT config
2. `volta/volta-12b-v2` symlink resolves to a real adapter directory
3. `.claude/CLAUDE.md` has a `## Canonical Adapter Paths` section
4. Phase 246 eval harness `verify_hf_availability.py` exits 0
5. All `swift test --no-parallel` tests still pass
