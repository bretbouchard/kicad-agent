    Follow the MazeDataset pattern exactly (frozen dataclass + JSONL + split):

    ```python
    """Real-world PCB board dataset for GRPO training.

    RW-04: SHA256 deduplication and quality filtering for real board data.
    RW-05: JSONL serialization compatible with Phase 9 GRPO training pipeline.

    Follows the MazeDataset pattern: frozen dataclass + JSONL streaming +
    train/val/test split. Designed for 50k+ board scale with streaming
    processing (parse one, serialize, discard, move to next).
    """
    ```

    **RealBoardSample** (frozen dataclass) -- mirrors BoardGraphResult fields exactly:
    - `sample_id: int`
    - `repo_url: str`
    - `repo_name: str`
    - `schematic_path: str`
    - `pcb_path: str`
    - `component_count: int`
    - `net_count: int`
    - `layer_count: int`
    - `board_width_mm: float`
    - `board_height_mm: float`
    - `difficulty: str`
    - `board_hash: str`
    - `graph_json: str`
    - `spatial_summary_json: str`

    **Serialization helpers** (follow _sample_to_dict / _dict_to_sample pattern from dataset.py):
    - `_sample_to_dict(s: RealBoardSample) -> dict` -- straightforward field copy (all fields are already primitives/strings)
    - `_dict_to_sample(d: dict) -> RealBoardSample` -- construct RealBoardSample from dict

    **RealBoardDataset** (mutable dataclass, mirrors MazeDataset):
    - `samples: list[RealBoardSample]` with default_factory=list
    - `metadata: dict` with default_factory=dict
    - `__len__` -> len(self.samples)
    - `to_jsonl(self, path: Path) -> int` -- write one JSON per line, create parent dirs, return line count
    - `from_jsonl(path: Path) -> RealBoardDataset` -- static method, read lines, parse JSON, construct samples
    - `split(self, train=0.8, val=0.1, test=0.1) -> tuple[RealBoardDataset, RealBoardDataset, RealBoardDataset]` -- deterministic split by sample_id order (same as MazeDataset), validate fractions sum to 1.0

    **Quality filter** (module-level functions):
    - `MIN_COMPONENTS = 3` -- boards with fewer components are trivially simple
    - `MIN_NETS = 2` -- boards with fewer nets lack meaningful connectivity
    - `is_valid_sample(sample: RealBoardSample) -> bool` -- returns True if sample.component_count >= MIN_COMPONENTS and sample.net_count >= MIN_NETS
    - `filter_quality(samples: list[RealBoardSample]) -> list[RealBoardSample]` -- returns filtered list, logs count removed

    **Deduplication**:
    - `dedup_by_hash(samples: list[RealBoardSample]) -> list[RealBoardSample]` -- uses board_hash field (already SHA256 from raw content), keeps first occurrence, logs duplicate count. O(n) via set lookup.

    **Pipeline integration**:
    - `run_pipeline(token: str, staging_dir: Path, max_repos: int = 500, output_dir: Path | None = None) -> RealBoardDataset` -- end-to-end pipeline:
      1. Create GithubDiscovery with token
      2. Call discover_pairs(max_repos) to get repos with pairs
      3. Create FileFetcher with staging_dir
      4. For each (repo_info, pairs), for each pair:
         a. fetch_pair to download both files
         b. build_board_graph on the downloaded files
         c. If result is not None, convert BoardGraphResult to RealBoardSample and add to list
      5. Dedup samples by board_hash
      6. Filter for quality
      7. Build RealBoardDataset with metadata (n_discovered, n_parsed, n_deduped, n_filtered, difficulty_counts)
      8. If output_dir provided, write train.jsonl, val.jsonl, test.jsonl via split()
      9. Return dataset

    Streaming: Process one board at a time. After build_board_graph returns BoardGraphResult, convert to RealBoardSample and append. Do NOT accumulate BoardGraphResult objects (they hold live parsed IR). Only RealBoardSample (all strings/ints/floats) accumulates.
