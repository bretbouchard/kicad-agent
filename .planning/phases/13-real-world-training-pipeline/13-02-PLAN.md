    **1. Create src/kicad_agent/training/graph_builder.py:**

    ```python
    """Board graph construction from schematic+PCB file pairs.

    RW-02: Parses schematic+PCB pairs into unified networkx graphs with
    component nodes and net edges.

    RW-03: Extracts spatial features from PCB and attaches them to graph
    nodes as attributes (bounding box, position, rotation).

    Composes existing infrastructure:
    - SchematicIR for schematic parsing (component references, values)
    - PcbIR for PCB parsing (footprints, pads, traces)
    - NetGraph for connectivity (net -> pad adjacency)
    - spatial/extractor for spatial primitives (points, boxes, paths)
    """
    ```

    Implement the following:

    **BoardGraphResult** (frozen dataclass):
    - `sample_id: int` -- sequential index
    - `repo_url: str` -- source repo URL
    - `repo_name: str` -- source repo full name
    - `schematic_path: str` -- path within repo
    - `pcb_path: str` -- path within repo
    - `component_count: int` -- number of component nodes in graph
    - `net_count: int` -- number of unique nets in graph
    - `layer_count: int` -- number of PCB layers (from PcbIR)
    - `board_width_mm: float` -- PCB width from general section
    - `board_height_mm: float` -- PCB height from general section
    - `difficulty: str` -- "easy" (<10 components), "medium" (10-50), "hard" (50+)
    - `board_hash: str` -- SHA256 of raw schematic+pcb content bytes (before kiutils parsing)
    - `graph_json: str` -- networkx graph as JSON string (node-link-data format)
    - `spatial_summary_json: str` -- JSON string with spatial feature counts and bounding box extents

    **build_board_graph** function:
    ```python
    def build_board_graph(
        sch_path: Path,
        pcb_path: Path,
        sample_id: int = 0,
        repo_url: str = "",
        repo_name: str = "",
        sch_repo_path: str = "",
        pcb_repo_path: str = "",
    ) -> BoardGraphResult | None:
        """Build unified graph from schematic+PCB pair.

        Parses both files, constructs connectivity graph, extracts spatial
        features, merges into a single networkx Graph, and serializes.

        Returns None if parsing fails (format version, corrupt file, etc).
        All exceptions are caught and logged -- never crashes the pipeline.
        """
    ```

    Implementation steps inside build_board_graph:
    1. Read raw file contents (for hashing BEFORE kiutils parsing, per STATE.md decision about raw content read)
    2. Compute SHA256 hash: `hashlib.sha256(sch_bytes + pcb_bytes).hexdigest()`
    3. Parse schematic: `parse_schematic(sch_path)` -> `SchematicIR(_parse_result=result)`
    4. Parse PCB: `parse_pcb(pcb_path)` -> extract UUIDs -> `PcbIR(_parse_result=result, _uuid_map=uuid_map)`
    5. Build connectivity: `NetGraph.from_pcb_ir(pcb_ir)`
    6. Extract spatial: `extract_all(pcb_ir)`
    7. Build unified `nx.Graph()`:
       a. Add component nodes from `sch_ir.components`: each symbol becomes a node keyed by reference designator, with attributes: `node_type="component"`, `value`, `footprint`
       b. Add net edges from `net_graph._net_index`: for each net (excluding empty/unconnected), connect all pads sharing the net. Edges keyed by component references (not pad refs), with attribute `net=net_name`
       c. Add spatial attributes to nodes: for each footprint in pcb_ir, find the matching component node by reference, add `x_mm`, `y_mm`, `rotation_deg`, `bbox_width_mm`, `bbox_height_mm` attributes from footprint position and bounding box
    8. Serialize graph: `nx.node_link_data(G)` -> `json.dumps(...)` as graph_json
    9. Build spatial summary: counts of points/boxes/paths/regions, min/max x/y extents
    10. Determine difficulty from component_count
    11. Extract board dimensions from PCB general section (or default 0.0 if unavailable)
    12. Return BoardGraphResult

    Wrap the entire function body in try/except Exception. On failure, log the error with `logger.warning` and return None.

    Key: Hash raw bytes BEFORE kiutils parsing. STATE.md documents that kiutils serialization is non-deterministic -- only raw content hashing provides stable deduplication.
