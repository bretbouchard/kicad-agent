    Create `scripts/ingest_cognee.py` as a standalone Python script following the `scripts/validate_op_count.py` pattern:

    ```python
    #!/usr/bin/env python3
    """Ingest KiCad reference documents into Cognee knowledge graph.

    Reads docs/*.md and uses Cognee MCP tools to build a searchable
    knowledge base for Claude Code sessions.

    Usage:
        python scripts/ingest_cognee.py
    """
    from __future__ import annotations

    import json
    import logging
    import sys
    from pathlib import Path

    ROOT = Path(__file__).resolve().parent.parent
    DATASET_NAME = "kicad-agent-reference"

    DOC_FILES = [
        "kicad_agent_reference.md",
        "pcb_editor_reference.md",
        "gerbview_reference.md",
        "kicad_docs.md",
    ]

    # Verification queries with expected content indicators
    VERIFICATION_QUERIES = [
        ("pin at coordinate", "kicad_agent_reference"),
        ("grid snap schematic", "kicad_agent_reference"),
        ("working with footprints", "pcb_editor_reference"),
        ("copper zones", "pcb_editor_reference"),
        ("Gerber", "gerbview_reference"),
        ("Schematic Editor", "kicad_docs"),
    ]

    logging.basicConfig(level=logging.INFO, format="%(levelname)s: %(message)s")
    logger = logging.getLogger(__name__)
    ```

    The script MUST:

    1. Resolve `docs/` as `ROOT / "docs"` (following validate_op_count.py pattern: `ROOT = Path(__file__).resolve().parent.parent`)
    2. Define `DOC_FILES` list with exactly these 4 filenames: `"kicad_agent_reference.md"`, `"pcb_editor_reference.md"`, `"gerbview_reference.md"`, `"kicad_docs.md"`
    3. Define `DATASET_NAME = "kicad-agent-reference"` (per RESEARCH Pitfall 4 -- avoid polluting main_dataset)
    4. Define a `ingest_doc(doc_name: str, content: str) -> bool` function that:
       - Calls Cognee MCP tool `mcp__cognee-local__remember` with `data=content, dataset_name=DATASET_NAME`
       - If that fails (tool not found), falls back to `mcp__cognee-local__add` followed by `mcp__cognee-local__cognify`
       - Returns True on success, False on failure
       - Logs which API path was used (v1.0 vs legacy)
    5. Define a `verify_ingestion() -> bool` function that:
       - Runs each query from `VERIFICATION_QUERIES` using `mcp__cognee-local__recall` (or `mcp__cognee-local__search` as fallback)
       - Checks that results are non-empty
       - Returns True if all queries return results
    6. Define `main() -> int` that:
       - Iterates over DOC_FILES, reads each file, calls ingest_doc
       - Tracks per-doc status: ingested/failed
       - After all ingestion, runs verify_ingestion()
       - Prints summary: "Ingested N/4 docs. Verification: PASS/FAIL"
       - Returns 0 (all success), 1 (partial failure), 2 (import/setup error)

    CRITICAL: Since this script runs outside the Claude Code MCP context, the actual Cognee MCP calls cannot be made via Python subprocess. Instead, the script should:
    - Define the ingestion LOGIC clearly as callable functions
    - Print the MCP tool calls as JSON that can be manually executed or piped to Claude
    - Provide a `--dry-run` flag that prints what would be ingested without calling MCP
    - Provide clear documentation comments explaining how to use with Claude Code MCP

    This is a "documented process" approach per D-01: "Create a helper script or documented manual process so re-ingestion is easy when docs update".
