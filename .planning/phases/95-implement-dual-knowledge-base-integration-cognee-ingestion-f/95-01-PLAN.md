    ```python
    #!/usr/bin/env python3
    """Ingest KiCad reference documents into Cognee knowledge graph.

    Reads docs/*.md and outputs JSON MCP command payloads for Cognee
    ingestion. Pipe the output to Claude Code or save to a file for
    manual execution.

    Usage:
        python scripts/ingest_cognee.py                  # Print JSON to stdout
        python scripts/ingest_cognee.py --output cmds.json  # Write to file
    """
    from __future__ import annotations

    import json
    import logging
    import sys
    from pathlib import Path

    ROOT = Path(__file__).resolve().parent.parent
    DATASET_NAME = "kicad-agent-reference"

    DOC_FILES = [
        "kicad_agent_reference.md",
        "pcb_editor_reference.md",
        "gerbview_reference.md",
        "kicad_docs.md",
    ]

    # Verification queries with expected content indicators
    VERIFICATION_QUERIES = [
        ("pin at coordinate", "kicad_agent_reference"),
        ("grid snap schematic", "kicad_agent_reference"),
        ("working with footprints", "pcb_editor_reference"),
        ("copper zones", "pcb_editor_reference"),
        ("Gerber", "gerbview_reference"),
        ("Schematic Editor", "kicad_docs"),
    ]

    logging.basicConfig(level=logging.INFO, format="%(levelname)s: %(message)s")
    logger = logging.getLogger(__name__)
    ```

    The script MUST define these pure functions and a main() dispatch:

    1. Resolve `docs/` as `ROOT / "docs"` (following validate_op_count.py pattern: `ROOT = Path(__file__).resolve().parent.parent`)
    2. Define `DOC_FILES` list with exactly these 4 filenames
    3. Define `DATASET_NAME = "kicad-agent-reference"` (per RESEARCH Pitfall 4 -- avoid polluting main_dataset)

    4. Define `ingest_doc(doc_name: str, content: str) -> dict[str, object]` as a PURE function that:

       - The `data` field includes a label prefix: `f"[Source: {doc_name}]\n\n{content}"`
       - Does NOT call any MCP tool -- it only returns the JSON payload
       - Raises ValueError if content is empty

    5. Define `verify_ingestion() -> list[dict[str, object]]` as a PURE function that:
       - Returns a list of dicts, one per verification query: `{"tool": "recall", "query": query, "datasets": [DATASET_NAME]}`
       - Does NOT call any MCP tool -- it only returns the JSON payloads

    6. Define `main() -> int` that:
       - Accepts `--output PATH` argument (default: None, meaning stdout)
       - Iterates over DOC_FILES, reads each file, calls ingest_doc()
       - Collects all ingestion payloads, then appends verification payloads
       - Writes all payloads as JSON (one JSON object per line, JSONL format) to stdout or to the --output file
       - Tracks per-doc status: ready/failed
       - Prints summary to stderr: "Prepared N/4 ingestion payloads + M verification queries"
       - Returns 0 (all success), 1 (partial failure), 2 (import/setup error)

    The script is a JSON payload generator per D-01: "Create a helper script or documented manual process so re-ingestion is easy when docs update". The generated JSON can be piped to Claude Code for actual MCP execution, or saved to a file for batch processing.
