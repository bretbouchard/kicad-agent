    **1. Create src/kicad_agent/placement/model.py:**

    - `BipartiteAttentionLayer(nn.Module)`:
      Multi-head attention layer for bipartite component-net message passing.
      - `__init__(comp_dim: int, net_dim: int, n_heads: int = 4)`: Create comp_proj (Linear), net_proj (Linear), and nn.MultiheadAttention(comp_dim, n_heads, batch_first=True). Project net features into comp dimension space.
      - `forward(comp_features, net_features, adj_matrix) -> comp_features`:
        1. Project net features to comp_dim: `projected_nets = self.net_proj(net_features)`
        2. Compute attention: comp nodes query against net keys/values using nn.MultiheadAttention
        3. Apply residual connection: `return comp_features + attn_out`
        Adjacency matrix masks attention (only attend to connected net nodes). Use the adj_matrix to create an attention mask where disconnected positions are set to -inf.

    - `PlacementModel(nn.Module)`:
      - `__init__(comp_feature_dim=32, net_feature_dim=16, hidden_dim=128, n_layers=3, n_heads=4)`:
        1. comp_embed: Linear(comp_feature_dim, hidden_dim)
        2. net_embed: Linear(net_feature_dim, hidden_dim)
        3. attn_layers: ModuleList of n_layers BipartiteAttentionLayer(hidden_dim, hidden_dim, n_heads)
        4. layer_norms: ModuleList of n_layers LayerNorm(hidden_dim)
        5. x_head, y_head, rot_head: Linear(hidden_dim, 1) each

      - `forward(comp_features, net_features, adj_matrix, board_w, board_h) -> predictions`:
        1. Embed: `h_comp = relu(comp_embed(comp_features))`, `h_net = relu(net_embed(net_features))`
        2. Message passing: for each attention layer, apply BipartiteAttentionLayer then LayerNorm then ReLU
        3. Predict per-component: x = sigmoid(x_head(h_comp)).squeeze(-1) * board_w, y = sigmoid(y_head(h_comp)).squeeze(-1) * board_h, rot = (sigmoid(rot_head(h_comp)).squeeze(-1) * 360) - 180
        4. Return torch.stack([x, y, rot], dim=-1) shape (batch, n_comp, 3)

        Input shapes: comp_features (batch, n_comp, 32), net_features (batch, n_net, 16), adj_matrix (batch, n_comp, n_net), board_w (batch,), board_h (batch,).

    **2. Create src/kicad_agent/placement/predict.py:**

    - `PlacementPrediction(frozen=True)`: Dataclass with:
      - positions: dict[str, tuple[float, float, float]]  # ref -> (x, y, rotation_degrees)
      - raw_output: numpy.ndarray  # (n_comp, 3) raw model output
      - model_confidence: float  # average softmax entropy across output heads

    - `PlacementPredictor`:
      - `__init__(model_path: Path | None = None, device: str = "cpu")`:
        1. Initialize PlacementModel with default architecture
        2. If model_path provided and exists, load state_dict with torch.load (weights_only=True for security)
        3. Set model to eval mode
        4. Store device

      - `predict(graph: PlacementGraph) -> PlacementPrediction`:
        1. Extract features: comp_features, net_features, adj_matrix from PlacementGraph methods
        2. Convert to torch tensors, add batch dimension, move to device
        3. Run model forward pass with torch.no_grad()
        4. Convert output to numpy, remove batch dimension
        5. Map predictions back to component references from graph.component_nodes()
        6. Return PlacementPrediction

      - `is_ready -> bool`: Returns True if model is loaded and parameters initialized.

      Lazy torch import: Wrap torch imports at function level so the module can be imported without torch installed (matches Phase 9 pattern).

    **2. Create src/kicad_agent/placement/training/dataset.py:**

    - `PlacementSample(frozen=True)`: Dataclass with:
      - sample_id: int
      - board_width: float
      - board_height: float
      - n_components: int
      - components_json: str  # JSON-serialized list of ComponentSpec dicts
      - nets_json: str  # JSON-serialized list of NetSpec dicts
      - optimal_positions_json: str  # JSON-serialized dict of ref -> (x, y, rot)
      - difficulty: str  # "easy", "medium", "hard"
      - board_hash: str  # SHA256 for deduplication

      - `to_dict() -> dict`: Serialize to plain dict.
      - `@classmethod from_dict(d: dict) -> PlacementSample`: Deserialize.

    - `PlacementDataset`:
      - `samples: list[PlacementSample]`
      - `to_jsonl(path: Path)`: Write each sample as JSON line (matches MazeDataset pattern).
      - `@classmethod from_jsonl(path: Path) -> PlacementDataset`: Read JSONL.
      - `__len__()`: Return len(samples).

    - `generate_placement_samples(n_samples: int, board_width: float, board_height: float, seed_base: int = 42) -> PlacementDataset`:
      Generate synthetic placement training data:
      1. For each sample, pick random component count (3-15 for easy, 10-30 for medium, 20-50 for hard).
      2. Generate random ComponentSpec instances with varied types (U, R, C, L, J, Q).
      3. Generate random NetSpec instances connecting components.
      4. Compute "optimal" positions using scipy.optimize.dual_annealing:
         - Objective: minimize HPWL (half-perimeter wirelength) + overlap penalty
         - Bounds: each component (x, y) in [margin, board_dim - margin]
         - Fixed seed for reproducibility
      5. Compute difficulty grade based on component count and net density.
      6. Compute board_hash from components + nets for deduplication.
      7. Return PlacementDataset.

      Cap at 100,000 samples max (matches MazeDataset _MAX_SAMPLES safety pattern).

    **3. Create src/kicad_agent/placement/training/reward.py:**

    - `compute_hpwl(positions: dict[str, tuple[float, float, float]], graph: PlacementGraph) -> float`:
      Half-Perimeter Wirelength: for each net, compute the bounding box of all connected component positions, sum the half-perimeters. Lower is better. Use only x, y from the tuple.

    - `compute_overlap_area(positions: dict[str, tuple[float, float, float]], graph: PlacementGraph, min_clearance: float = 1.0) -> float`:
      Sum of Shapely intersection areas between component bounding boxes (using estimated sizes from features). Convert each position + estimated_size to SpatialBox, compute pairwise intersections.

    - `compute_edge_penalty(positions: dict[str, tuple[float, float, float]], board_width: float, board_height: float, margin: float = 2.0) -> float`:
      Sum of penalties for components within margin of board edges. Each violation adds 1.0.

    - `compute_placement_loss(positions: dict[str, tuple[float, float, float]], graph: PlacementGraph, board_width: float, board_height: float, min_clearance: float = 1.0) -> dict[str, float]`:
      Returns {"hpwl": float, "overlap_area": float, "edge_penalty": float, "total_loss": float}.
      total_loss = hpwl + 10.0 * overlap_area + 5.0 * edge_penalty.
      This provides the component metrics for GRPO reward computation.

    - `placement_reward(predicted: dict[str, tuple[float, float, float]], reference: dict[str, tuple[float, float, float]] | None, graph: PlacementGraph, board_width: float, board_height: float) -> float`:
      Compute reward in [0, 1] for GRPO training:
      - Position accuracy (0.3 weight): if reference exists, average distance normalized by board diagonal
      - Wirelength quality (0.4 weight): HPWL normalized by board diagonal * component count
      - Clearance score (0.3 weight): 1.0 if no violations, decreasing with violation count
      Returns 0.3 * accuracy + 0.4 * wire_score + 0.3 * clearance_score.

    **4. Create src/kicad_agent/placement/training/train.py:**

    - `PlacementTrainConfig`:
      - n_epochs: int = 1
      - batch_size: int = 4
      - learning_rate: float = 1e-4
      - group_size: int = 4
      - seed: int = 42
      - checkpoint_dir: str = "checkpoints/placement/"

    - `PlacementTrainer`:
      - `__init__(model: PlacementModel, config: PlacementTrainConfig | None = None)`:
        Store model and config. Create a copy of model as ref_model (for KL penalty).
      - `train(dataset: PlacementDataset) -> dict`:
        1. For each epoch, iterate over dataset in batches
        2. For each batch sample: build PlacementGraph, run model prediction, compute placement_reward
        3. Generate group: for each sample, create group_size predictions with different random initializations
        4. Compute group-relative advantages using GRPOTrainer.compute_group_advantages()
        5. Backpropagate: compute loss = -(advantages * log_prob).mean() + kl_penalty
        6. Optimizer step with AdamW, gradient clipping at 1.0
        7. Log metrics (loss, reward_mean, hpwl)
        8. Return training history dict
      - `_compute_model_reward(model, graph, board_w, board_h, reference_positions) -> float`: Helper to run model + compute reward.
