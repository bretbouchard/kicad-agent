<objective>
Build the interactive placement mode with constraint propagation and the hybrid placement engine that ties together all placement components into a single end-to-end pipeline.

Purpose: Interactive placement lets users fix some components and let AI place the rest -- the most practical usage pattern for real PCB design. The hybrid engine provides the public API that consumers (Phase 15 AI Generation Wiring, CLI) call: it tries ML prediction first, falls back to rule-based placement if the model is unavailable, validates the result with DRC checks, and scores it. This addresses PLACE-05 (interactive mode) and completes the phase by wiring all components together.

Output: placement/interactive.py (constraint propagation + SA refinement), placement/engine.py (hybrid engine); updated barrel exports; passing tests.
</objective>

  <action>
    **1. Create src/kicad_agent/placement/interactive.py:**

    - `ConstraintSet`: Dataclass holding placement constraints:
      - fixed_positions: dict[str, tuple[float, float, float]]  # User-placed components (immutable)
      - keepout_zones: list[tuple[float, float, float, float]]  # (x1, y1, x2, y2) forbidden regions
      - min_clearance: float = 1.0
      - max_sa_iterations: int = 500

    - `interactive_placement(graph: PlacementGraph, constraints: ConstraintSet, predictor: PlacementPredictor | None = None) -> dict[str, tuple[float, float, float]]`:
      Place remaining (free) components respecting user-fixed positions.

      Algorithm:
      1. Partition components into fixed (from constraints.fixed_positions) and free (all others).
      2. If no free components, return fixed_positions immediately.
      3. Get initial positions for free components:
         a. If predictor is provided and is_ready: run predictor.predict(graph) and extract positions for free components
         b. Otherwise: use grid fallback (distribute free components evenly across board, avoiding fixed positions)
      4. Run scipy.optimize.dual_annealing to refine free component positions:
         a. Build parameter vector: [x1, y1, x2, y2, ...] for each free component
         b. Bounds: each (x, y) in [min_clearance, board_dim - min_clearance]
         c. Objective function:
            - Combine fixed + current free positions into single dict
            - Compute HPWL from net topology (using PlacementScorer's compute_hpwl_score)
            - Add clearance penalty: 10.0 * count of violations with fixed components
            - Add keepout penalty: 20.0 * count of free components inside keepout zones
            - Return total objective value
         d. x0 from model predictions or grid fallback
         e. maxiter from constraints.max_sa_iterations
         f. seed=42 for reproducibility
      5. Build final positions dict: merge fixed + optimized free positions.
      6. Validate with PlacementValidator.
      7. Return final positions dict.

      Safety:
      - Fixed components NEVER appear in the SA parameter vector (pitfall 5 from RESEARCH.md)
      - Validate free component bounds before passing to SA
      - Keepout zone violations penalized in objective

    - `suggest_placements(graph: PlacementGraph, fixed_positions: dict[str, tuple[float, float, float]], predictor: PlacementPredictor | None = None) -> dict[str, tuple[float, float, float]]`:
      Convenience wrapper: build ConstraintSet from fixed_positions, call interactive_placement().

    - `PlacementOutput(BaseModel)`: Pydantic model for placement result:
      - positions: dict[str, tuple[float, float, float]]  # ref -> (x, y, rotation)
      - score: float  # Composite quality score [0, 1]
      - hpwl: float  # Half-perimeter wirelength
      - valid: bool  # Passes DRC
      - violations: list[dict]  # Violation details
      - source: str  # "ml_prediction", "ml_refined", "rule_based", "interactive"
      - component_scores: dict[str, float]  # Per-component quality contribution

    - `HybridPlacementEngine`:
      - `__init__(model_path: Path | None = None, device: str = "cpu")`:
        1. Store PlacementEngine (rule-based) for fallback
        2. Try to initialize PlacementPredictor from model_path
        3. Store PlacementValidator and PlacementScorer instances (created per-request based on board dimensions)

      - `place(request: PlacementRequest) -> PlacementOutput`:
        1. Build PlacementGraph from components + nets + board dimensions
        2. If request.fixed_positions is non-empty:
           a. Build ConstraintSet from fixed positions + keepout zones
           b. Call interactive_placement() with graph, constraints, predictor
           c. source = "interactive"
        3. Else if request.use_ml and predictor.is_ready:
           a. Call predictor.predict(graph)
           b. source = "ml_prediction"
           c. If request.refine_sa: run SA refinement on all positions, source = "ml_refined"
        4. Else:
           a. Fall back to rule-based PlacementEngine.place_components()
           b. Convert result to (x, y, 0.0) format (no rotation from rule engine)
           c. source = "rule_based"
        5. Validate positions with PlacementValidator
        6. Score positions with PlacementScorer
        7. Build and return PlacementOutput

      - `place_components_simple(components: list[ComponentSpec], board_width: float, board_height: float) -> PlacementOutput`:
        Simplified API for basic usage (no nets, no fixed positions, no ML):
        1. Create PlacementRequest with defaults
        2. Call place()
        3. Return PlacementOutput
