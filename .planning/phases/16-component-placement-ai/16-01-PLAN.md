<objective>
Build the netlist-to-placement-graph converter that transforms schematic netlists into bipartite component-net graphs suitable for GNN-based placement prediction.

Purpose: The bipartite graph representation is the foundation for all placement prediction. It avoids O(n^2) edge explosion from power nets (pitfall 1) by routing component connectivity through net nodes instead of direct pairwise edges. Node features encode component size, type, and position constraints so the downstream model has geometric awareness (pitfall 2).

Output: placement/ module with graph.py (bipartite graph construction) and features.py (node feature extraction); passing tests.
</objective>

    **2. Create src/kicad_agent/placement/features.py:**
    Feature extraction for bipartite graph nodes.

    - `COMP_FEATURE_DIM: int = 32` -- documented constant for component feature vector size.
    - `NET_FEATURE_DIM: int = 16` -- documented constant for net feature vector size.

    - `extract_component_features(comp: ComponentSpec, board_width: float, board_height: float) -> numpy.ndarray`:
      Returns a float32 array of length COMP_FEATURE_DIM (32):
      - [0]: estimated_size (reuse PlacementEngine._estimate_size heuristic: IC=10, FET=8, L/D=5, R/C=2, default=3)
      - [1]: is_ic (1.0 if reference starts with 'U')
      - [2]: is_passive (1.0 if reference starts with 'R' or 'C')
      - [3]: is_connector (1.0 if reference starts with 'J')
      - [4]: is_fixed (1.0 if position is not None)
      - [5]: normalized_fixed_x (x / board_width, 0.0 if not fixed)
      - [6]: normalized_fixed_y (y / board_height, 0.0 if not fixed)
      - [7-14]: library_id character hash (first 8 chars, ord(ch)/255.0)
      - [15-22]: value string hash (first 8 chars, ord(ch)/255.0)
      - [23-31]: zero-padded reserved

    - `extract_net_features(net: NetSpec, components: list[ComponentSpec]) -> numpy.ndarray`:
      Returns a float32 array of length NET_FEATURE_DIM (16):
      - [0]: pin_count (number of pins)
      - [1]: component_count (unique component refs from pins)
      - [2]: is_power (1.0 if net name in POWER_NETS set: GND, VCC, +3V3, +5V, VDD, VSS, GNDA, VCCA)
      - [3]: criticality (3.0 for high-speed signal nets, 1.0 for power nets, 2.0 default)
      - [4]: fanout_ratio (pin_count / max(component_count, 1))
      - [5-15]: zero-padded reserved

    **3. Create src/kicad_agent/placement/graph.py:**

    - `netlist_to_placement_graph(components: list[ComponentSpec], nets: list[NetSpec], board_width: float, board_height: float) -> networkx.Graph`:
      1. Create an undirected networkx Graph with graph-level attributes: board_width, board_height, component_count, net_count.
      2. For each component, add node with id `f"comp:{comp.reference}"`, bipartite=0, node_type="component", plus all component fields (reference, library_id, value, estimated_size, is_fixed, fixed_x, fixed_y).
      3. For each net, extract unique component references from pins (split on ".": "U1.3" -> "U1"). Add net node with id `f"net:{net.name}"`, bipartite=1, node_type="net", plus net fields (name, pin_count, is_power, criticality).
      4. For each component ref participating in a net, add edge between comp node and net node with weight=1.0 and pin_count attribute (number of pins from that component on that net).
      5. Validate: raise ValueError if board_width <= 0 or board_height <= 0, or if component count > 500 (matches existing _MAX_COMPONENTS cap).

    - `PlacementGraph` class wrapping the networkx graph:
      - `__init__(graph: nx.Graph)`: Store the graph. Validate it has bipartite attribute.
      - `component_nodes() -> list[str]`: Return node IDs where bipartite==0.
      - `net_nodes() -> list[str]`: Return node IDs where bipartite==1.
      - `get_component_features(board_width: float, board_height: float) -> numpy.ndarray`: Shape (n_comp, COMP_FEATURE_DIM). Call extract_component_features for each component node.
      - `get_net_features() -> numpy.ndarray`: Shape (n_net, NET_FEATURE_DIM). Call extract_net_features for each net node.
      - `get_adjacency_matrix() -> numpy.ndarray`: Shape (n_comp, n_net) binary adjacency. Rows = component nodes, columns = net nodes. Entry [i,j] = 1 if comp i connects to net j.
      - `get_edge_weights() -> numpy.ndarray`: Shape (n_comp, n_net). Same layout as adjacency but with edge weight values (criticality from net node as weight).
      - `board_width -> float` and `board_height -> float` properties from graph attributes.
      - `n_components -> int` and `n_nets -> int` properties.

    Security:
    - Component count cap at 500 (matches existing _MAX_COMPONENTS)
    - Net count cap at 200 (matches existing GenerationIntent.nets max_length)
    - Board dimensions must be positive
