# Plan 09-04: GRPO Training Loop and Evaluation

**Requirements:** GRPO-04, GRPO-06, GRPO-07
**Depends on:** Plan 09-01 (MazeDataset), Plan 09-02 (chains), Plan 09-03 (reward model)

## Goal

Build the GRPO (Group Relative Policy Optimization) training loop and evaluation harness. The loop generates chains, scores them with the reward model, and updates the policy. Evaluation measures improvement on held-out tasks vs baseline.

## Tasks

### Task 1: Create `src/kicad_agent/training/grpo.py` — GRPO training loop

- `GRPOConfig` dataclass:
  - `learning_rate: float = 1e-5`
  - `group_size: int = 8` — chains generated per prompt for group comparison
  - `kl_coefficient: float = 0.1` — KL divergence penalty strength
  - `clip_range: float = 0.2` — PPO-style clipping
  - `max_generation_length: int = 512` — tokens per chain
  - `temperature: float = 0.7` — sampling temperature
  - `seed: int = 42` — deterministic seeding
  - `checkpoint_every: int = 100` — save every N steps
  - `output_dir: str = "checkpoints/"`

- `GRPOTrainer`:
  - `__init__(policy_model, reward_model, ref_model, config)`:
    - `policy_model`: the model being trained (starts from pretrained or scratch)
    - `reward_model`: the reward model from 09-03
    - `ref_model`: frozen reference model for KL penalty
  - `generate_chains(prompts, group_size) -> list[list[str]]`:
    - Generate `group_size` chains per prompt using policy model
    - Returns grouped chains for relative comparison
  - `compute_group_rewards(chains, samples) -> list[list[float]]`:
    - Score each chain using reward model
    - Normalize within group (group relative advantage)
  - `compute_kl_penalty(policy_logprobs, ref_logprobs) -> Tensor`:
    - KL divergence between policy and reference model
  - `train_step(batch) -> dict`:
    - 1. Generate chains (group_size per sample)
    - 2. Compute rewards
    - 3. Compute group-relative advantages: (reward - group_mean) / group_std
    - 4. Compute policy gradient with clipping
    - 5. Add KL penalty
    - 6. Backprop and update
    - 7. Return metrics dict (loss, reward_mean, kl_divergence)
  - `train(dataset, n_epochs=1)`:
    - Full training loop over dataset
    - Checkpointing, logging, evaluation between epochs
    - Returns training history

### Task 2: Create `src/kicad_agent/training/evaluation.py` — Evaluation harness

- `EvalResult` frozen dataclass:
  - `model_name: str`
  - `n_samples: int`
  - `avg_reward: float`
  - `avg_accuracy: float`
  - `coordinate_coverage: float` — % of correct coordinates referenced
  - `chain_length_mean: float`
  - `chain_length_std: float`
  - `pass_rate: float` — % of chains that reach correct solution

- `EvaluationHarness`:
  - `__init__(test_dataset, reward_model, config=None)`
  - `evaluate(model, n_samples=100) -> EvalResult`:
    - Generate chains for test samples
    - Score with reward model
    - Compute metrics
  - `compare(model_before, model_after) -> dict`:
    - Side-by-side comparison of pre/post training
    - Improvement deltas for each metric

- `run_baseline(test_dataset) -> EvalResult`:
  - Generate chains using rule-based synthesis (no learned policy)
  - Provides baseline for comparison

- `run_ablation(config_variant, test_dataset) -> EvalResult`:
  - Test with modified config (e.g., no KL penalty, different group_size)
  - Returns results for comparison

### Task 3: Create `src/kicad_agent/training/pipeline.py` — Single-command training pipeline

- `TrainingPipelineConfig` dataclass:
  - `n_samples: int = 100_000`
  - `n_chains_per_sample: int = 2` — solution + exploration
  - `seed: int = 42`
  - `device: str = "cpu"` — "cpu" or "cuda"
  - `output_dir: str = "training_output/"`
  - `reward_config: RewardConfig | None = None`
  - `grpo_config: GRPOConfig | None = None`

- `run_pipeline(config: TrainingPipelineConfig) -> dict`:
  1. **Generate dataset**: `generate_dataset(n_samples, seed)`
  2. **Split**: train/val/test (80/10/10)
  3. **Synthesize chains**: `build_training_chains(train_dataset, ...)`
  4. **Score chains**: `score_chain()` for ground truth rewards
  5. **Train reward model**: `train_reward_model(...)` on scored chains
  6. **Evaluate baseline**: `run_baseline(test_dataset)`
  7. **GRPO train**: `GRPOTrainer.train(train_dataset)`
  8. **Evaluate trained**: `evaluate(trained_model, test_dataset)`
  9. **Compare**: `compare(baseline, trained)`
  10. **Save report**: Write results to `output_dir/eval_report.json`
  - Returns summary dict with all metrics
  - Deterministic with seed

### Task 4: Create `tests/test_training_grpo.py`
- Test `GRPOConfig` defaults and validation
- Test `GRPOTrainer.compute_group_rewards()` normalizes within groups
- Test `compute_kl_penalty()` returns non-negative value
- Test `EvaluationHarness.evaluate()` returns valid EvalResult
- Test `run_baseline()` produces result with pass_rate in [0, 1]
- Test `TrainingPipelineConfig` construction with defaults
- Test `run_pipeline()` with tiny config (n_samples=5) completes without error
- Test deterministic seeding produces identical results
