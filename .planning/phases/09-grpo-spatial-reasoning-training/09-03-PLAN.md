# Plan 09-03: Reward Model Architecture

**Requirements:** GRPO-03, GRPO-05
**Depends on:** Plan 09-02 (MazeReasoningChain)

## Goal

Build a reward model that scores coordinate-grounded reasoning chains with per-step dense rewards. The model provides format, quality, and accuracy signals plus anti-hacking penalties for GRPO training.

## Tasks

### Task 1: Create `src/kicad_agent/training/reward.py` — Reward model and scoring

- `RewardSignal` frozen dataclass:
  - `step_index: int` — which step in the chain
  - `format_score: float` — 0..1, correct structure and coordinate format
  - `quality_score: float` — 0..1, reasoning coherence and specificity
  - `accuracy_score: float` — 0..1, coordinate correctness vs ground truth
  - `penalty: float` — 0..-1, anti-hacking penalties
  - `total: float` — weighted sum of components

- `ChainReward` frozen dataclass:
  - `chain_id: str`
  - `sample_id: int`
  - `step_rewards: tuple[RewardSignal, ...]`
  - `total_reward: float` — sum of step totals
  - `reward_density: float` — total / num_steps

- `RewardConfig` dataclass (mutable for configuration):
  - `format_weight: float = 0.2`
  - `quality_weight: float = 0.3`
  - `accuracy_weight: float = 0.5`
  - `penalty_scale: float = 1.0`
  - `coordinate_format_regex: str = r"<point\s+[\d.]+,\s*[\d.]+>"`
  - `min_steps: int = 3`
  - `max_steps: int = 50`

- `score_chain(chain: MazeReasoningChain, sample: MazeSample, config: RewardConfig | None = None) -> ChainReward`:
  - Per-step scoring:
    - **format_score**: Check for coordinate references (`<point x,y>`), step_type validity, content length
    - **quality_score**: Reasoning verb presence ("found", "navigating", "requires"), specificity (coordinates mentioned), logical flow
    - **accuracy_score**: Compare referenced coordinates against sample solution_path coordinates (within tolerance)
    - **penalty**: Apply smooth penalties for:
      - Repeated identical coordinates (reward hacking)
      - Excessive chain length beyond max_steps
      - Coordinates outside board bounds
      - Missing coordinate references in spatial steps
  - Returns ChainReward with all per-step signals

### Task 2: Create `src/kicad_agent/training/reward_hacking.py` — Anti-hacking detection

- `AnomalyReport` frozen dataclass:
  - `chain_id: str`
  - `anomaly_type: str` — "coordinate_repetition", "bounds_violation", "length_anomaly", "score_inflation"
  - `severity: float` — 0..1
  - `description: str`

- `detect_anomalies(rewards: list[ChainReward]) -> list[AnomalyReport]`:
  - Statistical anomaly detection across a batch of rewards:
    - **coordinate_repetition**: Same coordinate in >50% of steps
    - **bounds_violation**: Coordinates outside [0, board_width] x [0, board_height]
    - **length_anomaly**: Chain length > 3 standard deviations from mean
    - **score_inflation**: Accuracy score > 0.9 with < 3 unique coordinates

- `smooth_penalty(raw_penalty: float, severity: float) -> float`:
  - Smooth penalty function: `-severity * tanh(raw_penalty)`
  - Prevents discontinuous reward cliffs that encourage hacking

### Task 3: Create `src/kicad_agent/training/reward_model.py` — Neural reward model (PyTorch)

- `RewardModel(nn.Module)`:
  - Lightweight transformer-based model that predicts reward scores
  - Architecture:
    - Tokenizer: simple character/word-level (no external tokenizer dependency)
    - Encoder: 4-layer transformer encoder (dim=256, heads=4, ff=512)
    - Prediction heads: 3 outputs per step (format, quality, accuracy)
  - `forward(input_ids, attention_mask) -> tuple[Tensor, Tensor, Tensor]`
  - `predict_reward(chain_text: str) -> RewardSignal`

- `train_reward_model(model, train_chains, val_chains, config=None)`:
  - Training loop with MSE loss against ground-truth reward signals
  - Ground truth comes from `score_chain()` (rule-based initial rewards)
  - AdamW optimizer, cosine LR schedule
  - Validation every N steps, early stopping on val loss
  - Saves checkpoints to configurable directory

### Task 4: Create `tests/test_training_reward.py`
- Test `RewardSignal` and `ChainReward` construction
- Test `score_chain()` returns correct number of step rewards
- Test format_score: chains with `<point>` tags score higher
- Test accuracy_score: correct coordinates score higher than wrong ones
- Test penalty: repeated coordinates get penalized
- Test `detect_anomalies()` identifies coordinate repetition
- Test `smooth_penalty()` is monotonically increasing with severity
- Test `RewardModel` forward pass shape (unit test, CPU only)
- Test `predict_reward()` returns valid RewardSignal

## Architecture

```
training/
├── reward.py            # RewardSignal, ChainReward, score_chain(), RewardConfig
├── reward_hacking.py    # AnomalyReport, detect_anomalies(), smooth_penalty()
└── reward_model.py      # RewardModel (nn.Module), train_reward_model()
```

- T-09-06: All reward scores clipped to [-1, 1]
