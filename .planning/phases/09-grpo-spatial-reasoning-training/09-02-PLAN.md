# Plan 09-02: Cold-Start Reasoning Chain Synthesis at Scale

**Requirements:** GRPO-02
**Depends on:** Plan 09-01 (MazeDataset, MazeSample), Phase 8 (reasoning_chains.py, primitives.py)

## Goal

Synthesize cold-start reasoning chains at scale from maze-routing samples. Each chain demonstrates coordinate-grounded spatial reasoning: DFS exploration traces, verified chains with coordinate references, and difficulty-graded training samples.

## Tasks

### Task 1: Create `src/kicad_agent/training/chains.py` — Chain synthesis from maze samples

- `MazeReasoningChain` frozen dataclass:
  - `sample_id: int` — links back to MazeSample
  - `difficulty: str` — inherited from sample
  - `chain_text: str` — full reasoning chain as natural language text
  - `steps: tuple[dict, ...]` — structured steps with coordinates
  - `coordinates_referenced: tuple[tuple[float, float], ...]` — all coordinates mentioned
  - `is_correct: bool` — whether chain reaches correct solution
  - `exploration_branches: int` — number of DFS branches explored

- `synthesize_maze_chain(sample: MazeSample) -> MazeReasoningChain`:
  - Builds a 5-step coordinate-grounded chain from a maze sample:
    1. **observation**: "Board is {W}x{H}mm with {N} obstacles. Source via at {point}, target via at {point}."
    2. **spatial_context**: "The path from source to target must navigate around {N} obstacles. Nearest obstacle to source at {coord}, distance {D}mm."
    3. **coordinate_reference**: "Solution path: {point} → {point} → ... → {point} ({L} steps)"
    4. **diagnosis**: "The optimal route requires {L} steps, passing through clear cells while maintaining {C}mm clearance."
    5. **recommendation**: "Route trace from source {point} following the verified path to target {point}."
  - Each step includes coordinate references in `<point x,y>` format

- `synthesize_exploration_chain(sample: MazeSample) -> MazeReasoningChain`:
  - DFS exploration trace showing dead ends and backtracking:
    1. Start at source, explore neighbors in order (up, right, down, left)
    2. At each step: "At <point x,y>, exploring direction D. Hit obstacle at <point x,y>, backtracking."
    3. Dead-end detection: "Reached dead end at <point x,y>. Backtracking to <point x,y>."
    4. Solution found: "Path found! Route: <point> → <point> → ..."
  - Produces longer chains with more coordinate references (richer training signal)

### Task 2: Create `src/kicad_agent/training/chain_builder.py` — DFS exploration and chain construction

- `CellState` enum: UNVISITED, VISITED, DEAD_END, ON_PATH
- `ExplorationStep` frozen dataclass:
  - `position: tuple[float, float]` — current cell center
  - `action: str` — "explore", "backtrack", "dead_end", "found_target"
  - `direction: str` — "up", "right", "down", "left", "none"
  - `obstacle_hit: bool`
  - `coordinates: tuple[tuple[float, float], ...]` — referenced points

- `dfs_explore(grid, start, end, grid_size_mm) -> list[ExplorationStep]`:
  - DFS with backtracking through the boolean grid
  - Returns ordered list of exploration steps with coordinate data
  - Each step records the cell position converted to mm coordinates

- `exploration_to_chain(sample, steps) -> MazeReasoningChain`:
  - Converts DFS exploration steps into a natural language reasoning chain
  - Groups consecutive explore steps, summarizes backtracking
  - Preserves all coordinate references

### Task 3: Create `src/kicad_agent/training/chain_writer.py` — Batch chain writing

- `write_chains_jsonl(chains: list[MazeReasoningChain], path: Path)`:
  - Streaming JSONL writer for large chain datasets
  - Each line: `{"sample_id": ..., "difficulty": ..., "chain_text": ..., "steps": [...], "coordinates": [...], "is_correct": ..., "exploration_branches": ...}`

- `build_training_chains(dataset: MazeDataset, output_path: Path, chain_type: str = "solution") -> int`:
  - Generates chains for all samples in dataset
  - `chain_type="solution"` — direct solution chains (synthesize_maze_chain)
  - `chain_type="exploration"` — DFS exploration chains (synthesize_exploration_chain)
  - `chain_type="both"` — both types per sample (doubles dataset size)
  - Returns count of chains written

### Task 4: Create `tests/test_training_chains.py`
- Test `MazeReasoningChain` construction and immutability
- Test `synthesize_maze_chain()` produces 5-step chains with coordinates
- Test chain text contains `<point x,y>` coordinate references
- Test `dfs_explore()` returns exploration steps with correct positions
- Test exploration chain includes backtracking and dead-end mentions
- Test `write_chains_jsonl()` produces valid JSONL with correct fields
- Test `build_training_chains()` with small dataset (10 samples)
- Test chain correctness: solution chains reach target coordinate

