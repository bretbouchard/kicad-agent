# Plan 09-01: Synthetic Data Pipeline at Scale

**Requirements:** GRPO-01
**Depends on:** Phase 8 (maze_generator.py, primitives.py)

## Goal

Generate 100k+ synthetic PCB maze-routing samples with verified solutions and difficulty grading. The dataset feeds both cold-start chain synthesis (09-02) and GRPO training (09-04).

## Tasks

### Task 1: Create `src/kicad_agent/training/__init__.py`
- Module barrel exports for the training pipeline

### Task 2: Create `src/kicad_agent/training/dataset.py` — MazeDataset and generation pipeline
- `MazeSample` frozen dataclass:
  - `sample_id: int` — sequential index
  - `seed: int` — random seed for reproducibility
  - `board_width_mm: float`
  - `board_height_mm: float`
  - `grid_size_mm: float`
  - `obstacle_count: int`
  - `obstacle_positions: tuple[tuple[float, float], ...]` — center coords
  - `source_point: tuple[float, float]`
  - `target_point: tuple[float, float]`
  - `solution_path: tuple[tuple[float, float], ...]` — BFS-verified
  - `solution_length: int`
  - `difficulty: str` — "easy" | "medium" | "hard" | "adversarial"
  - `board_hash: str` — SHA256 of board content for dedup
- `MazeDataset` dataclass:
  - `samples: list[MazeSample]`
  - `metadata: dict` — generation timestamp, config, counts by difficulty
  - `to_jsonl(path)` — write samples as JSONL
  - `from_jsonl(path)` — load samples from JSONL
  - `split(train=0.8, val=0.1, test=0.1)` — deterministic train/val/test split
- `generate_dataset()` function:
  - Parameters: `n_samples=100_000`, `seed_base=42`, `board_configs=None`
  - Uses `generate_maze_board()` from Phase 8 under the hood
  - **Difficulty grading** based on solution path length and obstacle density:
    - easy: solution_length <= 5 AND obstacle_count < 30% of cells
    - medium: solution_length 6-15 OR obstacle_count 30-50%
    - hard: solution_length 16-30 OR obstacle_count 50-70%
    - adversarial: solution_length > 30 AND obstacle_count > 60%
  - Configurable board configs: vary width_mm, height_mm, grid_size_mm to create diversity
  - Default configs: mix of 30mm/30mm, 50mm/50mm, 80mm/60mm boards with grid sizes 3-7mm
  - Deduplication via board_hash (skip duplicates)
  - Progress callback for logging
  - Batch writing to JSONL (stream, not accumulate all in memory)

### Task 3: Create `src/kicad_agent/training/generator.py` — Parallel generation engine
- `generate_samples_parallel(n_samples, n_workers=4, seed_base=42)`:
  - Uses `concurrent.futures.ProcessPoolExecutor` for CPU-bound generation
  - Each worker generates a chunk of samples with unique seed ranges
  - Merges results, deduplicates, grades difficulty
  - Returns `MazeDataset`
- `generate_adversarial_samples(n_samples, seed_base)`:
  - Generates boards specifically tuned for high difficulty
  - Higher obstacle density (60-75%), larger boards, narrower clearances
  - Ensures BFS solvability (same regeneration loop as Phase 8)

### Task 4: Create `tests/test_training_dataset.py`
- Test `MazeSample` construction and immutability
- Test `MazeDataset` creation, JSONL round-trip, split ratios
- Test `generate_dataset()` with small n_samples (e.g., 10)
- Test difficulty grading logic (easy/medium/hard/adversarial boundaries)
- Test deduplication (same seed produces same hash, duplicates skipped)
- Test parallel generation produces correct total count
- Test adversarial generation produces high-difficulty samples

## Architecture

