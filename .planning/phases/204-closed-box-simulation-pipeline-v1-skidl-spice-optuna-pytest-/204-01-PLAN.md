<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User shell → system PATH | ngspice CLI is invoked by Phase 158's `subprocess.run(["ngspice", ...])`. PATH trust boundary — if a malicious `ngspice` binary is on PATH ahead of the brew-installed one, it executes. |
| pip → PyPI | `pip install -e ".[sim]"` pulls optuna from PyPI. Supply-chain trust boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-204-01 | Tampering | ngspice CLI on PATH | accept | User installs via `brew install ngspice` from official Homebrew tap. Phase 158's subprocess inherits the user's PATH; same trust level as any dev tool. Out of scope to mitigate further in v1. |
| T-204-02 | Information Disclosure | optuna sqlite DB (`sweeps/eurorack_preamp.db` — created in Plan 03) | accept | DB contains only resistor values + gain measurements — no PII, no secrets. Lives in repo working tree. Low value, low risk. |
| T-204-03 | DoS | Optuna n_trials unbounded | mitigate | Plan 03's `optimize_preamp(n_trials=50)` signature hard-caps at 50 by default. Callers can override but CONTEXT.md locks 50-trial budget for the 60s time limit. |

## Adversarial Scenario

A developer clones the repo, sees `pyproject.toml` requires `optuna>=4.5`, runs `pip install -e ".[sim]"`, and pip resolves to a typosquatted package `optuna-evil` from a hostile PyPI mirror because their `pip.conf` points at an unsigned index. Mitigation: pin to official PyPI (`pip` default), and the version constraint `>=4.5` combined with hash-locked lockfile (future Phase 99+ work) bounds the attack surface. For Phase 204 v1 we accept PyPI trust — same posture as every other dep in pyproject.toml.

The 2N3904 .MODEL card is a static string literal in `model_registry.py` — no external input crosses a trust boundary to reach it. The parameters come from the published OnSemi datasheet (cited in 204-RESEARCH.md). Adversarial manipulation would require a repo commit — visible in `git blame`, caught by code review.
</threat_model>
