# Plan 206-01: Vendor DRC Profiles

## Goal

Ship verified `.kicad_dru` files for 5+ PCB manufacturers and add the ability to run DRC against a specific vendor's manufacturing limits as a pre-flight gate. The `drc_vendor` op uses an **internal Python-based geometric evaluator** (reads board geometry from `PcbIR`, compares against `ManufacturerProfile` numeric limits) because `kicad-cli pcb drc --custom-rules` does NOT exist in KiCad 10 (verified empirically — RESEARCH RQ1).

## CRITICAL CONTEXT (from RESEARCH + CONTEXT)

1. **Internal evaluator pivot.** `kicad-cli pcb drc` has no `--custom-rules` flag. The `drc_vendor` op must NOT call kicad-cli with vendor rules. Instead, `manufacturing/vendor_drc.py` walks `PcbIR` geometry in Python and compares against `ManufacturerProfile` numeric limits (Option C). The `.kicad_dru` files ship for GUI use + documentation + source of truth for numeric values.

2. **Package-data is mandatory.** `pyproject.toml` currently has NO `[tool.setuptools.package-data]` section. Without adding it, the `.kicad_dru` files will work in editable/dev mode but be silently dropped from wheels/`pip install`.

3. **Registry count 154 → 156** for +2 ops (drc_vendor, list_vendor_drc_profiles).

4. **test_readonly_operations_count** (test_registry.py:108-156) asserts an EXACT set of readonly op types — both new ops must be added to `expected_readonly`.

5. **PCBWay/JLCPCB annular ring correction (DRC-07).** Both `_PCBWAY_STANDARD` (profiles.py:146) and `_JLCPCB_STANDARD` (profiles.py:113) currently have `min_annular_ring_mm=0.1`. Update to `0.15` to match vendors' published 0.15mm (6mil) spec.

## Integration Order

Tasks are sequenced to keep schema/registry in sync at each step. Tasks 4+5+6 (schema, schema.py union, registry) must land together — adding an op to the schema union without a registry entry (or vice versa) breaks `validate_registry_completeness`.

## <threat_model>

**Attack surface:** Query handlers accept a `vendor` name string; DRU files are static package data; the internal evaluator reads board geometry from `PcbIR`.

**Adversarial scenario 1 — Path traversal via vendor name.**
The `drc_vendor` op accepts `vendor: str` which resolves to a file path via `get_drc_profile_path(vendor) -> Path(__file__).parent / f"{vendor}.kicad_dru"`. An attacker (or buggy LLM) could supply `vendor="../../../etc/passwd"` to probe the filesystem, or `vendor="../../../../secret"` to read files outside the package directory.

**Mitigation (implemented in Task 1):** `get_drc_profile_path` validates the vendor name against `^[a-z0-9_]+$` pattern (no slashes, dots, or path separators) AND verifies the resolved path is a file before returning it. The `_PROFILE_INFOS` dict is the allowlist — only vendor keys present in it resolve. The schema's `DrcVendorOp.vendor` field carries `pattern="^[a-z0-9_]+$"` constraint so malformed names are rejected at validation time before reaching the handler. Combined defense: pattern validation at the schema layer + file-existence check at the resolver layer.

**Adversarial scenario 2 — Malformed board crashes evaluator.**
The internal evaluator iterates `NativeBoard.segments`, `.vias`, `.footprints[*].pads`. A malformed board (missing fields, None geometry, negative dimensions) could cause `TypeError`, `ValueError`, or infinite loops.

**Mitigation (implemented in Task 3):** Every geometry access uses `getattr(obj, field, default)` with safe defaults, and dimension extraction is wrapped in try/except per-feature (a single malformed track does not abort the whole evaluation). Negative or zero dimensions are treated as violations, not crashes. The evaluator NEVER re-raises — it collects what it can and returns a `VendorDrcResult` (possibly with an `error_message`).

**ASVS L1 coverage:** Input validation (vendor allowlist + pattern), no injection (static data files, no eval), graceful degradation (malformed boards don't crash).

</threat_model>

```python
"""Vendor DRC profile registry — bundled .kicad_dru files + capabilities metadata.

Phase 206: Ships verified .kicad_dru files for PCBWay, JLCPCB, AISLER (2/4/6/8L),
OSH Park, Advanced Circuits, and a generic conservative profile. The files are
the source of truth for vendor numeric limits and loadable in the KiCad GUI.
The automated drc_vendor op uses an internal evaluator (manufacturing/vendor_drc.py)
that checks board geometry against these limits, because kicad-cli does not load
.kicad_dru sidecars (RESEARCH RQ1).
"""
from __future__ import annotations

import re
from dataclasses import dataclass
from pathlib import Path

_PROFILES_DIR = Path(__file__).parent
_VENDOR_NAME_RE = re.compile(r"^[a-z0-9_]+$")


@dataclass(frozen=True)
class VendorDrcProfileInfo:
    """Capabilities metadata for one bundled vendor DRC profile (DRC-08)."""

    vendor: str
    display_name: str
    drc_rules_path: str
    min_trace_width_mm: float
    min_clearance_mm: float
    min_drill_mm: float
    min_annular_ring_mm: float
    min_via_diameter_mm: float
    supports_blind_vias: bool
    supports_castellated: bool
    source: str
    last_verified: str


# Co-located metadata registry — mirrors _PROFILES dict pattern in dfm/profiles.py.
_PROFILE_INFOS: dict[str, VendorDrcProfileInfo] = {
    "pcbway": VendorDrcProfileInfo(
        vendor="pcbway", display_name="PCBWay",
        drc_rules_path="pcbway.kicad_dru",
        min_trace_width_mm=0.127, min_clearance_mm=0.127,
        min_drill_mm=0.2, min_annular_ring_mm=0.15, min_via_diameter_mm=0.4,
        supports_blind_vias=True, supports_castellated=True,
        source="Cimos/KiCad-CustomDesignRules (MIT)", last_verified="2026-07-10",
    ),
    # ... jlcpcb, aisler_2layer..8layer, oshpark, advanced_circuits, generic
    # (one entry per file, values matching the .kicad_dru files above)
}


def get_drc_profile_path(vendor: str) -> Path:
    """Resolve a vendor name to its bundled .kicad_dru file path.

    Security (threat model scenario 1): validates vendor name against
    ^[a-z0-9_]+$ AND verifies the resolved path is a real file. This
    prevents path traversal (no slashes/dots in vendor name) and confirms
    only allowlisted vendors resolve.
    """
    if not _VENDOR_NAME_RE.match(vendor):
        raise ValueError(f"Invalid vendor name {vendor!r}: must match ^[a-z0-9_]+$")
    path = _PROFILES_DIR / f"{vendor}.kicad_dru"
    if not path.is_file():
        available = ", ".join(sorted(_PROFILE_INFOS.keys()))
        raise ValueError(
            f"Unknown vendor {vendor!r}. Available DRC profiles: {available}."
        )
    return path


def list_drc_profiles() -> list[VendorDrcProfileInfo]:
    """Return all bundled vendor DRC profiles with capabilities metadata (DRC-08)."""
    return list(_PROFILE_INFOS.values())


__all__ = ["VendorDrcProfileInfo", "get_drc_profile_path", "list_drc_profiles"]
```

   - **Drill size check** (`min_drill_mm`): iterate `board.vias` (via.drill) and `board.footprints[*].pads` where `pad.pad_type == "thru_hole"` (pad.drill). Emit `type="vendor_drill_size"`.

   - **Via diameter check** (`min_via_diameter_mm`): iterate `board.vias`, check `via.diameter`. Emit `type="vendor_via_diameter"`.

   - **Clearance check** (`min_clearance_mm`): pairwise check between tracks on the same layer. Use bounding-box pre-filtering per layer (skip pairs whose bounding boxes don't overlap within `min_clearance_mm`). For v1, use track-to-track checks with the corridor pattern from pcb_ir.py:978-986 (width/2 + clearance on both sides). Emit `type="vendor_clearance"`. If performance is a concern on large boards, document the O(n²) nature and note that STRtree (shapely, already a dependency) can optimize this later.

3. **Robustness (threat model scenario 2):** Wrap each per-feature extraction in try/except. A single malformed track/via/pad must NOT abort the whole evaluation. Use `getattr(obj, field, default)` everywhere. If a dimension is negative or zero where a positive is expected, treat it as a violation (not a crash). The function NEVER re-raises — it collects what it can. If board access fails entirely (e.g., `NativeParser.parse_pcb` raises), return `VendorDrcResult(vendor=profile.name, passed=False, error_message="...", checks_run=())`.

4. **`passed` logic:** `passed = len([v for v in violations if v.severity == Severity.ERROR]) == 0` — mirrors run_drc passed logic at erc_drc.py:439. Warnings do not fail.

5. **Reuse existing `Violation` + `Severity`** from `validation/erc_drc.py` — do NOT define new types. Import: `from kicad_agent.validation.erc_drc import Violation, Severity`.

6. **Board access:** The handler calls `NativeParser.parse_pcb(file_path)` to get a `NativeBoard` directly. Access `board.segments`, `board.vias`, `board.footprints`. Each footprint has `.pads` (tuple of `NativePad`). Use getattr-based access with the field names confirmed in pcb_native_types.py (NativeSegment: width, layer, net_name, start, end; NativeVia: drill, size, position, net_name, uuid; NativePad: drill, pad_type, size, position, number, net_name). NOTE: `NativeVia` uses `.size` for diameter (not `.diameter`). @register_query("drc_vendor")
```python
def _handle_drc_vendor(op: Any, ir: PcbIR, file_path: Path) -> dict[str, Any]:
    """Run vendor-specific DRC checks (DRC-01, DRC-04).

    Uses the internal geometric evaluator against ManufacturerProfile limits.
    Optionally also runs KiCad's built-in DRC (run_kicad_drc flag).
    """
    from dataclasses import asdict
    from kicad_agent.dfm.profiles import load_profile
    from kicad_agent.manufacturing.vendor_drc import run_vendor_drc

    profile = load_profile(op.vendor)  # raises ValueError if unknown
    result = run_vendor_drc(board, profile)  # board is NativeBoard from NativeParser.parse_pcb(file_path)

    kicad_drc_result = None
    if op.run_kicad_drc:
        try:
            from kicad_agent.validation.erc_drc import run_drc
            drc = run_drc(file_path)
            kicad_drc_result = {
                "passed": drc.passed,
                "violations": [asdict(v) for v in drc.violations],
            }
        except Exception as exc:
            # kicad-cli may be absent in test/dev — degrade gracefully.
            kicad_drc_result = {"error": str(exc)}

    out = asdict(result)
    out["kicad_drc"] = kicad_drc_result
    # `passed` reflects VENDOR DRC only. kicad_drc is separate — user can check both.
    # If kicad_drc failed with errors, that does NOT affect vendor passed status.
    return out


@register_query("list_vendor_drc_profiles")

    return {"profiles": profiles, "count": len(profiles)}
```
