  <behavior>
    - AtomicOperation.__init__ accepts a list of file paths (schematic + PCB pairs)
    - __enter__ opens a Transaction for each file path, storing them in order
    - If any Transaction fails to open, all previously opened Transactions are rolled back
    - commit() commits all Transactions; if any commit fails, all are rolled back
    - rollback() rolls back all Transactions in reverse order
    - __exit__ auto-rolls back on exception if not committed
    - AtomicResult dataclass with success, results list of TransactionResult, and error
  </behavior>
  <action>
    1. Create `src/kicad_agent/crossfile/` package directory.
    2. Create `__init__.py` with barrel exports for AtomicOperation and AtomicResult.
    3. Create `atomic.py` with the AtomicOperation class:
       - `__init__(self, file_paths: list[Path])` -- stores paths, validates all exist.
       - `__enter__(self)` -- opens a Transaction per file path. If the Nth Transaction
         fails to open (e.g., fcntl lock conflict, file not found), roll back all N-1
         already-opened Transactions and raise the error.
       - `commit(self) -> AtomicResult` -- iterate through all Transactions calling
         commit(). If any commit raises, call rollback() on all and return AtomicResult
         with success=False and the error.
       - `rollback(self) -> AtomicResult` -- iterate through all Transactions in reverse
         order calling rollback(). Return AtomicResult with success=False.
       - `__exit__(self, exc_type, exc_val, exc_tb)` -- if exception and not committed,
         auto-rollback all Transactions (matches Transaction.__exit__ pattern).
       - Store list of Transaction objects in `self._transactions: list[Transaction]`.
    4. Create `AtomicResult` frozen dataclass:
       - `success: bool`
       - `results: list[TransactionResult]`
       - `error: Optional[str] = None`
    5. Follow patterns from Transaction class (02-03): symlink protection on each path,
       logging for all rollback events, no silent failure.
