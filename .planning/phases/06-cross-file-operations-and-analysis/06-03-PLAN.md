    2. Create `src/kicad_agent/crossfile/project_context.py` with:

       - `ProjectContext` frozen dataclass:
         - `project_root: Path` -- absolute path to project root directory
         - `pro_file: Optional[Path]` -- path to .kicad_pro (None if not found)
         - `schematic_files: list[Path]` -- all .kicad_sch files
         - `pcb_files: list[Path]` -- all .kicad_pcb files
         - `sym_lib_files: list[Path]` -- all .kicad_sym files
         - `footprint_files: list[Path]` -- all .kicad_mod files
         - `library_paths: list[str]` -- lib_dir values from .kicad_pro

       - `detect_project_root(file_path: Path) -> Path`:
         - Start from file_path.parent (if file) or file_path (if directory)
         - Walk upward max 20 levels checking for `*.kicad_pro` files
         - If found, return the directory containing the .kicad_pro
         - If not found after 20 levels, raise FileNotFoundError with descriptive message
         - Use `file_path.resolve()` to handle symlinks consistently

       - `discover_project(project_root: Path) -> ProjectContext`:
         - Validate project_root is a directory (raise ValueError otherwise)
         - Use Path.glob to find all KiCad files:
           - `**/*.kicad_sch` -> schematic_files
           - `**/*.kicad_pcb` -> pcb_files
           - `**/*.kicad_sym` -> sym_lib_files
           - `**/*.kicad_mod` -> footprint_files (inside *.pretty dirs)
           - `*.kicad_pro` -> pro_file (just the first one found)
         - If pro_file exists, call `_parse_kicad_pro(pro_file)` to extract library_paths
         - Return ProjectContext with all discovered files
         - Sort file lists by path for deterministic output

       - `_parse_kicad_pro(pro_path: Path) -> list[str]`:
         - Read the .kicad_pro file as text
         - Use regex to find `(lib_dir "...")` entries
         - Return list of lib_dir values
         - Return empty list if file is malformed (log warning, don't crash)
  </action>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-06-11 | T (Tampering) | Malformed .kicad_pro content | mitigate | Regex parsing is tolerant -- malformed content returns empty list with warning |
| T-06-12 | D (DoS) | Deep directory structures | mitigate | 20-level upward walk cap; Path.glob uses ** but only within project root |
| T-06-13 | I (Info disclosure) | Project file contents exposed | accept | ProjectContext only returns file paths, not file contents |
</threat_model>
