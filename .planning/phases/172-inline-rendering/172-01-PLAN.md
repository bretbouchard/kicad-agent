# KiCad CLI Commands for Rendering (Phase 163 bundled)

```bash
# Schematic → SVG (inline preview)
kicad-cli sch export svg project.kicad_sch -o schematic.svg

# PCB → PNG (inline preview)
kicad-cli pcb render project.kicad_pcb -o pcb.png --side front
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Daemon → UI | Progress events from untrusted subprocess stdout |
| File system → UI | SVG/PNG files from daemon-rendered outputs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-172-01 | Tampering | SVG/PNG rendering | mitigate | Validate file magic bytes before rendering (SVG: `<?xml`, PNG: `\\x89PNG`). Reject non-image files. |
| T-172-02 | Spoofing | Progress events | mitigate | Parse progress events with strict JSON schema. Reject malformed events. Ignore unexpected step names. |
| T-172-03 | Denial of Service | Unbounded cache | mitigate | Cap in-memory cache at 100 renders. Evict oldest entries (LRU). |
| T-172-04 | Information Disclosure | Error messages | accept | Error messages may contain file paths — local-only, no external exposure. |

## Adversarial Scenario

**Scenario:** A malicious daemon subprocess sends a crafted progress event with a `step` name that causes SQL injection in the UI layer (e.g., step name contains SQL payload that gets logged to SwiftData).

**Mitigation:** Strict JSON schema validation on progress events. `step` must match enum values: `["design", "schematic", "erc", "pcb", "drc", "export"]`. Reject all other values. Never interpolate step names into queries without parameterization.
</threat_model>
