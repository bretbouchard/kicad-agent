    1. Create `src/kicad_agent/placement/signal_flow.py`:
       - `SignalFlowZone` frozen dataclass: zone_id (str), component_refs (tuple[str,...]), nets (tuple[str,...]), zone_type (str: "input"|"processing"|"output"|"power"|"ungrouped"), priority (int)
       - `SignalFlowGroup` frozen dataclass: group_id (str), ordered_zones (tuple[SignalFlowZone,...]), signal_entry_nets (tuple[str,...]), signal_exit_nets (tuple[str,...])
       - `SignalFlowGrouper` class:
         - `__init__()` with `_TYPE_PRIORITY` dict mapping SubcircuitType to int (PREAMP=10, INPUT prefix types=5, OUTPUT_STAGE=20, POWER_SUPPLY=30, UNKNOWN=40)
         - `group(subcircuits: list[Subcircuit], intents: list[SubcircuitIntent] | None = None) -> list[SignalFlowGroup]`:
           a. If intents provided, build subcircuit_id -> SubcircuitIntent lookup for I/O net info
           b. Build adjacency: for each pair of subcircuits, if they share boundary_nets, they are adjacent
           c. Find connected components via BFS on adjacency graph -- each connected component is a SignalFlowGroup
           d. Within each group, order zones by signal flow: subcircuits with input_nets that are NOT boundary_nets of any other subcircuit in the group are "entry" zones. Follow boundary_net chain to determine ordering.
           e. If signal flow ordering is ambiguous (no clear entry/exit), fall back to _TYPE_PRIORITY ordering
           f. Assign zone_type from SubcircuitType: map PREAMP/FILTER/EQ/VCA/MIXER -> "processing", OUTPUT_STAGE -> "output", POWER_SUPPLY -> "power", etc.
           g. For each group, identify signal_entry_nets (input_nets of entry zone minus boundary_nets) and signal_exit_nets (output_nets of exit zone minus boundary_nets)
         - Return list[SignalFlowGroup], sorted by first zone priority

    2. Create `src/kicad_agent/placement/footprint_geometry.py`:
       - `ComponentGeometry` frozen dataclass: reference (str), width_mm (float), height_mm (float), pad_positions (tuple[tuple[float,float],...]), thermal_area_mm2 (float), centroid_offset (tuple[float,float])
       - `extract_footprint_geometry(pcb_ir: PcbIR | None = None) -> dict[str, ComponentGeometry]`:
         a. If pcb_ir is None, return empty dict
         b. Iterate pcb_ir.footprints
         c. For each footprint, extract reference from fp.reference
         d. Compute bounding box from pads: iterate fp.pads, collect (pad.at[0], pad.at[1]) positions, also account for pad size via pad.size
         e. width = max(pad_x + pad_size_x/2) - min(pad_x - pad_size_x/2), same for height
         f. If no pads, default to 2.0 x 2.0 mm
         g. pad_positions = tuple of (x, y) relative to footprint origin
         h. thermal_area_mm2 = width * height (conservative estimate)
         i. centroid_offset = ((max_x+min_x)/2, (max_y+min_y)/2) relative to origin (for fine-tuning placement center)
         j. Return dict[str, ComponentGeometry]

         - board_width: float = Field(gt=0)
         - board_height: float = Field(gt=0)

         - intents: list[SubcircuitIntent] | None = None (from analysis.intent_schemas)

         - min_clearance: float = Field(default=1.0, gt=0)
         - use_ml: bool = True

       - `LayoutAwarePlacer` class:
         - `__init__(self, engine: HybridPlacementEngine | None = None)` -- creates default engine if None provided
         - `place_layout_aware(self, request: LayoutAwareRequest) -> PlacementOutput`:
           a. **Phase 1 -- Signal flow grouping**: If subcircuits provided, use SignalFlowGrouper.group() to produce SignalFlowGroup[]
           b. **Phase 2 -- Zone assignment**: For each SignalFlowGroup, compute a board zone (x1, y1, x2, y2) based on the number of groups and signal flow ordering. Groups are laid out left-to-right with configurable `_ZONE_MARGIN_MM = 2.0` between zones. Each group gets board_width/len(groups) width minus margins.
           c. **Phase 3 -- Constraint injection**: Convert zone assignments to fixed_positions for group center components and keepout_zones for zone boundaries. Merge with user-provided fixed_positions and keepout_zones.
           d. **Phase 4 -- Geometry injection**: If component_geometry provided, use it to build accurate component sizes dict. Otherwise fall back to default 2.0mm (the current heuristic from interactive.py `_DEFAULT_SIZE`).
           e. **Phase 5 -- Delegate placement**: Build a PlacementRequest from the LayoutAwareRequest with the computed constraints, call self._engine.place(request)
           f. **Phase 6 -- Post-placement validation**: Log zone adherence metrics (percentage of components placed within their assigned zone). Do NOT reject placements that violate zones -- zones are soft guidance for the SA objective.
           g. Return PlacementOutput with source="layout_aware"

         - `_assign_zones(self, groups: list[SignalFlowGroup], board_w: float, board_h: float) -> dict[str, tuple[float, float, float, float]]`:
           Returns group_id -> (x1, y1, x2, y2) zone rectangle. Groups laid out left-to-right in signal flow order, each taking board_w/len(groups) width.

         - `_compute_zone_constraints(self, zones: dict[str, tuple[float,float,float,float]], groups: list[SignalFlowGroup], geometry: dict[str, ComponentGeometry] | None) -> tuple[dict, list]`:
           Returns (fixed_positions, keepout_zones) for zone center components and zone boundaries.
