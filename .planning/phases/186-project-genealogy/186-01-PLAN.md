<task type="auto">
  <name>Task 2: Implement branch/fork operations with cycle detection</name>
  <files>KiCadAgent/Governance/ProjectBrancher.swift</files>
  <action>
Create ProjectBrancher with fork/merge/abandon operations:

**Fork operation:**
- createBranch(from: Project, at: ProjectSnapshot?) async throws → ProjectBranch
- Validates source project exists
- If snapshot provided, forks at point in time (copies conversation state)
- Cycle detection: new branch's parent cannot be its own descendant
- Sets branchStatus = .active, createdAt = Date()
- Creates child Project with parent reference
- CloudKit sync: insert new branch and child project

**Abandon operation:**
- abandonBranch(_ branch: ProjectBranch, reason: String?) async throws
- Sets branchStatus = .abandoned, abandonedReason = reason (optional, skip allowed)
- If reason skipped, use "Decided later" default (GEN-03)
- Updates updatedAt for LWW

**Merge operation:**
- mergeBranch(_ source: ProjectBranch, into: ProjectBranch) async throws
- Validates target branch exists and is active
- Detects conflicts (both branches modified same decision/value)
- If conflicts: triggers three-way merge UI (GEN-06, deferred to GEN-06 requirement)
- If no conflicts: sets source.branchStatus = .merged, mergedIntoBranch = target.id
- Creates MergeEvent in journal

**Cycle detection algorithm:**
- BFS from new branch's parent, check if reaches new branch ID
- If cycle detected: throw BranchError.cycleDetected
- Unit tests: cycleA→B→A (invalid), cycleA→B→C (valid)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User → Branch Operations | Untrusted branch names, abandon reasons |
| CloudKit Sync | Remote branch state changes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-186-01 | S | ProjectBrancher | mitigate | Validate branch names (max 255 chars, no special chars), sanitize abandon reason before SwiftData storage |
| T-186-02 | T | CloudKit sync | accept | CloudKit encrypted in transit, Apple handles security |
| T-186-03 | I | FamilyTree query | mitigate | Cycle detection prevents infinite loops on recursive queries, max depth 1000 |
| T-186-04 | D | Branch abandonment | mitigate | Abandon is irreversible, requires confirmation dialog (irreversible action guard) |
| T-186-05 | E | Fork operation | accept | Fork is append-only, no data loss risk |

## Adversarial Scenario

Name: Branch Name Injection Attack
An attacker crafts a malicious branch name containing control characters or extremely long strings (10K+ chars) to corrupt the family tree UI or cause memory exhaustion.

Mitigation: 
1. Branch names limited to 255 chars max
2. Control characters stripped (.controlCharacterCharSet removed)
3. UI truncates display at 50 chars with ellipsis
4. SwiftData validation rejects names >255 chars
</threat_model>
