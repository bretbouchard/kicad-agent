<objective>
Build a net connectivity graph from LTspice .asc wire geometry, FLAG net labels, and component pin positions.

Purpose: LTspice does not explicitly store net assignments per component. Connectivity must be derived geometrically by tracing wire segments, matching FLAG positions to net names, and computing component pin positions from symbol pin offsets + component position/rotation. This graph enables querying "which pins are on net X" -- essential for KiCad-to-LTspice bridge mapping.

Output: `net_graph.py` with `LTspiceNetGraph` class using networkx (consistent with Phase 5 NetGraph pattern).
</objective>

```python
# After parse_asc(), the AscEditor is not directly available.
# Pin positions must be computed from component position + rotation + symbol pin offsets.
# Symbol pin offsets come from the .asy stub files loaded by AscEditor.
# AscEditor stores symbol info in component.symbol.pins (list of Text objects with .coord)
#
# Rotation transform for pin at (px, py) relative to component at (cx, cy):
# R0:   (cx + px, cy + py)
# R90:  (cx - py, cy + px)
# R180: (cx - px, cy - py)
# R270: (cx + py, cy - px)
# M0:   (cx - px, cy + py)  (mirror X)
# M90:  (cx + py, cy + px)
# M180: (cx + px, cy - py)
# M270: (cx - py, cy - px)
#
# Pin names come from PINATTR PinName in the .asy file
# For res.asy: Pin 1 at (32,48), Pin 2 at (32,-32)
# For cap.asy: Pin 1 at (32,32), Pin 2 at (32,-32)
# For voltage.asy: Pin 1 at (32,64), Pin 2 at (32,-32)
```

    **Step 1: Implement LTspiceNetGraph.**

    Create `src/kicad_agent/ltspice/net_graph.py` with:

    ```python
    """Net connectivity derivation from LTspice wire geometry.

    LTSPICE-03: Build a networkx graph from wire segments, assign net names
    from FLAG positions, and match component pins to nets.

    Algorithm:
    1. Add wire segment endpoints as graph nodes, wire segments as edges
    2. Merge nodes at same (x,y) coordinate (they are the same electrical point)
    3. Propagate FLAG net names across connected components
    4. Compute component pin absolute positions from symbol offsets + rotation
    5. Match pins to wire graph nodes to assign net membership
    """
    ```

    Key class: `LTspiceNetGraph` (regular dataclass, consistent with `NetGraph`):
    - `graph: nx.Graph` -- nodes are `(x, y)` coordinate tuples, edges connect wire endpoints
    - `_flag_map: dict[tuple[int,int], str]` -- coordinate -> net name from FLAGs
    - `_net_index: dict[str, list[tuple[str, int]]]` -- net name -> list of (reference, pin_number)
    - `_component_pins: dict[str, list[tuple[int, int, int]]]` -- reference -> list of (pin_number, abs_x, abs_y)

    Class method `from_schematic(schematic: LTspiceSchematic, *, editor: AscEditor | None = None) -> LTspiceNetGraph`:
    1. Build wire graph: for each `LTspiceWire`, add edge `((x1,y1), (x2,y2))` to nx.Graph. Since LTspice wires are always axis-aligned segments, two wires connect where one wire's endpoint equals another's endpoint (exact integer coordinate match).
    2. Map FLAG positions: for each `LTspiceFlag`, record `{(x, y): text}` in `_flag_map`.
    3. Propagate net names: for each flag at position P, find the connected component containing P (using `nx.node_connected_component`). Assign the flag's net name to all nodes in that component. Multiple flags on the same component should have the same net name (validation check).
    4. Compute component pin positions: for each component, load its symbol's pins from the bundled .asy stubs. For each pin at offset (px, py), compute absolute position using the rotation transform from the RESEARCH.md interfaces section above. Add the pin's absolute position to the graph as a node (if not already present from a wire endpoint).
    5. Match pins to nets: for each component pin at absolute position (ax, ay), check if (ax, ay) is a graph node. If it is, find its connected component and retrieve the net name (if any flag assigned one). Add (reference, pin_number) to `_net_index[net_name]`.
    6. Return constructed LTspiceNetGraph.

    For reading symbol pins from .asy files, use `spicelib.editor.asc_editor.AsyReader` to load the .asy file from `asy_stubs/{symbol_name}.asy`. The reader provides `pins` list where each pin is a `Text` object with `coord` (Point). Pin names come from the PINATTR entries parsed into the `text` attribute (format: "PinName=1;"). Extract pin number from this text.

    The `editor` parameter allows passing an existing AscEditor instance (which already loaded the .asy symbols) to avoid re-reading them. When editor is provided, use `editor.get_component(ref).symbol.pins` to get pin positions directly. When None, fall back to loading .asy files manually.

    Query methods (matching NetGraph API):
    - `get_net_names() -> set[str]` -- all named nets from flags
    - `get_pins_on_net(net_name: str) -> list[tuple[str, int]]` -- (reference, pin_number) pairs
    - `get_connected_component(point: tuple[int,int]) -> set[tuple[int,int]]` -- all coordinates connected to this point
    - `are_connected(p1: tuple[int,int], p2: tuple[int,int]) -> bool` -- two points are electrically connected
    - `get_net_stats() -> dict[str, int]` -- {total_nets, total_nodes, total_edges, total_pins}
