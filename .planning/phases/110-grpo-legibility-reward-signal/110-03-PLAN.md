<objective>
Build the GRPO data path: take base schematics, generate N controlled variations per base (placement perturbations via D-04 alignment jitter), score each via the verified SRS chain, emit reward deltas for exploration training. This is half of D-02 ("real SFT + synthetic GRPO").

Purpose: Per CONTEXT.md D-02, GRPO needs controlled variations to compute reward deltas — "did this perturbation make the schematic more or less legible?" The reward delta (variation_srs - base_srs) is the per-step advantage signal that GRPO trains on. Without it, GRPO has no group-relative signal.

Output: `scripts/generate_grpo_variations.py` (CLI), `src/kicad_agent/training/grpo_data_builder.py` (library), test file. Output dataset at `/Volumes/Storage/models/kicad-agent/datasets/grpo/exploration.jsonl`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Base .kicad_sch -> parse_schematic | Source schematic may be corrupt, oversized, or contain unusual structures. |
| Variation .kicad_sch -> SchematicRawWriter | Generated variations must still be parseable KiCad (atomic writes + raw writer mitigate). |
| exploration.jsonl -> Phase 110 Plan 04 GRPO loop | Downstream training pipeline — bad reward deltas corrupt policy gradient. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-110-09 | Tampering | perturb_schematic | mitigate | Use SchematicRawWriter (paren-balanced, UUID-safe) — never raw regex on S-expressions (Phase 100 H-2 pattern). Validate output is parseable by kiutils before emitting JSONL row. |
| T-110-10 | Denial of Service | build_exploration_rows | mitigate | Wrap each base in try/except; one corrupt base must not abort the full corpus run. Log skip, continue. |
| T-110-11 | Repudiation | exploration.jsonl | mitigate | Each row includes `seed` for full reproducibility — any row can be regenerated and verified. |
| T-110-12 | Tampering | reward_delta computation | mitigate | Assert reward_delta in [-1.0, 1.0] (SRS scorer contract); skip row with warning if violated. |

## Adversarial Scenario

A base schematic has components at extreme coordinates (X=1e9 mm) — perturbing these by ±0.1mm produces no meaningful variation, and the SRS scorer might OOM trying to compute bounding box. Mitigation: (1) `perturb_schematic` validates base coordinate range before perturbation, raises `GRPODataBuilderError` if any coordinate exceeds ±10000mm (realistic schematic limit); (2) `build_exploration_rows` catches this per-base and skips; (3) the corpus is iterated, not batched — partial progress survives.
</threat_model>
