<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Crawled GitHub .kicad_sch -> parse_schematic | Untrusted external content. May be corrupt, oversized, contain exotic KiCad syntax not yet supported by kiutils. |
| SFT JSONL -> Phase 97 SFT trainer | Downstream training pipeline consumes these labels — bad labels corrupt training silently. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-110-05 | Denial of Service | SFTLabeller.score_file | mitigate | Wrap parse_schematic + chain in try/except -> SFTLabellerError. Single corrupt schematic must not abort a 100k-file corpus run. (Test 5) |
| T-110-05b | Denial of Service | SFTLabeller.score_file (size guard) | mitigate | Skip files > max_file_mb (default 50MB) BEFORE parsing — prevents kiutils OOM on pathological S-expressions (ME-110-10). (Test 7) |
| T-110-06 | Tampering | label_corpus | mitigate | Validate SRS factor values in [0, 1.0] before emitting JSONL row (defensive — ReadabilityReport enforces this but defense in depth); skip with warning if out of range. |
| T-110-07 | Information Disclosure | score_sft_dataset.py CLI | accept | Logs only file paths and aggregate counts. No schematic contents, no secrets. |
| T-110-08 | Repudiation | JSONL output | mitigate | Each row includes `scored_at` ISO-8601 timestamp and `source` tag for audit trail. |

## Adversarial Scenario

A malicious crawled repo contains a 500MB `.kicad_sch` file designed to OOM the kiutils parser (pathological nested S-expression) and crash the SFT pipeline partway through, losing the labels collected so far. Mitigation: (1) `max_file_mb=50` guard (Test 7, ME-110-10) skips the file BEFORE parsing — deterministic skip, not a try/except race; (2) `label_corpus` catches `Exception` broadly inside the per-file loop, logs the skip, and continues — even a 49MB corrupt file is non-fatal; (3) the script writes JSONL incrementally (one row at a time via atomic append) so partial progress persists across crashes; (4) `--limit N` flag allows smoke-testing on small N before committing to a full corpus run.
</threat_model>
