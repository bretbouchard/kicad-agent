<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Crawled GitHub .kicad_sch → SFTLabeller | Untrusted external content. May be corrupt, oversized, contain exotic KiCad syntax not yet supported by kiutils. |
| SFT JSONL → Phase 97 SFT trainer | Downstream training pipeline consumes these labels — bad labels corrupt training silently. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-110-05 | Denial of Service | SFTLabeller.score_file | mitigate | Wrap kiutils parse in try/except → SFTLabellerError. Single corrupt schematic must not abort a 100k-file corpus run. (Test 5) |
| T-110-06 | Tampering | label_corpus | mitigate | Validate SRS factor values in [0, 1.0] before emitting JSONL row; skip with warning if out of range (don't poison SFT dataset with NaN/infinite labels). |
| T-110-07 | Information Disclosure | score_sft_dataset.py CLI | accept | Logs only file paths and aggregate counts. No schematic contents, no secrets. |
| T-110-08 | Repudiation | JSONL output | mitigate | Each row includes `scored_at` ISO-8601 timestamp and `source` tag for audit trail. |

## Adversarial Scenario

A malicious crawled repo contains a 500MB `.kicad_sch` file designed to OOM the kiutils parser (pathological nested S-expression) and crash the SFT pipeline partway through, losing the labels collected so far. Mitigation: (1) `label_corpus` catches `Exception` broadly inside the per-file loop, logs the skip, and continues — the 500MB file is skipped, not fatal; (2) the script writes JSONL incrementally (one row at a time via atomic append) so partial progress persists across crashes; (3) `--limit N` flag allows smoke-testing on small N before committing to a full corpus run.
