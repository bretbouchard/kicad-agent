**Critical MPS constraints (from smoke testing):**
- fp16=True and bf16=True in SFTConfig cause ValueError on MPS. Set both False.
- Load model with `torch_dtype=torch.float16`, device_map="mps".
- dataloader_pin_memory=False (MPS does not support pinned memory).
- Use max_length=512 in SFTConfig (NOT max_seq_length -- removed in trl 1.0+).
- Set tokenizer.pad_token = tokenizer.eos_token before training.

**Training data format:** training_output/sft_prepared/train.jsonl with each line having a "messages" array of 3 dicts (system/user/assistant). SFTTrainer needs a "text" field -- use the ChatML-formatted text produced by tokenizer.apply_chat_template().

**Training speed:** ~20s/step at batch=1. With batch=4 and ~25K steps for 102K samples, expect 2-4 hours for 3 epochs.

```python
@dataclass
class SFTTrainingConfig:
    """Configuration for SFT training on Apple MPS."""
    model_name: str = "Qwen/Qwen2.5-1.5B-Instruct"
    output_dir: str = "training_output/sft_final"
    train_data_path: str = "training_output/sft_prepared/train.jsonl"
    val_data_path: str = "training_output/sft_prepared/val.jsonl"
    num_train_epochs: int = 3
    per_device_train_batch_size: int = 4
    learning_rate: float = 2e-4
    max_length: int = 512
    lora_r: int = 16
    lora_alpha: int = 32
    lora_dropout: float = 0.05
    warmup_ratio: float = 0.1
    logging_steps: int = 10
    seed: int = 42
    device: str = "auto"
```

Functions:
- `_get_device(config) -> str`: If device="auto", detect MPS/CUDA/CPU. Return "mps", "cuda", or "cpu".
- `_build_lora_config(config) -> LoraConfig`: Create LoRA config with `task_type=CAUSAL_LM`, `r=config.lora_r`, `lora_alpha=config.lora_alpha`, `lora_dropout=config.lora_dropout`, `target_modules=["q_proj", "k_proj", "v_proj", "o_proj"]`.
- `_build_sft_config(config) -> SFTConfig`: Create SFTConfig with fp16=False, bf16=False, dataloader_pin_memory=False, packing=False, max_length=config.max_length, all other params from config.
- `_prepare_dataset(data_path: str, tokenizer) -> Dataset`: Load JSONL via `Dataset.from_json()`. For each row, format the messages using `tokenizer.apply_chat_template(row["messages"], tokenize=False)` and store in "text" column. Return the dataset with only the "text" column.
- `run_sft_training(config: SFTTrainingConfig | None = None) -> dict`: Main training function:
  1. Resolve config (default if None)
  2. Detect device
  3. Load model: `AutoModelForCausalLM.from_pretrained(config.model_name, torch_dtype=torch.float16, device_map=device_str)`
  4. Load tokenizer: `AutoTokenizer.from_pretrained(config.model_name)`, set `tokenizer.pad_token = tokenizer.eos_token`
  5. Build LoRA config and SFT config
  6. Prepare train and val datasets
  7. Create SFTTrainer(model, args, train_dataset, eval_dataset, peft_config, processing_class=tokenizer)
  8. Call `trainer.train()`
  9. Save model: `trainer.save_model(config.output_dir)`
  10. Save tokenizer: `tokenizer.save_pretrained(config.output_dir)`
  11. Return training history dict

  Use lazy imports: `from __future__ import annotations` and import torch/transformers/peft/trl inside functions.

2. **`src/kicad_agent/training/sft/evaluator.py`** -- SFT evaluation:

- `generate_chain(model, tokenizer, prompt: str, max_new_tokens: int = 512) -> str`: Generate text from the model given a user prompt. Format as ChatML (system + user messages), tokenize, generate with `temperature=0.7, do_sample=True, top_p=0.9`, decode output.
- `evaluate_sft_model(adapter_path: str, test_samples: list[dict], reward_model_dir: str, n_samples: int = 50) -> dict`:
  1. Load base model with adapter: `AutoModelForCausalLM.from_pretrained("Qwen/Qwen2.5-1.5B-Instruct", torch_dtype=torch.float16, device_map="mps")`, then `PeftModel.from_pretrained(base_model, adapter_path)`.
  2. Load tokenizer.
  3. For each test sample, generate a chain using the user prompt from sample["messages"][1]["content"].
  4. Score generated chain with reward model: `predict_reward(reward_model, generated_text)`.
  5. Return dict with avg_reward, avg_format, avg_quality, avg_accuracy, n_samples, sample_outputs (first 5 for inspection).
- `compare_base_vs_sft(adapter_path: str, test_data_path: str, reward_model_dir: str, n_samples: int = 50) -> dict`:
  1. Run evaluate_sft_model with the adapter (SFT model).
  2. Run evaluate_sft_model with adapter_path=None (base model only).
  3. Return comparison dict with delta_reward, delta_format, delta_quality, delta_accuracy, base_results, sft_results.
