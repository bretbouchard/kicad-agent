Run the full SFT evaluation and write integration tests:

1. **Load test data** from `training_output/sft_prepared/test.jsonl`. Read the first 50 samples for evaluation (balance between coverage and speed).

2. **Run compare_base_vs_sft()**:
   - `adapter_path="training_output/sft_final"`
   - `test_data_path="training_output/sft_prepared/test.jsonl"`
   - `reward_model_dir="training_output/unified"`
   - `n_samples=50`
   This loads both base and SFT models, generates chains for the same test prompts, scores them with the reward model, and returns comparison metrics.

3. **Write eval_report.json** to `training_output/sft_final/eval_report.json` with:
   - `base_results`: {avg_reward, avg_format, avg_quality, avg_accuracy, n_samples}
   - `sft_results`: {avg_reward, avg_format, avg_quality, avg_accuracy, n_samples}
   - `delta_reward`: sft_avg_reward - base_avg_reward
   - `delta_format`: sft_avg_format - base_avg_format
   - `delta_quality`: sft_avg_quality - base_avg_quality
   - `delta_accuracy`: sft_avg_accuracy - base_avg_accuracy
   - `sample_outputs`: list of 5 example comparisons with prompt, base_output, sft_output, base_score, sft_score

4. **Write tests/test_sft_evaluator.py** with integration tests:
   - `test_sft_generates_valid_chain`: Load the SFT adapter, generate a response to a test prompt ("Analyze the PCB routing problem: Board is 30x30mm with 15 obstacles."), verify the output is non-empty and contains coordinate-grounded language (check for "<point" or "obstacle" or "mm").
   - `test_eval_report_exists_and_valid`: Load eval_report.json, verify it has base_results, sft_results, and delta_reward keys. Verify delta_reward is a number.
   - `test_sft_scores_higher_than_base`: Load eval_report.json, verify sft_results["avg_reward"] > base_results["avg_reward"].

   These are integration tests that depend on the trained model artifacts. Mark them with `@pytest.mark.skipif` that checks for the adapter file existence, so they pass gracefully if artifacts are not yet built.

5. **Validation**: After running evaluation, print a summary:
   - Base model composite score: X.XXX
   - SFT model composite score: X.XXX
   - Improvement: +X.XXX (+X.X%)
   - Sample output comparison (first example)

If the SFT model does NOT score higher than base (delta_reward <= 0), this is a valid result to report -- it means the training did not improve reward-model-scored quality. Document it in the eval report with the actual metrics and note that hyperparameter tuning (learning rate, epochs, LoRA rank) may be needed. Do NOT adjust scores to make it look like improvement.
