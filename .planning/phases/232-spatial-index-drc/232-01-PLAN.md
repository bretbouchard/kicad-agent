# Plan 232-01: Spatial Index for DRC Performance

<objective>
Current DRC implementation does O(n^2) pairwise checks on copper pads,
courtyards, and holes. Replace with a spatial index (R-tree or
SpatialHash) for O(log n) lookups. Target: 1000-component board DRC
runs in <5 seconds.

## Approach

1. Profile current DRC to confirm n^2 bottleneck
2. Add R-tree dependency (`rtree>=1.0`)
3. Build spatial index from PCB geometry at DRC start
4. Replace pairwise iteration with index queries
5. Validate parity (Phase 234B C1 SpatialHash parity test)

audit before implementing.
