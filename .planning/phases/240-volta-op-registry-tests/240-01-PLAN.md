<context>
## Why this exists
Gap A1 from GAP-ANALYSIS-CURRENT.md. The 268-op Volta registry has zero tests. Every op change risks silent regression. The 50/50 ERC parity test (Phase 218) was a one-shot; we need durable coverage.

## Background
Phases 191-200 set up the Swift testing framework (XCTest, snapshot, property-based, mutation, accessibility, concurrency, daemon, CI gates) but no phase shipped op-level coverage. macos-app/Tests has ~25 files; none iterate the op registry.

## Scope (in)
- Coverage test: iterate the registry, assert each op has a test method
- Property tests for read-only ops (list, count, get)
- Round-trip tests for write ops (add then remove, set then unset)
- 20+ .kicad_sch fixtures covering: LED + R, RC filter, op-amp, 555 timer, voltage regulator, MCU breakout, motor driver, USB-C, sensor breakout, hierarchical sheet, multi-unit IC, power supply, audio amp, RF filter, antenna, FPGA, BGAs, fiducials, test points
- Each fixture has a SHA256 pin in the test code

## Scope (out)
- Mutation testing (Phase 194 already covered)
- Property-based generation of schematics (start with fixtures; expand later)
</context>

### Task 1: OpRegistry introspection
Files: macos-app/Sources/Volta/Parsing/VoltaOpRegistry.swift (extend)

Add a static `allOpTypes: [String]` property that lists every op name. Use reflection on the three enums in VoltaEngine*.swift.

### Task 2: Coverage test
Files: macos-app/Tests/VoltaTests/VoltaOpRegistryTests.swift (new)

XCTest case that iterates `VoltaOpRegistry.allOpTypes` and for each op, looks up `test_<opName>` via reflection. Fails if any op lacks a test.

### Task 3: Test fixture corpus
Files: macos-app/Tests/VoltaTests/Fixtures/ (new)

20+ minimal .kicad_sch files covering common topologies. Each fixture has a sibling .json with SHA256 + description.

### Task 4: Per-op tests
Files: macos-app/Tests/VoltaTests/VoltaOpTests/ (new directory)

For each op, a test method. Read-only ops: assert return value matches expected. Write ops: apply, parse, assert state changed; apply reverse, assert state restored.

### Task 5: CI gate
Files: macos-app/scripts/ci_op_coverage.sh (new)

Bash script: run tests, parse xcresult, fail if coverage < 80%. Wire into existing CI workflow (Phase 200).
