<context>
## Why this exists
Gap A8 from GAP-ANALYSIS-CURRENT.md. The streaming chat pipeline (router → provider → stream → chunk → bubble render) has no end-to-end test. The echo bug that started this whole conversation is exactly the kind of regression this gap allowed.

## Background
Individual pieces have tests: NoopChatStream works, ContentChunker has logic. But the integration is untested. We need a test that runs the full pipeline and asserts the final output matches expectations.

## Scope (in)
- E2E test with canned responses
- Echo-stripping verification (the bug from this session)
- Sentence/paragraph boundary chunking verification
- Cost callback verification
- Loop-collapse verification (ContentChunker dedup)

## Scope (out)
- Live provider tests (use NoopChatStream; live tests are flaky)
- SwiftUI render tests (out of scope for E2E; covered by view snapshot tests)
</context>

### Task 3: Full pipeline test
Files: macos-app/Tests/VoltaTests/ChatPipelineE2ETests.swift

For each canned fixture:
1. Build a [ChatMessage] with a user prompt
2. Run RouterStreamProvider with the canned stream
3. Collect all emitted chunks
4. Run ContentChunker.chunk on the concatenated text
5. Assert:
   - Echoed prompt is stripped from the first chunk
   - Paragraph breaks produce separate chunks
   - Sentence-end chunks fire on boundary
   - Loop is collapsed with (xN) annotation
