- macos-app/Tests/KiCadAgentTests/ModelRouterTests.swift
</read_first>

<action>
The current `KiCadModelRouter` has best-effort fallback. Add:
1. **Detect 429**: In the provider response handler, catch `HTTPError.status(429)` and treat as a fallback signal.
2. **Exponential backoff**: Wait 2^attempt seconds (capped at 30s) before falling through to the next provider.
3. **Provider chain fallback**: On 429, switch to the next provider in the chain.
4. **User-facing toast**: Emit `ModelRouterEvent.providerSwitched(from:to:reason: .rateLimit)` for the UI to bind to a toast.

- `test_429_falls_through_to_next_provider` — mock provider A returns 429, assert provider B is called
- `test_backoff_doubles_per_attempt` — assert timing ~2s, 4s, 8s (allow slop)
- `test_429_emits_providerSwitched_event` — assert the toast event fires
- `test_non_429_does_not_trigger_chain` — 500 errors should not silently switch
