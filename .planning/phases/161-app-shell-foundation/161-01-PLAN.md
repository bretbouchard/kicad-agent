<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User → App | Untrusted input via text fields, buttons |
| App → SwiftData | Trusted internal storage (local-first) |
| App → macOS | System APIs (appearance preferences, window management) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-161-01 | S | Project titles, conversation names | accept | User input only, no external processing. XSS not applicable (SwiftUI text rendering). |
| T-161-02 | I | SwiftData persistence | mitigate | Validate ModelContainer schema on launch, use @Model macro type safety, no raw SQL injection possible. |
| T-161-03 | D | File system (documents, downloads) | mitigate | App Sandbox entitlements restrict access. User explicitly grants file access via NSOpenPanel. |
| T-161-04 | E | App lifecycle (launch/quit) | accept | Standard macOS app lifecycle. No sensitive data in crash logs. |
| T-161-05 | R | Multi-window state | accept | WindowGroup managed by macOS. No cross-window privilege escalation (same app context). |

## Adversarial Scenario

Name at least ONE specific, plausible attack vector relevant to THIS plan's scope:
**Attack:** Malicious project file with crafted title causing app crash or UI confusion. **Mitigation:** SwiftData @Model validates string types, SwiftUI text rendering sanitizes input, app uses system-provided text components (not custom parsing).
</threat_model>
