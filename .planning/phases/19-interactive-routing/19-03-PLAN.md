<objective>
Build the interactive routing session with suggestion/approval API and constraint adaptation.

Purpose: The interactive mode closes the loop between AI suggestions and human engineering judgment. Users review proposed routes, approve the good ones, reject problematic ones, and optionally add constraints (e.g., "increase clearance around this net"). The session re-routes rejected nets with updated constraints, iterating until all nets are resolved or max_iterations is reached.

Output: interactive.py with RoutingSuggestion, SuggestionStatus, InteractiveRoutingSession classes, updated __init__.py exports, and passing tests covering ROUTE-04.
</objective>

    1. interactive.py -- SuggestionStatus enum, RoutingSuggestion dataclass, InteractiveRoutingSession class:

       SuggestionStatus (str, Enum):
       - PENDING = "pending"
       - APPROVED = "approved"
       - REJECTED = "rejected"

       RoutingSuggestion (regular dataclass, NOT frozen -- status mutates):
       - net_name: str
       - path: list[tuple[float, float]]
       - length_mm: float
       - clearance_violations: list[str] (DRC violations if any, initially empty)
       - status: SuggestionStatus = SuggestionStatus.PENDING
       - reject_reason: str = ""
       - user_constraints: dict[str, float] = {} (e.g., {"min_clearance_mm": 0.4})
       - is_differential_pair: bool = False
       - diff_pair_complement: str = "" (net name of the other half, if diff pair)

       InteractiveRoutingSession:
       Constructor:
       - graph: RoutingGraph (initial graph)
       - netlist: dict[str, list[tuple[float, float]]] (net name -> pin positions)
       - constraints: RoutingConstraints (default constraints)
       - max_iterations: int = 5
       - differential_pairs: list[tuple[str, str]] = [] (pairs of net names for diff pair routing)

       On construction, call _generate_suggestions() which:
       - Routes all nets via route_all_nets from pathfinder.py
       - For nets in differential_pairs, uses route_differential_pair instead
       - Creates RoutingSuggestion for each result
       - Sets iteration = 0

       Methods:
       - approve(net_name: str) -> None:
         Find suggestion by net_name, set status=APPROVED, append to self.locked_routes.
         If net is part of a diff pair, also approve its complement.
         Raise KeyError if net_name not found in suggestions.

       - reject(net_name: str, reason: str = "") -> None:
         Find suggestion by net_name, set status=REJECTED, set reject_reason.
         If net is part of a diff pair, also reject its complement.
         Raise KeyError if net_name not found.

       - set_constraint(net_name: str, key: str, value: float) -> None:
         Find suggestion by net_name, update user_constraints[key] = value.

       - reroute_rejected() -> list[RoutingSuggestion]:
         If iteration >= max_iterations, raise RuntimeError("Max iterations reached").
         Collect all rejected nets and their user_constraints.
         If any rejected net has increased clearance constraint, rebuild RoutingGraph with updated RoutingConstraints (use max of global constraint and per-net constraint for the rebuild -- this is conservative but correct for a single-layer router).
         Re-route rejected nets only. Locked routes remain in place (their waypoints are added as obstacles to the graph to prevent overlap).
         Create new RoutingSuggestion for each re-routed net, set status=PENDING.
         Replace rejected suggestions with new ones in self.suggestions.
         Increment iteration counter.
         Return list of new suggestions.

       - summary() -> dict:
         Returns {"iteration": int, "total_nets": int, "approved": int, "rejected": int, "pending": int, "max_iterations": int}

       Properties:
       - suggestions: list[RoutingSuggestion] (all current suggestions)
       - locked_routes: list[RoutingSuggestion] (approved only)
       - iteration: int (current iteration count)
       - is_complete: bool (all suggestions are either approved or no more iterations)
