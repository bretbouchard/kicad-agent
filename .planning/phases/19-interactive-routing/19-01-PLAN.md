  <action>
    Create the routing module from scratch with 4 files:

    1. constraints.py -- RoutingConstraints frozen dataclass with fields:
       - clearance_mm: float = 0.2
       - grid_resolution_mm: float = 0.5
       - trace_width_mm: float = 0.25
       - via_diameter_mm: float = 0.8
       - via_drill_mm: float = 0.4
       - max_nodes: int = 500_000
       Validate that clearance_mm > 0, grid_resolution_mm >= 0.1, max_nodes <= 1_000_000.

    2. graph.py -- RoutingGraph class that wraps a networkx.Graph:
       - Constructor takes board_bounds (x_min, y_min, x_max, y_max), obstacles (list[SpatialBox]), constraints (RoutingConstraints), and an optional SpatialQueryEngine.
       - Creates grid nodes at grid_resolution_mm intervals within board_bounds. Skip nodes whose point is inside any obstacle bounding box (use Shapely Point.within for exact check).
       - Creates edges between adjacent grid nodes (4-directional: up/down/left/right). For each edge, compute cost as the segment length (grid_resolution_mm) plus a DRC penalty if the edge segment is within clearance_mm of any obstacle. Use SpatialQueryEngine.proximity() on the midpoint of the edge to check for nearby obstacles. If proximity returns obstacles, add penalty proportional to how close the nearest obstacle is.
       - Edges that would violate clearance (edge midpoint is closer than clearance_mm to an obstacle surface, accounting for trace_width_mm/2 margin) are omitted entirely.
       - Cap total nodes at max_nodes; raise ValueError if exceeded.
       - Store the graph as self._graph (networkx.Graph). Expose a read-only property.
       - Provide snap_to_node(x, y) method that returns the nearest graph node within grid_resolution_mm tolerance, or None if no node is close enough.

    3. pathfinder.py -- RouteResult frozen dataclass and pathfinding functions:
       - RouteResult(net_name, path as tuple of (x,y) tuples, length_mm, success bool)
       - route_net(graph: RoutingGraph, source: tuple, target: tuple, net_name: str) -> RouteResult | None
         Uses networkx.astar_path with Euclidean distance heuristic and "weight" edge attribute. Returns None on NetworkXNoPath.
       - route_all_nets(graph: RoutingGraph, netlist: dict[str, list[tuple[float,float]]]) -> dict[str, RouteResult]
         Takes a netlist mapping net names to lists of pin positions. For each net with >=2 pins, route from first to last pin (multi-pin nets use first-to-last for this phase). Sort nets by estimated Euclidean distance (shortest first) to reduce conflicts. Return dict of net_name -> RouteResult.
       - _path_length helper computes total Euclidean length of a waypoint list.

    4. __init__.py -- barrel exports: RoutingConstraints, RoutingGraph, RouteResult, build_routing_graph (convenience function), route_net, route_all_nets.

    5. tests/test_routing.py -- unit tests covering all behaviors listed above. Use SpatialBox/SpatialPoint from spatial.primitives for test fixtures. Create small boards (20x20mm) for fast tests. Test graph construction, pathfinding success/failure, DRC clearance enforcement, and netlist routing.

    Follow the frozen dataclass pattern from spatial/primitives.py. Use lazy networkx import. Do not depend on kicad-cli or real PCB files -- all tests use synthetic spatial data.
