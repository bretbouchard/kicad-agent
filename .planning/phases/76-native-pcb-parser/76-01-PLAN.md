    Parsing strategy (per D-01 using sexpdata, reusing pcb_netlist.py patterns):

    0. CRITICAL-1 FIX (Council Security): Depth pre-scan BEFORE sexpdata.loads().
    Add a module-level function `_pre_scan_depth(content: str, max_depth: int = 200) -> int`
    that counts the maximum nesting depth of parentheses in the content WITHOUT
    parsing it. Uses a simple counter scanning '(' and ')' characters. If depth
    exceeds max_depth, raise ValueError before calling sexpdata.loads(). This prevents
    the RecursionError entirely — CPython's RecursionError is unsafe to catch because
    it can leave the interpreter in an inconsistent state. Also wrap sexpdata.loads()
    in sys.setrecursionlimit() guard as defense-in-depth.

    1. Load content. Apply size limit from raw_parser.py (50MB max). Run depth pre-scan.
       Call sexpdata.loads(content) only if depth is within limits.

    2. Find the (kicad_pcb ...) root symbol. If not found, return empty NativeBoard with logging.

    3. Extract version and generator from the root: _find_symbol(root, "version"), _find_symbol(root, "generator").

    4. Extract nets. Walk root for all top-level (net N "NAME") symbols. Per D-08, parse BEFORE any normalization -- extract both the integer N and string NAME. Do NOT call _preprocess_nets() here; that function normalizes away the number. Instead, parse the raw sexpdata tree where (net 1 "GND") appears as [Symbol("net"), 1, "GND"], and extract both number=1 and name="GND".

    5. Extract net_classes. Walk for (net_class "Name" ...) symbols. Extract name, clearance, track_width, via_diameter, via_drill from child symbols. Extract (add_net "name") children into add_nets list.

    6. Extract footprints. Walk for (footprint "lib:fp" ...) symbols. For each:
       - lib_id: second element of the footprint list (string after "footprint" symbol)
       - position: _find_at() returns [x, y, angle] -- store as (x, y, angle)
       - uuid: look for (uuid "...") child
       - layer: look for (layer "...") child
       - properties: walk for (property "Name" "Value" ...) -- store in properties dict
       - pads: walk for (pad ...) children. For each pad:
         - number: second element (string)
         - pad_type: third element (e.g. "smd", "thru_hole")
         - shape: fourth element (e.g. "rect", "circle")
         - position: _find_at() within pad
         - size: look for (size W H) within pad
         - layers: look for (layers "...") within pad
         - drill: look for (drill D) within pad
         - net: look for (net N "NAME") within pad. Per D-08, extract both number and name from the raw tree. Handle both forms: (net N "NAME") and (net "NAME") -- the latter has no number, default to 0.
       - graphic_items: walk for (fp_line ...), (fp_arc ...), (fp_circle ...) children

    7. Extract segments. Walk for (segment ...) symbols. Extract start, end, width, layer, net.

    8. Extract vias. Walk for (via ...) symbols. Extract position, drill, diameter, net, layers.

    9. Extract zones. Walk for (zone ...) symbols. Extract net_name, net_number, layer, priority, uuid, clearance, polygon points. Also populate CRITICAL-2 compatibility fields: zone.net = zone.net_number, zone.netName = zone.net_name, zone.layers = [zone.layer], zone.minThickness = 0.25 (KiCad default). Extract min_thickness from (min_thickness V) child if present.

    10. Extract graphic items. Walk for (gr_line ...), (gr_arc ...), (gr_circle ...), (gr_rect ...), (gr_poly ...), (gr_curve ...). Council HIGH-2: added rect, poly, curve types present in real boards. For each, extract coordinates, layer, width, uuid. Build item_type from the symbol name (gr_line→"line", gr_arc→"arc", gr_circle→"circle", gr_rect→"rect", gr_poly→"poly", gr_curve→"curve").

    11. Extract general section. Walk for (general ...) symbol. Extract thickness from (thickness T) child (default 1.6). Extract layers from (layers ...) child. Store in NativeBoard.general (NativeGeneral dataclass).

    12. Extract setup section. Walk for (setup ...) symbol. If present, create NativeSetup with placeholder NativeStackup. Full stackup parsing deferred to future phase — the placeholder ensures hasattr(board, "setup") returns True and hasattr(board.setup, "stackup") returns True, preventing AttributeError in spatial/layer_stackup.py.

    13. Build board_outline. Collect all graphic_items where layer == "Edge.Cuts" into a NativeBoardOutline.

    Tree-walking helpers: Reuse the pattern from pcb_netlist.py. Define _sym(), _find_symbol(), _find_all_symbols(), _find_at() as module-level functions (copy from pcb_netlist.py, they are small and generic).
