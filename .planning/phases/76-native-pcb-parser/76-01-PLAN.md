    Create src/kicad_agent/parser/pcb_native_parser.py implementing the native sexpdata-based PCB parser per D-01 and D-05.

    Structure the module as a NativeParser class with two class methods:
    - parse_pcb(path: Path) -> NativeBoard -- reads file, parses via sexpdata, returns NativeBoard
    - parse_pcb_content(content: str, file_path: str = "") -> NativeBoard -- parses raw content string

    Parsing strategy (per D-01 using sexpdata, reusing pcb_netlist.py patterns):

    1. Load content. Apply size limit from raw_parser.py (50MB max). Call sexpdata.loads(content).

    2. Find the (kicad_pcb ...) root symbol. If not found, return empty NativeBoard with logging.

    3. Extract version and generator from the root: _find_symbol(root, "version"), _find_symbol(root, "generator").

    4. Extract nets. Walk root for all top-level (net N "NAME") symbols. Per D-08, parse BEFORE any normalization -- extract both the integer N and string NAME. Do NOT call _preprocess_nets() here; that function normalizes away the number. Instead, parse the raw sexpdata tree where (net 1 "GND") appears as [Symbol("net"), 1, "GND"], and extract both number=1 and name="GND".

    5. Extract net_classes. Walk for (net_class "Name" ...) symbols. Extract name, clearance, track_width, via_diameter, via_drill from child symbols. Extract (add_net "name") children into add_nets list.

    6. Extract footprints. Walk for (footprint "lib:fp" ...) symbols. For each:
       - lib_id: second element of the footprint list (string after "footprint" symbol)
       - position: _find_at() returns [x, y, angle] -- store as (x, y, angle)
       - uuid: look for (uuid "...") child
       - layer: look for (layer "...") child
       - properties: walk for (property "Name" "Value" ...) -- store in properties dict
       - pads: walk for (pad ...) children. For each pad:
         - number: second element (string)
         - pad_type: third element (e.g. "smd", "thru_hole")
         - shape: fourth element (e.g. "rect", "circle")
         - position: _find_at() within pad
         - size: look for (size W H) within pad
         - layers: look for (layers "...") within pad
         - drill: look for (drill D) within pad
         - net: look for (net N "NAME") within pad. Per D-08, extract both number and name from the raw tree. Handle both forms: (net N "NAME") and (net "NAME") -- the latter has no number, default to 0.
       - graphic_items: walk for (fp_line ...), (fp_arc ...), (fp_circle ...) children

    7. Extract segments. Walk for (segment ...) symbols. Extract start, end, width, layer, net.

    8. Extract vias. Walk for (via ...) symbols. Extract position, drill, diameter, net, layers.

    9. Extract zones. Walk for (zone ...) symbols. Extract net_name, net_number, layer, priority, uuid, clearance, polygon points.

    10. Extract graphic items. Walk for (gr_line ...), (gr_arc ...), (gr_circle ...). For each, extract coordinates, layer, width. Build item_type from the symbol name.

    11. Build board_outline. Collect all graphic_items where layer == "Edge.Cuts" into a NativeBoardOutline.

    Tree-walking helpers: Reuse the pattern from pcb_netlist.py. Define _sym(), _find_symbol(), _find_all_symbols(), _find_at() as module-level functions (copy from pcb_netlist.py, they are small and generic). Also add _find_first_value(tree, name, default) for extracting single-valued children like (width 0.25).

    Important details:
    - Per D-08, do NOT call _preprocess_nets() before parsing. The native parser needs net numbers. Parse the raw content directly.
    - Use logging.getLogger(__name__) for warnings on parse failures.
    - Do NOT import kiutils in this module. Only sexpdata and the native types.
    - All tree walking should be defensive -- use try/except around float() conversions, return defaults for missing children.
