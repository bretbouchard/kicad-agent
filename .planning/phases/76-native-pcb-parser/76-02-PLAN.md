  <action>
    Modify PcbIR to work with NativeBoard via the adapter pattern (D-04 LOCKED).

    The key insight: PcbIR currently accesses self.board which returns a kiutils Board. The adapter approach means PcbIR's board property returns EITHER a NativeBoard OR a kiutils Board. PcbIR methods must handle both cases.

    Strategy: Add a `_native_board` field to PcbIR. When the native parser succeeds, `_native_board` is a NativeBoard. When it fails (fallback), `_native_board` is None and PcbIR falls back to the existing kiutils path.

    Step 1: Modify PcbIR.__post_init__
    - Add `_native_board: NativeBoard | None = None` as a class-level field default
    - Relax the UUID map requirement: when _native_board is set, UUID map is optional (D-07 -- UUIDs are preserved automatically in the native path). Only require UUID map when _native_board is None (kiutils fallback).

    Step 2: Modify PcbIR.board property
    - If _native_board is not None, return _native_board
    - Otherwise return self._parse_result.kiutils_obj (existing behavior)

    Step 3: Modify PcbIR methods that access self.board attributes
    The methods access these board attributes: .nets, .footprints, .graphicItems. For NativeBoard:
    - .nets -> board.nets (same name, list of NativeNet with .number, .name)
    - .footprints -> board.footprints (same name, list of NativeFootprint)
    - .graphicItems -> board.graphic_items (different name! Add a property or check hasattr)

    For PcbIR methods that iterate footprints and access footprint attributes:
    - fp.libId -> fp.lib_id (underscore difference). Use getattr(fp, 'libId', fp.lib_id) or check type.
    - fp.position.X -> fp.position[0] (tuple vs kiutils Position). Add a helper.
    - fp.position.Y -> fp.position[1]
    - fp.position.angle -> fp.position[2] if len(fp.position) > 2 else 0.0
    - fp.properties.get("Reference") -> fp.properties.get("Reference") (same interface -- both are dicts)
    - fp.pads -> fp.pads (same name)
    - pad.net -> need adapter: NativePad has .net_name and .net_number, kiutils pad has .net (Net object). Check type.
    - pad.net.name -> pad.net_name
    - pad.net.number -> pad.net_number
    - pad.number -> pad.number (same)
    - pad.position.X -> pad.position[0]
    - pad.position.Y -> pad.position[1]
    - fp.layer -> fp.layer (same)
    - fp.graphicItems -> fp.graphic_items (check hasattr, use getattr)

    Implementation approach: Add a `_is_native` helper property that returns True when _native_board is set. In each PcbIR method, use if self._is_native: native_path else: kiutils_path.

    For the methods that mutate (add_net, remove_net, rename_net, swap_footprint):
    - add_net: create NativeNet instead of kiutils Net. Append to board.nets.
    - remove_net: filter board.nets (same list operation). For pad disconnection, iterate board.footprints and set pad.net_name = "" and pad.net_number = 0.
    - rename_net: update NativeNet in board.nets. For pad propagation, update pad.net_name.
    - swap_footprint: modify fp.lib_id (string attribute, same name). For pad net preservation, store pad.net_name/net_number before swap, restore after.

    For get_board_bounds():
    - If native: iterate board.graphic_items (not graphicItems). Check item.layer == "Edge.Cuts". Items have .start and .end tuples.
    - If kiutils: existing logic unchanged.

    For extract_netlist():
    - If native: iterate board.footprints. fp.position[0] for x, fp.position[1] for y. pad.position[0] for pad_x. pad.net_name for net name.
    - If kiutils: existing logic unchanged.

    Step 4: Modify executor._execute_pcb
    - Before the existing parse_pcb() call, try NativeParser.parse_pcb() first.
    - If NativeParser succeeds (returns NativeBoard with footprints or nets), create PcbIR with _native_board set and no UUID map requirement.
    - If NativeParser raises or returns empty, fall back to existing parse_pcb() + extract_uuids() path. Log a warning per D-03.
    - The raw_written gate and serialize_pcb call remain unchanged for the kiutils fallback path. For the native path, raw_written continues to work the same way (PcbRawWriter writes, sets flag, executor skips kiutils serialization).
