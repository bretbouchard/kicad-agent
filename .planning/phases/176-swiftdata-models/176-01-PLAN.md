## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SwiftData write | Any code path can insert models — enforced at call site |
| CloudKit sync | External iCloud service syncs private database — encrypted transit |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-176-01 | T (Tampering) | SwiftData models | mitigate | All @Attribute(.unique) on UUID prevents clone attacks. CloudKit metadata (ckRecordChangeTag) detects server-side tampering. |
| T-176-02 | I (Information disclosure) | ValueChange old/new values | accept | Values stored plain-text for diff rendering. No PII in schematic values. Acceptable for v1 (encrypted at rest via macOS). |
| T-176-03 | D (Denial of Service) | Decision.eventData payload size | mitigate | Limit eventData to 1MB (validate on insert). Reject oversized payloads with user error. |
| T-176-04 | E (Elevation) | Message.role field | mitigate | Role enum restricted to (user/assistant/system). No custom roles. Validation gate on insert. |
| T-176-05 | R (Repudiation) | ValueChange.actor field | mitigate | Actor field auto-populated from currentUser (read-only). Cannot spoof identity. |

## Adversarial Scenario

**Name:** SwiftData relationship injection via crafted JSON payload

**Attack:** Malicious JSON-RPC from compromised daemon inserts Decision with linkedMessageId pointing to non-existent Message, causing relationship query to crash app.

**Mitigation:** SwiftData validates foreign key constraints on insert. Invalid UUIDs rejected at ModelContext.insert(). Add test: insert Decision with fake linkedMessageId → expect SwiftData ValidationError.

**Detection:** Unit test `testDecisionWithInvalidLinkedMessageIdThrows` validates this path.
</threat_model>
