## Adversarial Scenario

**Name:** "Journal injection attack via malformed conversation JSON"

**Scenario:** Attacker crafts malicious conversation JSON with:
- Circular net references (Net A connects to Net B, Net B connects to Net A)
- Duplicate component refs (two "R1" components)
- Empty footprints (components without footprint assignment)

**Attack vector:** If compiler doesn't validate, it emits invalid SKIDL IR → downstream generator produces corrupt KiCad files → ERC/DRC fail silently or produce garbage PCBs.

**Mitigation (when v6.1 Tasks 2-5 execute):**
1. `validate_component_references()` checks for duplicate refs → fails with "Duplicate component ref: R1"
2. `validate_footprint_assignments()` rejects empty footprints → fails with "Missing footprint for component R1"
3. Cycle detection in `validate_net_connectivity()` → fails with "Circular net reference detected: A ↔ B"

All validation failures raise StructuredError with actionable message, preventing invalid IR from reaching downstream stages.
</threat_model>
