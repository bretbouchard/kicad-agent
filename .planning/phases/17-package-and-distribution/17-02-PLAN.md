  <action>
Create `.github/workflows/build.yml` (create the .github/workflows/ directory first if it does not exist).

The workflow should:

```yaml
name: Build

on:
  push:
    branches: [master, main]
  pull_request:
    branches: [master, main]

jobs:
  build:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ["3.11", "3.12", "3.13"]

    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # Full history for setuptools-scm versioning

      - name: Set up Python ${{ matrix.python-version }}
        uses: actions/setup-python@v5
        with:
          python-version: ${{ matrix.python-version }}

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install ".[dev]"
          pip install build setuptools-scm

      - name: Lint with ruff
        run: ruff check src/ tests/

      - name: Type check with mypy
        run: mypy src/

      - name: Run tests
        run: python -m pytest tests/ -x -q --tb=short

      - name: Build package
        run: python -m build

      - name: Verify wheel contents
        run: |
          unzip -l dist/*.whl | head -30
          # Verify no tests in wheel
          if unzip -l dist/*.whl | grep -q "tests/"; then
            echo "ERROR: tests/ found in wheel"
            exit 1
          fi
          # Verify version is not fallback
          VERSION=$(unzip -p dist/*.whl '*/METADATA' | grep "^Version:" | head -1 | cut -d' ' -f2)
          echo "Package version: $VERSION"
          if [ "$VERSION" = "0.0.0" ]; then
            echo "ERROR: Version is fallback 0.0.0 -- setuptools-scm failed"
            exit 1
          fi
```

Key details:
- `fetch-depth: 0` is CRITICAL -- without full git history, setuptools-scm returns "0.0.0"
- Test matrix covers Python 3.11, 3.12, 3.13 (matching classifiers from Plan 01)
- Build verification step checks wheel excludes tests/ and version is not fallback

<task type="auto">
  <name>Task 2: Create PyPI publishing workflow with Trusted Publishing</name>
  <files>.github/workflows/publish.yml</files>
  <action>
Create `.github/workflows/publish.yml`:

```yaml
name: Publish to PyPI

on:
  push:
    tags:
      - "v*.*.*"

jobs:
  publish:
    runs-on: ubuntu-latest
    environment: pypi
    permissions:
      id-token: write  # Required for Trusted Publishing (OIDC)

    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # Full history for setuptools-scm versioning

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.13"

      - name: Install build dependencies
        run: |
          python -m pip install --upgrade pip
          pip install build setuptools-scm

      - name: Build package
        run: python -m build

      - name: Verify package version matches tag
        run: |
          TAG_VERSION="${GITHUB_REF#refs/tags/v}"
          PKG_VERSION=$(unzip -p dist/*.whl '*/METADATA' | grep "^Version:" | head -1 | cut -d' ' -f2)
          echo "Tag version: $TAG_VERSION"
          echo "Package version: $PKG_VERSION"
          if [ "$TAG_VERSION" != "$PKG_VERSION" ]; then
            echo "ERROR: Tag version ($TAG_VERSION) does not match package version ($PKG_VERSION)"
            exit 1
          fi

      - name: Run tests before publish
        run: |
          pip install ".[dev]"
          python -m pytest tests/ -x -q

      - name: Publish to PyPI
        uses: pypa/gh-action-pypi-publish@release/v1
```

Key details:
- Triggers ONLY on version tags matching `v*.*.*` (e.g., `v0.1.0`)
- `environment: pypi` -- the user must create a "pypi" environment in GitHub repo settings and configure Trusted Publishing on PyPI to trust this repo+environment+workflow
- `permissions: id-token: write` enables OIDC token for Trusted Publishing
- Version verification step catches tag/metadata mismatches before upload
- Tests run as a gate before publish (ensures release only happens on green tests)
- Uses `pypa/gh-action-pypi-publish@release/v1` -- the official PyPA action for Trusted Publishing (no twine, no API tokens)

After creating the workflow file, add a note in the plan SUMMARY about the one-time setup the user must do:
1. Create "pypi" environment in GitHub repo Settings > Environments
2. On PyPI, configure Trusted Publishing for this repository, workflow, and environment
3. First release: `git tag v0.1.0 && git push origin v0.1.0`
