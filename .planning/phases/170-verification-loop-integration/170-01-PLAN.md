## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Swift app → Intent Gate | Untrusted intent JSON crosses here — all fields validated |
| Verification Gate → validation_gates.py | Trusted internal boundary — gates assume valid KiCad files |
| PersistentUndoStack → File system | Trusted rollback — only writes within project boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-170-01 | Spoofing | Intent validation | mitigate | Validate op_type against registry, reject unknown ops, validate Pydantic schema |
| T-170-02 | Tampering | Verification bypass | mitigate | All ops must pass gate, no skip flag, failures auto-rollback |
| T-170-03 | Repudiation | Verification audit | mitigate | All verification results logged to OpJournal with fsync, rollback reasons recorded |
| T-170-04 | Information disclosure | Validation errors | accept | Error messages describe file state (no secrets exposed) |
| T-170-05 | Denial of Service | Verification loop | mitigate | Gates timeout after 30s (watchdog), rollback caps retries at T4 escalation |
| T-170-06 | Elevation of Privilege | File rollback | mitigate | PersistentUndoStack only writes within base_dir, no directory traversal |

## Adversarial Scenario

**Scenario:** Malicious Swift app sends intent with {"op_type":"add_component","target_file":"../../../../etc/passwd","lib_id":"Device:R"}

**Mitigation:**
- IntentGate validates target_file against allowed file_types for add_component (only .kicad_sch, .kicad_pcb)
- File type mismatch triggers drift detection (file not in approved list)
- Gate returns IntentGateResult(valid=False, errors=["Target_file /etc/passwd not allowed for add_component (expected .kicad_sch or .kicad_pcb)"])
- OpJournal logs attempted bypass (operation_id, intent, result="rejected_by_intent_gate")
- EscalationLadder records failure (task="intent_gate_bypass", severity=critical)

**Response:** Gate rejects intent, returns error, no file operations. Escalation triggered if repeated.
