# Plan 236-01: Vision Input (Camera → Schematic)

<objective>
Activate the multimodal capability of the Volta v2 12B adapter (Phase 245) by
letting users attach a photo of a schematic, breadboard, or hand-drawn circuit
directly in the chat and have the model convert it to SKiDL or operator JSON.

## Approach

1. Inventory current `KCAttachment` model and chat attachment UI
2. Audit `InferencePipeline` for vision-capable entry points
3. Verify Volta v2 LoRA adapter accepts image tokens (PaliGemma 2-style)
4. Wire `KCAttachment.imageData` into the prompt builder alongside text
5. Add "Generate from photo" affordance in chat composer
6. Test on representative schematic + breadboard images from corpus

## Constraints

- App Store sandbox: must use `NSOpenPanel` for file picker and
  `AVCaptureSession` for camera, with Info.plist permission strings
- Vision tokens count toward context window — enforce ~2 image max per turn
- iOS/iPadOS/visionOS share the same SwiftUI composer (only macOS targets now)

## Resolution state

PLANNED — vision pipeline plumbing requires camera entitlement + image
routing through the v2 adapter. Multi-day effort; deferred from v11.0 to
v12.0 (mobile-first milestone).
