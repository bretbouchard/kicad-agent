<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem -> manifest | JSONL files hashed by SHA256, content verification against stored hashes |
| eval results -> baseline store | EvalResult serialized to JSON on disk |
| filesystem -> cleanup | Training output directories scanned and deleted based on retention policy |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-04 | Tampering | manifest.py | mitigate | SHA256 content verification detects any file modification. verify() returns False on mismatch |
| T-37-05 | Denial of Service | cleanup.py | mitigate | Dry-run mode required before deletion. Retention policy keeps minimum N runs. No rm -rf -- explicit file-by-file deletion with logging |
| T-37-06 | Denial of Service | cleanup.py | mitigate | Cleanup skips directories that contain lock files or are less than 1 hour old (active runs) |
</threat_model>
