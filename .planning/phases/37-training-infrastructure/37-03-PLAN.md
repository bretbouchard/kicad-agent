| health_check output | Server state exposed to MCP clients (uptime, in-flight count) |
| smoke test -> filesystem | Temporary directories used for training output |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-07 | Denial of Service | shutdown handler | mitigate | Atomic flag check before operation dispatch prevents new work during drain. In-flight counter ensures operations complete before exit |
| T-37-08 | Information Disclosure | health_check | accept | health_check exposes uptime, in-flight count, executor readiness -- no secrets, no file paths beyond project_dir (already known to MCP client) |
| T-37-09 | Denial of Service | smoke test | accept | Smoke tests are developer tools, not production endpoints. timeout=120s prevents infinite hangs |
