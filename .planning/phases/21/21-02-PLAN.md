Create the evaluator module, write tests, then execute the GRPO training run:

**Part A: Create evaluator module**

1. **`src/kicad_agent/training/grpo_evaluator.py`** -- GRPO evaluation:

- `evaluate_grpo_model(adapter_path: str, test_data_path: str, reward_model_dir: str, model_name: str = "Qwen/Qwen2.5-0.5B-Instruct", n_samples: int = 50) -> dict`:
  1. Load model with adapter via mlx-lm `load(model_name, adapter_path=adapter_path)`
  2. Load reward model via `RewardModel.load_trained(reward_model_dir)`
  3. Load test prompts from test_data_path (ChatML JSONL)
  4. For each test prompt, generate 1 completion (temperature=0.7, max_tokens=512)
  5. Score each completion with reward model
  6. Return dict: {avg_reward, avg_format, avg_quality, avg_accuracy, n_samples, sample_outputs: list[dict]}

- `run_discrimination_test(adapter_path: str, reward_model_dir: str, test_data_path: str, model_name: str = "Qwen/Qwen2.5-0.5B-Instruct", n_samples: int = 50) -> dict`:
  1. Load model with adapter and reward model
  2. For each test prompt:
     a. Generate a completion (the model's "correct" answer)
     b. Score it with reward model
     c. Create a corrupted version of the completion (swap coordinates, remove steps, or use synthesize_corrupted_chain if MazeSample available)
     d. Score the corrupted version
     e. Record whether correct score > corrupted score
  3. discrimination_rate = count(correct > corrupted) / n_samples
  4. Return dict: {discrimination_rate, n_tested, avg_correct_score, avg_corrupted_score, gap, per_sample: list[dict]}

  For corruption: since we're generating from the LLM (not maze chains), use text-level corruption:
  - Randomly shuffle sentences
  - Replace coordinate `<point x,y>` references with wrong values
  - Remove every other sentence
  Pick one corruption strategy per sample at random.

- `compare_sft_vs_grpo(sft_adapter_path: str, grpo_adapter_path: str, test_data_path: str, reward_model_dir: str, model_name: str = "Qwen/Qwen2.5-0.5B-Instruct", n_samples: int = 50) -> dict`:
  1. Run evaluate_grpo_model for SFT adapter
  2. Run evaluate_grpo_model for GRPO adapter
  3. Run run_discrimination_test for GRPO adapter
  4. Return: {sft_results, grpo_results, discrimination, delta_reward, delta_format, delta_quality, delta_accuracy, grpo_wins_all_dimensions: bool}
