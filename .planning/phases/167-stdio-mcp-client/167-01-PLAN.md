## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Swift app → Python daemon | Untrusted stdio input crosses here — JSON-RPC must validate all fields |
| Python daemon → KiCad files | Trusted internal boundary — daemon has full file write access |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-167-01 | Spoofing | MCP stdio transport | mitigate | Validate JSON-RPC requests against schema (op_type must exist in registry), reject unknown methods |
| T-167-02 | Tampering | JSON-RPC messages | mitigate | Use JSON-RPC 2.0 spec with strict validation, reject malformed JSON, validate op arguments against Pydantic schema |
| T-167-03 | Repudiation | Daemon crash | mitigate | Audit log every daemon start/stop with timestamp and PID (logged to stderr), watchdog kills on 30s timeout |
| T-167-04 | Information disclosure | stdout buffering | accept | Line buffering prevents data loss, but stdio is local-only — no network exposure |
| T-167-05 | Denial of Service | Hung daemon | mitigate | 30-second watchdog kills and restarts daemon, heartbeat every 10s detects liveness |
| T-167-06 | Elevation of Privilege | subprocess spawn | mitigate | Daemon runs as same user as app (no privilege escalation), spawned via Process() with no special permissions |

## Adversarial Scenario

**Scenario:** Malicious JSON-RPC request sends {"jsonrpc":"2.0","id":"attack","method":"call_tool","params":{"name":"__import__('os').system('rm -rf /')","arguments":{}}}

**Mitigation:**
- MCP server validates tool name against registry (list_tools() only returns registered ops)
- Unknown tool names return {"error":{"code":-32601,"message":"Method not found"}}
- Pydantic schema validation prevents injection attacks (all arguments typed)
- Daemon subprocess has no special privileges (runs as app user)

**Response:** Server rejects unknown tool, logs attempt, returns JSON-RPC error. No code execution beyond registered ops.
</threat_model>
