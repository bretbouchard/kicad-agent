<context>
## Why this exists
Gap A2 from GAP-ANALYSIS-CURRENT.md. `safe_sync_pcb_from_schematic` is currently a stub returning a placeholder message (VoltaEngineRemaining.swift:933-942). Without a working sync, the iterate loop is broken: the user can edit the schematic but the PCB stays stale. This kills the "describe → refine → ship" flow.

## Background
Phase 235 was marked Complete on 2026-07-14 but did not actually implement this op. The current op just parses the PCB and returns a message saying "full sync requires schematic+PCB pair."

## Scope (in)
- A diff algorithm that takes (schematicIR, pcbIR) and emits a sequence of atomic PCB mutations
- Mutate: footprint add/remove/move (by ref), net rename, value update
- Journal the whole sync as one transaction (one-shot undo)
- Wire into VoltaEngineRemaining.swift's safe_sync_pcb_from_schematic

## Scope (out)
- Hierarchical sheet sync (Phase 2 follow-up)
- Annotation re-numbering sync (already in `safe_annotate`)
- DRC re-validation post-sync (the user runs DRC explicitly)
</context>

<tasks>

### Task 1: SchematicPcbDiffer
**Files:**
- `macos-app/Sources/Volta/Parsing/SchematicPcbDiffer.swift` (new, ~200 LOC)

**Implementation:**
- Inputs: `SchematicIR` (components + nets) and `PCBIR` (footprints + tracks + vias)
- Match components by reference designator
- For each mismatch, emit one of:
  - `.addFootprint(ref, footprint, position, value)` if schematic has it but PCB doesn't
  - `.removeFootprint(ref)` if PCB has it but schematic doesn't
  - `.moveFootprint(ref, position)` if positions differ
  - `.updateValue(ref, value)` if values differ
  - `.updateNet(pin, net)` if pin-to-net mapping changed
- Output: `[DifferOp]`

**Tests:** Property tests with synthetic (schematic, pcb) pairs.

### Task 2: Sync executor
**Files:**
- `macos-app/Sources/Volta/Parsing/SchematicPcbSync.swift` (new, ~150 LOC)

**Implementation:**
- Open a journal transaction
- Apply each `DifferOp` via the existing op registry (`add_footprint`, `move_footprint`, etc.)
- On any failure: roll back the transaction, return the error
- On success: return the diff summary `{added, removed, moved, valueUpdated, netUpdated}`
