<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Python Daemon → Swift App | stdio MCP bridge (untrusted input from Swift) |
| Python Daemon → KiCad Files | File I/O (untrusted file paths from ops) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-199-01 | I | MCP tool call arguments | mitigate | Pydantic schema validation before execution, reject malformed input |
| T-199-02 | I | stdio MCP input (RPC injection) | mitigate | JSON-RPC schema validation, reject non-JSON-RPC messages |
| T-199-03 | D | OpJournal file corruption | mitigate | fsync durability, atomic append writes, crash recovery on malformed lines |
| T-199-04 | E | Mutation testing false positives | mitigate | Human review required for surviving mutants, document why mutant is equivalent |
| T-199-05 | T | Coverage gaps (untested code paths) | mitigate | 100% line + branch enforced, mutation testing catches logic gaps |

## Adversarial Scenario

Name: Attacker crafts malicious MCP tool call that bypasses Pydantic validation. Attacker sends JSON-RPC with Python pickle payload in arguments field, attempts deserialization in daemon. Pydantic schema fails validation, daemon rejects request. But attacker finds schema loophole (unicode normalization bypass) and injects `__import__('os').system('rm -rf /')` in op arguments.

Mitigation: Pydantic schema uses strict mode (extra=forbid). Unicode normalization handled by Python json module ( rejects non-canonical unicode). All op arguments go through strict Pydantic BaseModel validation. Coverage test ensures every validation path is tested (100% enforcement). Mutation testing validates validation logic (mutants that bypass validation are killed).
</threat_model>
