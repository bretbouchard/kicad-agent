<objective>
Iterative refinement loop with LLM-driven error fixing: augment the existing deterministic refinement loop (refinement.py) with an LLM layer that handles the "other" error category, using Claude to interpret violations and generate fix operations, with iteration history and stagnation detection to ensure convergence.

Purpose: The existing refinement loop (Phase 10) handles pin_not_connected, wire_not_connected, and missing_power_symbol deterministically. The "other" category (anything not matching those patterns) currently has no auto-fix. This plan wires Claude into that gap: deterministic fixes run first (fast, free, reliable), then LLM fixes what deterministic code cannot.

Output: ErrorFixer class that converts violations to operations via Claude; llm_refine_design() function that combines deterministic and LLM fixes in a convergence-guaranteed loop; mocked tests.
</objective>

          ```
          def llm_refine_design(
              sch_path: Path,
              pcb_path: Path | None = None,
              max_iterations: int = 5,
              target_erc_clean: bool = True,
              target_drc_clean: bool = False,
              error_fixer: ErrorFixer | None = None,  # Inject for testing

          ```
          Algorithm:
          a. Enforce hard cap: max_iterations = min(max_iterations, 10).
          b. Create ErrorFixer if not injected.
          c. Track iteration_history: list[str] of what was tried each iteration.
          d. Track stagnation_counter: count consecutive iterations with same error count.
          e. For each iteration i in 1..max_iterations:
             i. Run ERC via run_erc(sch_path).
             ii. If ERC passed and targets met, return converged.
             iii. Analyze errors via analyze_erc_errors(erc_result) (imported from existing refinement.py).
             iv. Apply deterministic fixes FIRST (pin_not_connected -> place_no_connects, wire_not_connected -> wire_snapping). These are fast, free, reliable. Reuse _apply_place_no_connects and _apply_wire_snapping from existing refinement.py.

             vi. If ErrorFixer returned operations, execute them via OperationExecutor (try/except per operation, log failures).
             vii. Record iteration: error counts, fixes applied, LLM fixes count.
             viii. Append to iteration_history: f"Iteration {i}: {erc_errors} ERC errors, tried: {fixes}".
             ix. Stagnation check: if current error count equals previous error count, increment stagnation_counter. Else reset to 0. If stagnation_counter >= 3, stop with converged=False, stagnation_detected=True.
             x. Re-run ERC to check if fixes helped.
          f. Return LLMRefinementResult.
