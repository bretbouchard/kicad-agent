    1. Create `src/kicad_agent/llm/design_critic.py`:

       a. Define frozen dataclasses:
          - CritiqueSeverity: enum with INFO, WARNING, CRITICAL values.
          - CritiqueFinding: severity (CritiqueSeverity), category (str: "clearance", "congestion", "thermal", "placement"), description (str), coordinates (tuple of (x,y) tuples).
          - CritiqueReport: findings (tuple of CritiqueFinding), summary (str), overall_quality_score (float 0.0-1.0).

       b. Define build_spatial_context(engine: SpatialQueryEngine) -> str function:
          - Use engine.proximity(0, 0, radius_mm=10000) to get all entities.
          - Group entities by entity_type, count each group.
          - For "component" type entities (SpatialBox), include bounding box in format: "  {entity_id}: box({x1:.1f},{y1:.1f},{x2:.1f},{y2:.1f}) layer={layer} ref={reference}".
          - Cap component box output at 20 entries to manage context window.
          - Return multi-line string suitable for LLM consumption.

       c. Define CRITIC_SYSTEM_PROMPT: "You are a PCB design critic. Analyze the spatial layout data for clearance issues (components too close together), routing congestion (dense component clusters with many nets between them), and thermal hotspots (clustered power components). For each issue, provide: severity (info/warning/critical), category, description with specific coordinate references, and the coordinates of the affected area."

       d. Define CRITIC_TOOL: Claude tool definition with name="design_critique", input_schema containing:
          - findings: array of objects with severity (enum: info/warning/critical), category (string), description (string), coordinates (array of [x,y] number pairs)
          - summary: string
          - overall_quality_score: number (0.0-1.0)

       e. Define DesignCritic class:
          - __init__(model: str | None = None): Creates LLMClient.
          - critique(engine: SpatialQueryEngine, erc_result: ErcResult | None = None, drc_result: DrcResult | None = None) -> CritiqueReport:
            a. Build spatial context via build_spatial_context(engine).
            b. Build error context via ContextBuilder.build_error_summary(erc_result, drc_result) if violations provided.
            c. Assemble user message combining spatial context and optional error context.
            d. Call LLMClient.create_message() with:
               - system=CRITIC_SYSTEM_PROMPT
               - tools=[CRITIC_TOOL]
               - tool_choice={"type": "tool", "name": "design_critique"}
               - thinking={"type": "enabled", "budget_tokens": 8000}
               - max_tokens=16000 (extended thinking needs larger output budget)
               - messages=[{"role": "user", "content": assembled_context}]
            e. Extract tool_use block for "design_critique".
            f. Parse findings from block.input into list of CritiqueFinding.
            g. Compute overall_quality_score from finding severities: critical = -0.3, warning = -0.1, info = -0.02, clamped to [0.0, 1.0]. Start from 1.0 and subtract.
            h. Return CritiqueReport.

    2. Create `tests/test_llm_design_critic.py`:
       - Mock LLMClient.create_message to return tool_use block with sample findings.
       - Test build_spatial_context with mocked SpatialQueryEngine (empty, with components).
       - Test DesignCritic.critique returns CritiqueReport with parsed findings.
       - Test quality score computation from finding severities.
       - Test that extended thinking parameters are passed to LLMClient.

    3. Update `src/kicad_agent/llm/__init__.py` to add DesignCritic, CritiqueFinding, CritiqueReport to exports.
