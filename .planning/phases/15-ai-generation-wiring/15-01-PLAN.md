<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User NL input -> IntentParser | Untrusted natural language crosses into structured pipeline |
| Claude API response -> Pydantic validation | LLM output (untrusted) crosses into trusted code via validation gate |
| KiCad file content -> LLM prompt | File data (potentially user-controlled) enters LLM context |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Tampering | IntentParser | mitigate | Pydantic model_validate on all LLM output; rejects structurally valid but semantically wrong JSON |
| T-15-02 | Tampering | ComponentSuggester | mitigate | _validate_safe_id on all library_id values from LLM; rejects injection characters |
| T-15-03 | Information Disclosure | client.py | mitigate | API key from environment variable only; never logged or included in error messages |
| T-15-04 | Tampering | context_builder.py | mitigate | sanitize() strips instruction-like patterns from file content before LLM inclusion |
| T-15-05 | Denial of Service | IntentParser | accept | LLM call cost is user-borne; iteration caps in refinement loop (Plan 03) prevent runaway costs |
| T-15-06 | Tampering | tools.py | mitigate | Tool input_schema derived from Pydantic models with length constraints (500 components, 200 nets) |
</threat_model>
