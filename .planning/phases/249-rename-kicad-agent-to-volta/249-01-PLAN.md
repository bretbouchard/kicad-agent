# Phase 249: Project-wide rename kicad-agent → Volta

**Status:** Planned (2026-07-14)
**Bead:** kicad-agent-dbj
**Priority:** P1
**Effort:** LARGE (multi-day, ~16-32 hours)
**Plans:** TBD via /gsd-plan-phase 249 (likely 3-5 plans)

## Goal

Rename the project from "kicad-agent" to "Volta" across the entire codebase, documentation, packaging, app naming, and external references. Make "Volta" the single canonical name. Preserve all 6300+ tests, all parity reports, all eval reports.

## Why

User request (2026-07-14): "we also need to find the use of 'kicad agent' and change it to 'Volta'". The product is internally called Volta (Phase 245, app sandbox dir is `~/Library/Application Support/VoltaPCB/`), but the repo, package, and code identifiers still use `kicad-agent` (a holdover from when the product was KiCad-only). The mismatch is confusing — the eval harness Phase 246 exposes it clearly.

## Discovery-First Strategy

The user said "we also need to find the use of 'kicad agent' and change it to 'Volta'". This means:

**Plan 1 — Discovery & Impact Assessment:**
- Comprehensive grep for all `kicad-agent`, `kicad_agent`, `KicadAgent`, `KiCad Agent`, `kicad agent` (case-insensitive)
- Categorize by file type (code, doc, config, generated, fixture)
- Build a complete rename map
- Estimate file count, LOC impact
- Identify which renames are SAFE (sed) vs NEED REVIEW (semantic, brand choice)

**Plan 2 — Brand Decision:**
- "Volta" vs "VoltaPCB" vs "Volta KiCad" — pick the canonical display name
- Python package: `volta` vs `volta_pcb` vs `voltakicad`?
- CLI: `volta` vs `voltapcb` vs `vk`?
- macOS app: `Volta.app` vs `VoltaPCB.app`?
- App sandbox dir: `Volta/` vs `VoltaPCB/`
- HuggingFace: keep `volta-pcb-adapter-v2` or rename

**Plan 3 — Code rename (Python):**
- Rename `src/kicad_agent/` → `src/volta/` (or chosen name)

- Update `pyproject.toml` / `setup.py`
- Update CLI entry points

- Update `benchmarks/`, `scripts/`, `training/`, `sweeps/`

**Plan 4 — Code rename (Swift):**

**Plan 5 — macOS app rebrand:**

**Plan 6 — Documentation sweep:**
