# Plan 205-01: Board Metadata Foundation

## Goal

Read and write KiCad `title_block` metadata (title, date, rev, company, comments) from `.kicad_pcb` files, and persist manufacturing specs (surface finish, copper weight, mask/silk color, impedance) in a sidecar JSON file. This is the foundation for Phase 207 (Versioned Builds), which depends on the `title_block` rev field for `board_rev`.

## Threat Model

<threat_model>
**Attack Surface: Sidecar JSON writes (`.kicad_build_spec.json`)**

**Adversarial Scenario 1: Path traversal via crafted PCB filename.**
A malicious actor provides a PCB file path containing traversal sequences (e.g., `../../etc/cron.d/evil.kicad_pcb`). The `save_board_spec` function computes the sidecar path via `pcb_path.with_suffix(".kicad_build_spec.json")`. If the PCB path itself is attacker-controlled and not validated upstream, the sidecar could be written outside the project directory.

**Mitigation:** The sidecar path is derived from `pcb_path` using `Path.with_suffix()`, which only replaces the final suffix — it cannot inject new path components. The `TargetFile` validator in `schema.py` (lines 153-166) validates file paths before handlers execute. The `atomic_write` function writes to the sidecar's parent directory (`file_path.parent`), which is the same directory as the PCB file — no traversal is possible from a suffix replacement. Additionally, execution-layer path confinement (the `base_dir` check referenced in CONTEXT canonical_refs `test_pcb_ops.py:266-282`) rejects absolute paths outside the working directory. **Severity: Low.** No additional code needed — existing path validation suffices.

**Adversarial Scenario 2: Raw S-expression corruption via crafted title_block content.**
A malicious PCB file contains a title_block with unbalanced parentheses or malformed content designed to corrupt the file when the raw writer modifies it.

**Mitigation:** The raw writer's `set_title_block_fields` uses `_find_matching_close` (pcb_raw_writer.py:938-975) which correctly handles nested parens and KiCad's doubled-quote escaping convention. The method reads the existing title_block to extract current values (partial update support), then rebuilds the entire block from scratch with proper quoting. Since the output is built from typed Python strings (not string interpolation into existing content), there is no injection vector. Field values are always double-quoted in the output, and embedded double-quotes are doubled per KiCad convention. **Severity: Low.** The block-level rebuild strategy eliminates partial-update corruption.

**Adversarial Scenario 3: Sidecar JSON overwrite of unintended files.**
A user runs `save_board_spec` where a `.kicad_build_spec.json` already exists from a different purpose.

**Mitigation:** `atomic_write` uses `os.replace` which is atomic and overwrites the destination. This is the intended behavior — the sidecar is owned by volta. The `.kicad_build_spec.json` extension is volta-specific and unlikely to collide with other tools. **Severity: Informational.**

**ASVS L1 controls verified:**
- A1 (Input validation): Field values in BoardSpec are validated by pydantic (enums reject unknown values; Field constraints on lengths). Title block field values from raw content are extracted via sexpdata parsing (structured, not regex on user input).
- A5 (Secure file operations): `atomic_write` uses tempfile + `os.replace` — crash-safe, no torn writes.
- Read-only query path does NOT mutate files: `execute_query` dispatches via `_QUERY_HANDLERS` with no Transaction, no serialization, no file write.
</threat_model>

## Mutation Path Decision (CRITICAL — from RESEARCH RQ2)

The PCB serializer (`serialize_pcb` in `pcb_ser.py:64-66`) uses `kiutils_obj.to_file()` and does NOT emit `NativeBoard` fields. Native-path mutation (`replace + _record_mutation`) will NOT work because the serializer ignores the typed `title_block` field.

**All title_block mutations use the RAW WRITER path:**
1. Handler calls `PcbRawWriter.set_title_block_fields(ir.raw_content, ...)` → returns modified content string
2. Handler calls `ir.commit_raw_content(new_content)` → writes to disk via `atomic_write`, sets `raw_written = True`
3. Executor sees `raw_written = True` and SKIPS `serialize_pcb` (execution.py:533-534)

This matches the `move_footprint` handler pattern (handlers/pcb.py:225-245).

## Query Path Decision (CRITICAL — from RESEARCH RQ1)

`execute_query` (execution.py:193-230) builds PcbIR via the kiutils path (`parse_pcb`), NOT the native parser. So `ir.board` is a kiutils `Board` object — `ir.board.title_block` does NOT exist even after we add the native parser field.

**The `read_board_metadata` handler parses title_block from `ir.raw_content` directly** using `sexpdata.loads` + the native parser's helper functions (`_find_symbol`, `_find_string_child`). This does NOT require a NativeBoard.

```python
@classmethod
def _extract_title_block(cls, root: list) -> NativeTitleBlock | None:
    """Extract (title_block ...) metadata. Returns None if absent."""
    tb_block = _find_symbol(root, "title_block")
    if tb_block is None:
        return None
    title = _find_string_child(tb_block, "title")
    date = _find_string_child(tb_block, "date")
    rev = _find_string_child(tb_block, "rev")
    company = _find_string_child(tb_block, "company")
    # Numbered comments: (comment N "text") where N is 1-9, non-sequential
    comments_map: dict[int, str] = {}
    for item in tb_block:
        if isinstance(item, list) and len(item) >= 3 and _sym(item[0]) == "comment":
            try:
                num = int(item[1])
                text = item[2] if isinstance(item[2], str) else str(item[2])
                comments_map[num] = text
            except (ValueError, TypeError):
                continue
    # Convert to tuple (index 0 = comment 1, gaps become empty strings)
    if comments_map:
        max_n = max(comments_map)
        comments = tuple(comments_map.get(i, "") for i in range(1, max_n + 1))
    else:
        comments = ()
    return NativeTitleBlock(title=title, date=date, rev=rev, company=company, comments=comments)
```

```python
"""BoardSpec — manufacturing specification model with sidecar JSON persistence.

Phase 205: META-04, META-05. Persists surface finish, copper weight, soldermask/
silkscreen color, and controlled impedance requirements as a sidecar JSON file
(.kicad_build_spec.json) alongside the .kicad_pcb file.
"""
from __future__ import annotations

from enum import Enum
from pathlib import Path

from pydantic import BaseModel, Field

from volta.io.atomic_write import atomic_write


class SurfaceFinish(str, Enum):
    """PCB surface finish options."""
    HASL = "HASL"
    ENIG = "ENIG"
    HASL_LEAD_FREE = "HASL_LEAD_FREE"
    HARD_GOLD = "HARD_GOLD"
    OSP = "OSP"
    ENEPIG = "ENEPIG"


class SoldermaskColor(str, Enum):
    """Soldermask color options (matches KiCad color names)."""
    GREEN = "GREEN"
    RED = "RED"
    BLUE = "BLUE"
    BLACK = "BLACK"
    WHITE = "WHITE"
    YELLOW = "YELLOW"
    PURPLE = "PURPLE"
    MATTE_BLACK = "MATTE_BLACK"


class SilkscreenColor(str, Enum):
    """Silkscreen color options."""
    WHITE = "WHITE"
    BLACK = "BLACK"


class ImpedanceRequirement(BaseModel):
    """Controlled impedance requirement for a specific net (META-05)."""
    net_name: str = Field(min_length=1, max_length=128, description="Net with impedance requirement")
    target_ohms: float = Field(gt=0, description="Target impedance in ohms (50, 75, 90, 100, 120 common)")
    reference_layer: str = Field(min_length=1, max_length=64, description="Reference layer name (e.g., 'GND', 'L02')")


class BoardSpec(BaseModel):
    """Manufacturing specification for a PCB board (META-04, META-05).

    Persisted as a sidecar JSON file (.kicad_build_spec.json) alongside the
    .kicad_pcb file. NOT a KiCad file — this is volta's own extension.
    """
    schema_version: int = 1
    surface_finish: SurfaceFinish = SurfaceFinish.HASL
    copper_weight_outer_oz: float = Field(default=1.0, gt=0, description="Outer layer copper weight in oz")
    copper_weight_inner_oz: float = Field(default=0.5, gt=0, description="Inner layer copper weight in oz")
    soldermask_color: SoldermaskColor = SoldermaskColor.GREEN
    silkscreen_color: SilkscreenColor = SilkscreenColor.WHITE
    impedance_requirements: tuple[ImpedanceRequirement, ...] = ()


def load_board_spec(pcb_path: Path) -> BoardSpec | None:
    """Load BoardSpec from sidecar JSON. Returns None if sidecar absent."""
    sidecar = pcb_path.with_suffix(".kicad_build_spec.json")
    if not sidecar.is_file():
        return None
    return BoardSpec.model_validate_json(sidecar.read_text(encoding="utf-8"))


def save_board_spec(pcb_path: Path, spec: BoardSpec) -> Path:
    """Save BoardSpec to sidecar JSON atomically. Returns sidecar path.

    For board.kicad_pcb -> board.kicad_build_spec.json.
    Uses atomic_write (tempfile + os.replace) for crash-safe writes.
    """
    sidecar = pcb_path.with_suffix(".kicad_build_spec.json")
    atomic_write(sidecar, spec.model_dump_json(indent=2))
    return sidecar
```

### 3a. Add `set_title_block_fields` static method to `PcbRawWriter` in `src/volta/ops/pcb_raw_writer.py`

This is a BLOCK-LEVEL rebuild strategy (per PATTERNS file #12c recommendation): read existing values for partial update, build the new `(title_block ...)` S-expression from scratch, replace the existing block or insert a new one if absent.

Add the method to the `PcbRawWriter` class. Use `_find_matching_close` for block boundary detection. Use regex `re.search(r"^\s*\(title_block\b", content, re.MULTILINE)` to locate the block.

```python
@staticmethod
def set_title_block_fields(
    content: str,
    title: Optional[str] = None,
    date: Optional[str] = None,
    rev: Optional[str] = None,
    company: Optional[str] = None,
    comments: Optional[list[str]] = None,
) -> str:
    """Set title_block fields in PCB content. None fields are left unchanged.

    Reads existing title_block values first (to support partial update),
    then rebuilds the entire block. If no title_block exists, inserts one
    after the (paper ...) line (title_block comes after paper, before layers).

    Uses block-level replacement to avoid partial-update edge cases with
    non-sequential comments.
    """
    import sexpdata
    from volta.parser.pcb_native_parser import _find_symbol, _find_string_child, _sym

    # Read existing title_block values for partial update
    existing_title = ""
    existing_date = ""
    existing_rev = ""
    existing_company = ""
    existing_comments: list[str] = []
    try:
        tree = sexpdata.loads(content)
        tb = _find_symbol(tree, "title_block")
        if tb is not None:
            existing_title = _find_string_child(tb, "title")
            existing_date = _find_string_child(tb, "date")
            existing_rev = _find_string_child(tb, "rev")
            existing_company = _find_string_child(tb, "company")
            comments_map: dict[int, str] = {}
            for item in tb:
                if isinstance(item, list) and len(item) >= 3 and _sym(item[0]) == "comment":
                    try:
                        num = int(item[1])
                        text = item[2] if isinstance(item[2], str) else str(item[2])
                        comments_map[num] = text
                    except (ValueError, TypeError):
                        continue
            if comments_map:
                max_n = max(comments_map)
                existing_comments = [comments_map.get(i, "") for i in range(1, max_n + 1)]
    except Exception:
        pass  # If parsing fails, use empty defaults

    # Apply partial updates (None = keep existing)
    new_title = existing_title if title is None else title
    new_date = existing_date if date is None else date
    new_rev = existing_rev if rev is None else rev
    new_company = existing_company if company is None else company
    new_comments = existing_comments if comments is None else comments

    # Build new title_block S-expression
    def _escape_kicad_string(s: str) -> str:
        """Escape a string for KiCad S-expression: double internal quotes."""
        return s.replace('"', '""')

    lines = ["  (title_block"]
    lines.append(f'    (title "{_escape_kicad_string(new_title)}")')
    lines.append(f'    (date "{_escape_kicad_string(new_date)}")')
    lines.append(f'    (rev "{_escape_kicad_string(new_rev)}")')
    lines.append(f'    (company "{_escape_kicad_string(new_company)}")')
    for i, c in enumerate(new_comments):
        lines.append(f'    (comment {i + 1} "{_escape_kicad_string(c)}")')
    lines.append("  )")
    new_block = "\n".join(lines)

    # Find existing title_block and replace it, or insert new one
    match = re.search(r"^[ \t]*\(title_block\b", content, re.MULTILINE)
    if match:
        start = match.start()
        end = PcbRawWriter._find_matching_close(content, start + 1)
        if end is not None:
            return content[:start] + new_block + content[end + 1:]

    # No existing title_block — insert after (paper ...) line
    paper_match = re.search(r"^[ \t]*\(paper[^\)]*\)[^\n]*\n", content, re.MULTILINE)
    if paper_match:
        insert_pos = paper_match.end()
        return content[:insert_pos] + new_block + "\n" + content[insert_pos:]

    # No (paper ...) either — insert after the first (kicad_pcb ...) line
    kicad_match = re.search(r"^[ \t]*\(kicad_pcb[^\n]*\n", content, re.MULTILINE)
    if kicad_match:
        insert_pos = kicad_match.end()
```

```python
@register_query("read_board_metadata")
def _handle_read_board_metadata(op: Any, ir: PcbIR, file_path: Path) -> dict[str, Any]:
    """Read title_block fields + board_spec sidecar.

    CRITICAL: execute_query uses kiutils path, NOT native parser.
    Parse title_block from ir.raw_content using native parser helpers directly.
    """
    import sexpdata
    from volta.parser.pcb_native_parser import _find_symbol, _find_string_child, _sym

    tree = sexpdata.loads(ir.raw_content)
    tb_block = _find_symbol(tree, "title_block")

    if tb_block is not None:
        title = _find_string_child(tb_block, "title")
        date = _find_string_child(tb_block, "date")
        rev = _find_string_child(tb_block, "rev")
        company = _find_string_child(tb_block, "company")
        comments: list[str] = []
        comments_map: dict[int, str] = {}
        for item in tb_block:
            if isinstance(item, list) and len(item) >= 3 and _sym(item[0]) == "comment":
                try:
                    num = int(item[1])
                    text = item[2] if isinstance(item[2], str) else str(item[2])
                    comments_map[num] = text
                except (ValueError, TypeError):
                    continue
        if comments_map:
            max_n = max(comments_map)
            comments = [comments_map.get(i, "") for i in range(1, max_n + 1)]
    else:
        title = ""
        date = ""
        rev = ""
        company = ""
        comments = []

    # Load board_spec sidecar if present
    from volta.manufacturing.board_spec import load_board_spec
    spec = load_board_spec(file_path)

    return {
        "title": title,
        "date": date,
        "rev": rev,
        "company": company,
        "comments": comments,
        "board_spec": spec.model_dump() if spec else None,
    }
```
