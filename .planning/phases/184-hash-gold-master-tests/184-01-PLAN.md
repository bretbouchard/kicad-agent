## Adversarial Scenario

**Name:** "Fixture poisoning attack via malicious gold master hashes"

**Scenario:** Attacker with git access poisons gold master fixtures:
- Modifies fixture JSON to include malicious expected hashes (matches tampered circuit)
- Tampered circuit has backdoor (hidden net connecting to external pad)
- Gold master test passes (matches tampered hashes)
- Tampered circuit committed to main, undetected by CI

**Mitigation:**
1. Fixture metadata includes git commit SHA of trusted pipeline version
2. Verify fixture integrity: `git log --oneline tests/generative/fixtures/` shows no tampering
3. Pre-commit hook: Run fixture generation workflow, compare to committed hashes
4. Code review required for fixture changes (PR must modify fixture + tests together)

Per GENOUT-04: Hash-based gold master tests catch drift. If fixture is poisoned, test passes but CI should fail due to fixture integrity check.
</threat_model>
