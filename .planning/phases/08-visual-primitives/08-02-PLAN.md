    **1. Create src/kicad_agent/spatial/maze_generator.py** (VP-04):

    - `MazeBoard` frozen dataclass: Fields `pcb_path: Path`, `board_width_mm: float`, `board_height_mm: float`, `obstacles: tuple[SpatialBox, ...]`, `source_point: SpatialPoint`, `target_point: SpatialPoint`, `solution_path: tuple[tuple[float, float], ...]`, `clearance_mm: float`. This represents a generated maze with known solution.

    - `generate_maze_board(output_path: Path, width_mm: float = 50.0, height_mm: float = 50.0, grid_size_mm: float = 5.0, clearance_mm: float = 0.5, seed: int | None = None) -> MazeBoard`:
      1. Set random seed if provided (`random.seed(seed)` for deterministic generation)
      2. Create a minimal kiutils Board object:
         ```python
         from kiutils.board import Board
         board = Board()  # Empty board with default general settings
         board.generalSettings.thickness = 1.6
         ```
      3. Add board outline (Edge.Cuts) as a rectangle: Create line segments on Edge.Cuts layer forming a rectangle (0,0) to (width_mm, height_mm). Use `kiutils.items.brditems` graphic line items.
      4. Generate obstacle grid: Divide board into grid cells of `grid_size_mm`. Randomly mark ~40% of cells as obstacles (using `random.random()`). Create footprint rectangles for obstacles on F.Cu layer:
         ```python
         from kiutils.items.brditems import Footprint
         from kiutils.items.common import Position, Net
         ```
         Each obstacle is a simple footprint with a single copper rectangle pad covering the cell area. Place at cell center coordinates. Use `import uuid; uuid.uuid4()` for timestamps.
      5. Add source via at a clear cell near (grid_size_mm * 1.5, grid_size_mm * 1.5) on Net 1
      6. Add target via at a clear cell near (width_mm - grid_size_mm * 1.5, height_mm - grid_size_mm * 1.5) on Net 2
      7. Compute solution path using BFS through the grid (avoiding obstacle cells). The solution path is a list of (x, y) center coordinates of clear cells from source to target.
      8. Serialize board to output_path using `board.write(str(output_path))`
      9. Return MazeBoard with all metadata

    - `_create_board_outline(board: Board, width: float, height: float) -> None`:
      Add four graphic line segments on Edge.Cuts layer forming a rectangle. Each line from corner to corner using `kiutils.items.brditems` graphic items.

    - `_add_obstacle_footprint(board: Board, cx: float, cy: float, size: float, index: int) -> None`:
      Add a footprint at (cx, cy) with a single rectangular pad covering the cell. The footprint represents an obstacle zone. Use `Footprint()` constructor with appropriate pad definition.

    - `_add_via(board: Board, x: float, y: float, net_number: int, net_name: str) -> None`:
      Add a via at (x, y) connecting F.Cu and B.Cu layers on the specified net. Ensure the net exists in board.nets.

    - `_solve_maze(grid: list[list[bool]], start: tuple[int,int], end: tuple[int,int]) -> list[tuple[int,int]]`:
      BFS pathfinder through the boolean grid. Returns list of (row, col) cells from start to end. Returns empty list if no path exists (regenerate in that case).

    Round-trip validation: After generating and serializing, parse the file back and verify it produces a valid Board:
    ```python
    from kiutils.board import Board
    verified = Board.from_file(str(output_path))
    assert len(verified.footprints) > 0 or len(verified.traceItems) > 0
    ```

    **1. Create src/kicad_agent/spatial/reasoning_chains.py** (VP-05):

    - `ReasoningStep` frozen dataclass: Fields `step_type: str` (one of "observation", "spatial_context", "coordinate_reference", "diagnosis", "recommendation"), `content: str`, `coordinates: tuple[tuple[float, float], ...] = ()` (optional coordinate data for this step), `metadata: dict = {}`.

    - `ReasoningChain` frozen dataclass: Fields `violation_type: str`, `violation_description: str`, `severity: str`, `steps: tuple[ReasoningStep, ...]`, `spatial_primitives: tuple[dict, ...]` (JSON-serialized primitives involved), `chain_id: str = ""`.

    - `synthesize_chain(violation: Violation, pcb_primitives: list | None = None) -> ReasoningChain`:
      Build a reasoning chain from a single DRC/ERC violation:
      1. Extract coordinates from violation items: Each item dict may have `pos.x` and `pos.y` keys (from kicad-cli JSON report)
      2. Step 1 (observation): "DRC violation detected: {violation.description}" with coordinates from violation items
      3. Step 2 (spatial_context): Describe what is near the violation. If pcb_primitives provided, find primitives within 5mm of violation coordinates using Shapely distance queries. Format: "Near the violation point <point> [{x}, {y}], found {count} footprints, {count} traces within 5mm"
      4. Step 3 (coordinate_reference): Create precise coordinate reference: "The violation involves entities at: {list of coordinates as <point> [x, y] format}"
      5. Step 4 (diagnosis): "Violation type '{violation.type}' at coordinates [{coordinates}] indicates {diagnosis_text}" where diagnosis_text is a rule-based mapping from violation type to description:
         - "clearance" -> "insufficient spacing between copper features"
         - "width" -> "trace width below minimum design rule"
         - "unconnected_items" -> "net with dangling connections"
         - "silk_overlap" -> "silkscreen overlapping solder mask or pad"
         - default -> "design rule constraint not met"
      6. Step 5 (recommendation): "Consider: {recommendation}" where recommendation is derived from violation type:
         - "clearance" -> "Increase spacing between the violating features to at least the minimum clearance distance"
         - "width" -> "Widen the trace to meet the minimum width requirement for the net class"
         - "unconnected_items" -> "Route the missing connections or add no-connect flags if intentional"
         - default -> "Review the design rules and adjust the affected features"
      Return ReasoningChain with all steps.

    - `synthesize_chains(drc_result: DrcResult | None = None, erc_result: ErcResult | None = None, pcb_primitives: list | None = None) -> list[ReasoningChain]`:
      Process all violations from a DRC or ERC result. Return list of ReasoningChain objects. If both provided, process DRC first then ERC. Empty violations returns empty list.

    - `_extract_violation_coordinates(violation: Violation) -> list[tuple[float, float]]`:
      Extract (x, y) coordinates from violation.items. Each item is a dict with potential "pos" key containing {"x": float, "y": float}. Return list of coordinate tuples. Items without pos data get (0.0, 0.0).

    - `_find_nearby_primitives(coords: tuple[float, float], primitives: list, radius_mm: float = 5.0) -> list[dict]`:
      Given a coordinate and a list of spatial primitives, find all within radius_mm using Shapely Point buffer + intersection. Return list of primitive.to_json() dicts.
