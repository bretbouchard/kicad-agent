    **2. Create src/kicad_agent/spatial/primitives.py** with four frozen dataclasses (VP-03):

    - `SpatialPoint(frozen=True)`: Fields `x: float`, `y: float`, `entity_type: str` (one of "pin", "via", "pad", "vertex", "drc_item"), `entity_id: str`, `layer: str = ""`, `net: str = ""`. Methods: `to_json() -> dict` (keys: type, x, y, entity_type, entity_id, layer, net -- coordinates rounded to 4 decimal places), `to_shapely() -> shapely.Point`.

    - `SpatialBox(frozen=True)`: Fields `x1: float` (min X), `y1: float` (min Y), `x2: float` (max X), `y2: float` (max Y), `entity_type: str` (one of "footprint", "component", "pad"), `entity_id: str`, `layer: str = ""`, `reference: str = ""`. Methods: `to_json() -> dict` (keys: type, x1, y1, x2, y2, entity_type, entity_id, layer, reference), `to_shapely() -> shapely.box(x1, y1, x2, y2)`.

    - `SpatialPath(frozen=True)`: Fields `points: tuple[tuple[float, float], ...]`, `entity_type: str` (one of "segment", "arc", "wire"), `entity_id: str`, `layer: str = ""`, `net: str = ""`, `width: float = 0.0`. Methods: `to_json() -> dict` (keys: type, points as list of [x,y] pairs, entity_type, entity_id, layer, net, width), `to_shapely() -> shapely.LineString` from points.

    - `SpatialRegion(frozen=True)`: Fields `boundary: tuple[tuple[float, float], ...]` (polygon vertices), `entity_type: str` (one of "zone", "keepout", "copper_pour", "net_class_region"), `entity_id: str`, `layer: str = ""`, `net: str = ""`, `region_type: str = ""`. Methods: `to_json() -> dict` (keys: type, boundary as list of [x,y], entity_type, entity_id, layer, net, region_type), `to_shapely() -> shapely.Polygon` from boundary.

    All dataclasses use `from __future__ import annotations` for forward refs. Each `to_json()` returns a plain dict (not JSON string). Import shapely lazily in `to_shapely()` methods to avoid import-time failures if shapely is missing.

    **3. Create src/kicad_agent/spatial/extractor.py** with extraction functions (VP-02):

    - `extract_points(pcb_ir: PcbIR) -> list[SpatialPoint]`: Iterate `pcb_ir.trace_items`, check `hasattr(item, 'position')` for vias (items with position but no start/end). For each footprint pad: compute ABSOLUTE position by adding footprint.position.X + pad.position.X and footprint.position.Y + pad.position.Y (for 0-degree footprints; for rotated footprints, apply rotation matrix using `math.cos`/`math.sin` on `math.radians(fp.position.angle)`). Entity_id for vias uses `str(item.tstamp)`, for pads uses `f"{fp_ref}.{pad.number}"`.

    - `extract_boxes(pcb_ir: PcbIR) -> list[SpatialBox]`: For each footprint in `pcb_ir.footprints`, compute bounding box from pad positions. Get all pad local positions, find min/max X and Y, add 1.0mm margin, then translate to absolute coordinates (footprint.position + rotated offset). Entity_id uses `fp.libId`, reference uses `fp.properties.get("Reference", "")`.

    - `extract_paths(pcb_ir: PcbIR) -> list[SpatialPath]`: For each item in `pcb_ir.trace_items` that has `start` and `end` attributes (segments and arcs). For segments: `points = ((start.X, start.Y), (end.X, end.Y))`. For arcs (have `mid` attribute): `points = ((start.X, start.Y), (mid.X, mid.Y), (end.X, end.Y))`. Entity_type is "arc" if hasattr(item, 'mid'), else "segment". Entity_id uses `str(item.tstamp)`.

    - `extract_regions(pcb_ir: PcbIR) -> list[SpatialRegion]`: Iterate `pcb_ir.board.zones`. Extract polygon vertices from `zone.polygons` (first polygon's outline). Boundary as tuple of (x,y) tuples. Entity_id uses `str(zone.tstamp)` if available, else index-based. Entity_type is "zone".

    - `extract_all(pcb_ir: PcbIR) -> dict[str, list]`: Returns `{"points": extract_points(pcb_ir), "boxes": extract_boxes(pcb_ir), "paths": extract_paths(pcb_ir), "regions": extract_regions(pcb_ir)}`.

    CRITICAL: Pad positions are LOCAL to footprint origin. Must compute absolute position:
    ```python
    import math
    angle_rad = math.radians(fp.position.angle)
    cos_a, sin_a = math.cos(angle_rad), math.sin(angle_rad)
    abs_x = fp.position.X + (pad.position.X * cos_a - pad.position.Y * sin_a)
    abs_y = fp.position.Y + (pad.position.X * sin_a + pad.position.Y * cos_a)
    ```

    **1. Create src/kicad_agent/spatial/renderer.py** (VP-01):

    Two rendering approaches, with kicad-cli SVG export as primary and Pillow-only fallback:

    - `render_pcb_layer(pcb_path: Path, layer: str = "F.Cu", output_path: Path | None = None, dpi: int = 300, theme: str = "default") -> dict`:
      1. Find kicad-cli via `shutil.which("kicad-cli")` (same pattern as erc_drc.py `_find_kicad_cli`)
      2. Create temp directory via `tempfile.mkdtemp(prefix="kicad-agent-render-")`
      3. Run `kicad-cli pcb export svg --layers {layer} --exclude-drawing-sheet --output {tmpdir} {pcb_path}` with 60s timeout
      4. Find the generated SVG file in tmpdir (pattern: `{layer.replace('.', '_')}.svg` or glob for `*.svg`)
      5. If SVG exists, rasterize to PNG using cairocffi:
         ```python
         import cairocffi
         surface = cairocffi.SVGSurface(str(svg_path))
         # Or: parse SVG with xml.etree.ElementTree, extract viewBox,
         # render at target DPI using cairocffi.ImageSurface
         ```
      6. If cairocffi fails or SVG not found, use Pillow fallback:
         - Parse PCB with parse_pcb, create PcbIR, extract primitives via extractor
         - Render primitives as simple shapes on a white image (footprints as rectangles, traces as lines, vias as circles)
      7. Add coordinate grid overlay using Pillow ImageDraw:
         - Determine board bounds from PCB general settings or from extracted primitives
         - Draw grid lines every N mm (configurable, default 10mm)
         - Label grid lines with mm coordinates (using ImageDraw.text)
         - Use light gray color for grid lines, dark text for labels
      8. Save to output_path (default: temp file with .png extension)
      9. Clean up temp directory in finally block
      10. Return dict: `{"image_path": str, "width_px": int, "height_px": int, "board_width_mm": float, "board_height_mm": float, "mm_per_pixel": float, "layer": layer}`

    - `render_pcb_layer_grid(pcb_path: Path, layers: list[str] | None = None, output_dir: Path | None = None, dpi: int = 300) -> list[dict]`:
      Render multiple layers. Default layers = ["F.Cu", "B.Cu", "F.SilkS", "Edge.Cuts"]. Returns list of render results.

    - `_get_board_bounds(pcb_ir: PcbIR) -> tuple[float, float, float, float]`:
      Extract board bounding box from Edge.Cuts graphics or from the union of all footprint bounding boxes. Returns (min_x, min_y, max_x, max_y).

    - `_add_coordinate_grid(image: PIL.Image.Image, board_bounds: tuple, grid_spacing_mm: float = 10.0) -> PIL.Image.Image`:
      Draw grid lines and labels on the image. Compute pixel-to-mm mapping from image dimensions and board bounds. Use `PIL.ImageDraw.Draw` for lines and text.

    Error handling:
    - If kicad-cli not found: raise FileNotFoundError with same message as erc_drc.py
    - If SVG export fails: log warning, fall back to Pillow-only primitive rendering
    - Timeout: 60s for kicad-cli subprocess

    Security:
    - Validate pcb_path suffix is .kicad_pcb
    - Validate output_path does not contain path traversal
    - Use os.chmod on output files (consistent with erc_drc.py pattern)
