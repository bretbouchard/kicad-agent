# Phase 250: Portable build setup (adapter resolution + first-run script)

**Status:** Planned (2026-07-14)
**Priority:** P1
**Effort:** Small-medium (~4-8 hours)
**Plans:** TBD via /gsd-plan-phase 250

## Goal

Make `kicad-agent` buildable on any machine (Bret's Mac mini, a fresh CI runner, a teammate's laptop) — not just the machine that has `/Volumes/Storage` mounted with the canonical adapter at a specific path. Add a `scripts/setup_local.py` (or shell equivalent) that detects, resolves, or downloads the canonical Volta v2 adapter, then creates the symlink + verifies the build environment.

## Why

User flagged 2026-07-14: "we don't want this to just work on my computer we need this to build properly". The `volta-12b-v2` symlink is currently hard-coded to `/Volumes/Storage/models/kicad-agent/adapters/kicad-vision-v2`. This breaks on:
- Fresh CI runners (no /Volumes/Storage)
- Teammate machines (different mount point or no Storage at all)
- Bret's laptop vs Mac mini (different paths)
- Anyone running `git clone` + `make test`

## Problem Statement

Three portability issues identified during Phase 246/248 work:

1. **Adapter path is hard-coded** — `kicad-agent/volta-12b-v2` symlink points to an absolute Storage path
2. **Adapter name divergence** — Storage has `kicad-vision-v2` (legacy name), app expects `volta-pcb-adapter-v2` (HF name)
3. **Build env is implicit** — No first-run script checks Python version, kicad-cli presence, base model availability

## Requirements

- **REQ-250-01:** `scripts/setup_local.py` creates the `volta-12b-v2` symlink at repo root, pointing to whichever local copy of the adapter is resolvable
- **REQ-250-02:** Setup script searches known locations in order: `/Volumes/Storage/models/kicad-agent/adapters/kicad-vision-v2` → `~/.cache/huggingface/hub/models--bretbouchard--volta-pcb-adapter-v2/` → local fallback
- **REQ-250-03:** If no local copy found, setup script offers to download from `bretbouchard/volta-pcb-adapter-v2` on HuggingFace
- **REQ-250-04:** Setup script verifies Python 3.11+, kicad-cli 10.0.1, base model `gemma-4-12b-it` availability
- **REQ-250-05:** `.gitignore` excludes the machine-local symlink (`/volta-12b-v2`) — added 2026-07-14
- **REQ-250-06:** Setup script is idempotent
- **REQ-250-07:** Setup script exits non-zero with clear error message if prerequisites missing
- **REQ-250-08:** CLAUDE.md `## Tool Inventory` section documents `scripts/setup_local.py` as the first-run command
- **REQ-250-09:** `bootstrap-project` skill integration — `/bootstrap-project` calls `setup_local.py` as part of its workflow

## Key Work

### 1. Setup script (`scripts/setup_local.py`)

```python
# Pseudocode (TDD-first)
def main():
    # 1. Check Python version (>= 3.11)
    # 2. Search for canonical adapter in known locations
    adapter = find_local_adapter()  # Storage → HF cache → local
    if not adapter:
        if user_consents_to_download():
            adapter = download_from_hf()  # bretbouchard/volta-pcb-adapter-v2
    if not adapter:
        sys.exit("ERROR: No adapter found and download declined")
    # 3. Create / verify symlink at <repo>/volta-12b-v2
    create_symlink(target=adapter, linkpath=Path.cwd() / "volta-12b-v2")
    # 4. Verify kicad-cli 10.0.1
    check_kicad_cli()
    # 5. Verify base model (gemma-4-12b-it) available
    check_base_model()
    # 6. Print summary
    print("Build environment ready. Adapter:", adapter)
```

1. Fresh `git clone` on a new machine → `python3 scripts/setup_local.py` → symlink created, build environment ready
