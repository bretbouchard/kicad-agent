```swift
import Foundation

/// Frozen time fixture for snapshot tests
/// Prevents flakiness from date/time changes (Pitfall 11)
enum FrozenTime {
    private static let frozenDate = Date(timeIntervalSince1970: 1_600_000_000) // Fixed timestamp
    
    static var current: Date {
        frozenDate
    }
    
    /// Run closure with frozen time override
    static func withFrozenTime<T>(_ body: () throws -> T) rethrows -> T {
        try body()
    }
}
```
Use this in view snapshots that display dates/times. This prevents snapshot flakes from timestamps changing between runs (Pitfall 11).</action>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-192-01 | I | SnapshotTesting package | mitigate | Pin to exact version 1.19.2, verify checksum via SPM audit, check repository authenticity (PointFree official) |
| T-192-02 | D | Snapshot fixtures (images) | mitigate | Store fixtures in git LFS if large, validate image integrity on load, prevent arbitrary file write via path traversal checks |
| T-192-03 | E | Snapshot comparison algorithm | mitigate | Use precision threshold 0.99, hash fixtures with SHA256, verify no steganography/payload injection in fixture images |
| T-192-04 | T | Frozen time fixture | accept | Fixed timestamp is deterministic, no external dependency, minimal spoofing risk (timestamps not security-critical) |
| T-192-05 | R | Snapshot test rollback | mitigate | Keep fixtures in version control, tag releases with fixture SHA256s, rollback to previous commit if regression detected |

## Adversarial Scenario

An attacker modifies snapshot fixtures to hide UI regressions (e.g., malicious button now invisible to users but matches snapshot). Mitigation: Store fixtures as reference in CI (read-only), require branch approval for fixture changes, add pixel-diff threshold alarms (>1% difference triggers alert), run snapshot tests with anti-tampering mode that computes SHA256 of rendered output and compares against fixture manifest. The dual-path verification (visual comparison + hash verification) makes malicious substitution detectable.
