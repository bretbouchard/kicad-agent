<objective>
Batch operation mode that parses once, validates all, applies all mutations, and writes once per file.

Purpose: When an LLM agent sends 100 property modifications for one schematic, the current executor performs 100 full parse-serialize-write cycles. execute_batch() eliminates this by grouping ops by target file, parsing each file once, validating all operations upfront, applying all mutations to a single IR instance, and writing once.

Output: execute_batch() method on OperationExecutor, comprehensive batch tests, performance benchmark.
</objective>

    Add `execute_batch()` method to `OperationExecutor` in `src/kicad_agent/ops/executor.py`:

    1. Add method after the existing `execute()` method (around line 770):
    ```python
    def execute_batch(self, ops: list[Operation]) -> dict[str, Any]:
        """Execute multiple operations with single parse/write per file.

        PERF-02: Groups ops by target file, parses each once, applies all
        mutations, serializes once per file.
        PERF-03: Validates ALL operations before executing ANY. Rejects
        entire batch on validation failure with full error report.

        Not supported: cross-file ops (propagate_symbol_change) and
        create ops (create_schematic, etc.) -- use execute() for those.

        Returns:
            Dict with success, results list, and validation_errors on failure.
        """
    ```

    2. Implementation structure:
       a. **Early return** for empty ops list: `{"success": True, "results": []}`

       b. **Reject unsupported op types**: Check each op's `root.op_type` against `_CROSS_FILE_OP_TYPES` and `_CREATE_OP_TYPES`. If any match, reject with error listing unsupported ops.

       c. **Group by target file**: Build `dict[Path, list[Operation]]` mapping resolved file paths to their operations. Apply T-24-01 path confinement check on every op, not just the first.

       d. **Phase 1 -- Parse and validate ALL**:
          - Call `_clear_registry()` once at start
          - For each file group:
            - Check cache (if self._cache): `cached = self._cache.get(file_path)`
            - On cache hit: create IR from cached ParseResult (respect one-IR-per-ParseResult invariant -- only ONE IR per file)
            - On cache miss: parse file, extract UUIDs (for PCB), create IR, store in cache
            - Store IR in `ir_map: dict[Path, BaseIR]`
          - For each operation in each file group:
            - Try to dispatch to handler (dry run validation): look up handler in `_SCHEMATIC_HANDLERS` or `_PCB_HANDLERS` based on file extension. If handler not found, record validation error with op_type and error message.
            - Note: Pydantic schema validation already happened at Operation construction time. This check catches unknown op_types and wrong-handler-file-type mismatches.
          - If any validation errors: return `{"success": False, "results": [], "validation_errors": errors, "error": f"Batch rejected: {len(errors)} validation failures"}`

       e. **Phase 2 -- Apply mutations and serialize**:
          - For each file group:
            - Open `Transaction(file_path)` context
            - For each op: dispatch to correct handler via `_dispatch()` or `_dispatch_pcb()`, collect result detail
            - Serialize once per file (schematic: `serialize_schematic()` + `normalize_kicad_output()`; PCB: `serialize_pcb()` unless `ir._raw_written`)
            - `txn.commit()`
            - On commit success: invalidate old cache entry, store new `CacheEntry(parse_result=ir._parse_result, uuid_map=...)`
          - Collect all results into list

       f. **Return**: `{"success": True, "results": results_list}`

    3. Helper method `_parse_file(file_path) -> BaseIR`:
       - Extract the parse+IR-construction logic from `_execute_schematic` and `_execute_pcb` into a reusable private method:
       ```python
       def _parse_file(self, file_path: Path) -> BaseIR:
           if file_path.suffix == ".kicad_pcb":
               parse_result = parse_pcb(file_path)
               uuid_map = extract_uuids(parse_result.raw_content, "pcb")
               return PcbIR(_parse_result=parse_result, _uuid_map=uuid_map)
           else:
               parse_result = parse_schematic(file_path)
               return SchematicIR(_parse_result=parse_result)
       ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Batch input -> validation | Untrusted operation list crosses into executor; each op validated independently |
| Batch -> Transaction | All mutations in a file group share one Transaction; failure rolls back entire group |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-03 | Tampering | Batch path traversal via mixed target_files | mitigate | T-24-01 path confinement checked on every op in batch, not just first |
| T-32-04 | Tampering | Batch injection with unsupported op types | mitigate | Reject create and cross-file ops before any mutation |
| T-32-05 | Denial of Service | Extremely large batch overwhelms memory | accept | LLM-controlled batch sizes are naturally bounded; no external batch API |
</threat_model>
