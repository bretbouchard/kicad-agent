<objective>
IR caching layer that eliminates redundant file parsing for sequential operations on the same file.

Purpose: Every execute() call currently performs a full parse-mutate-serialize-write cycle. When an LLM agent makes 100 sequential property modifications to one schematic, that means 100 redundant parses. The IRCache stores ParseResult + UUIDMap keyed by (file_path, mtime_ns), returning cached data when the file hasn't changed.

Output: New ir_cache.py module, modified executor.py with cache integration, and comprehensive tests.
</objective>
