**Design notes:**
- kicad-cli writes JSON to CWD as `{stem}-erc.json` / `{stem}-drc.json` (not stdout). The wrapper must handle this by running in a temp directory or capturing the output file.
- kicad-cli does NOT output to stdout with `--format json`; it writes a file. Use `--output -` to write to stdout, OR use a tempdir approach. IMPORTANT: Test which works first. If `--output /dev/stdout` works on macOS, prefer that. If not, use a tempdir and read the file.
- If kicad-cli is not found on PATH, raise a clear `FileNotFoundError` with install instructions.
- Parse the JSON report into frozen dataclasses for immutability (project coding style rule).

**Module structure:**

```python
"""ERC and DRC wrappers invoking kicad-cli with structured result parsing.

VAL-01: ERC gate via kicad-cli with structured pass/fail/warning results.
VAL-02: DRC gate via kicad-cli with structured pass/fail/warning results.

kicad-cli writes JSON report files (not stdout). This module handles:
  1. kicad-cli discovery and invocation
  2. JSON report file capture and parsing
  3. Structured result dataclass construction
  4. Error handling for missing CLI, failed runs, malformed output

Usage:
    from kicad_agent.validation.erc_drc import run_erc, run_drc

    result = run_erc(Path("my_schematic.kicad_sch"))
    if result.passed:
        print("ERC clean")
    else:
        for v in result.errors:
```

**Types to define:**

```python
import json
import logging
import shutil
import subprocess
import tempfile
from dataclasses import dataclass, field
from enum import Enum
from pathlib import Path
from typing import Any, Optional

logger = logging.getLogger(__name__)


class Severity(str, Enum):
    """Violation severity levels from kicad-cli reports."""
    ERROR = "error"
    WARNING = "warning"
    EXCLUSION = "exclusion"


@dataclass(frozen=True)
class Violation:
    """A single ERC/DRC violation with description, severity, type, and affected items."""
    description: str
    severity: Severity
    type: str
    items: tuple[dict[str, Any], ...] = ()
    sheet_path: str = "/"  # ERC only -- which sheet the violation is on


@dataclass(frozen=True)
class ErcResult:
    """Structured result from an ERC check.

    VAL-01: Structured pass/fail/warning result for schematic validation.
    """
    passed: bool
    file_path: Path
    violations: tuple[Violation, ...] = ()
    ignored_checks: tuple[dict[str, str], ...] = ()
    kicad_version: str = ""
    error_message: Optional[str] = None  # Set if kicad-cli invocation failed

    @property
    def errors(self) -> tuple[Violation, ...]:
        """Violations with severity=error."""
        return tuple(v for v in self.violations if v.severity == Severity.ERROR)

    @property
    def warnings(self) -> tuple[Violation, ...]:
        """Violations with severity=warning."""
        return tuple(v for v in self.violations if v.severity == Severity.WARNING)

    @property
    def error_count(self) -> int:
        return len(self.errors)

    @property
    def warning_count(self) -> int:
        return len(self.warnings)


@dataclass(frozen=True)
class DrcResult:
    """Structured result from a DRC check.

    VAL-02: Structured pass/fail/warning result for PCB validation.
    Also captures unconnected_items and schematic_parity for VAL-03.
    """
    passed: bool
    file_path: Path
    violations: tuple[Violation, ...] = ()
    unconnected_items: tuple[Violation, ...] = ()
    schematic_parity: tuple[dict[str, Any], ...] = ()
    ignored_checks: tuple[dict[str, str], ...] = ()
    kicad_version: str = ""
    error_message: Optional[str] = None

    @property
    def errors(self) -> tuple[Violation, ...]:
        return tuple(v for v in self.violations if v.severity == Severity.ERROR)

    @property
    def warnings(self) -> tuple[Violation, ...]:
        return tuple(v for v in self.violations if v.severity == Severity.WARNING)

    @property
    def error_count(self) -> int:
        return len(self.errors)

    @property
    def warning_count(self) -> int:
        return len(self.warnings)
```

**Functions to implement:**

1. `_find_kicad_cli() -> str` -- Find kicad-cli on PATH. Raise `FileNotFoundError` with helpful message if not found. Use `shutil.which("kicad-cli")`.

2. `run_erc(schematic_path: Path, *, timeout: int = 120) -> ErcResult` -- Run ERC on a schematic file:
   - Validate file exists and has `.kicad_sch` extension
   - Create tempdir, run `kicad-cli sch erc --format json --severity-all <file>` in that tempdir
   - Read the generated `{stem}-erc.json` file
   - Parse JSON, extract violations from all sheets, construct ErcResult
   - `passed = True` if zero errors (warnings are non-fatal)
   - Handle: kicad-cli not found, subprocess timeout, non-zero exit, malformed JSON, missing output file
   - Clean up tempdir in a `finally` block

3. `run_drc(pcb_path: Path, *, check_schematic_parity: bool = False, timeout: int = 300) -> DrcResult` -- Run DRC on a PCB file:
   - Same pattern as run_erc but with `kicad-cli pcb drc --format json --severity-all`
   - If `check_schematic_parity=True`, add `--schematic-parity` flag
   - Parse violations AND unconnected_items AND schematic_parity from JSON
   - `passed = True` if zero errors and zero unconnected items
   - DRC takes longer than ERC, hence longer default timeout

4. `_parse_violations(violations_json: list[dict], sheet_path: str = "/") -> tuple[Violation, ...]` -- Parse a list of violation dicts from the JSON report into Violation tuples.

For both `run_erc` and `run_drc`, use `subprocess.run()` with:
  - `capture_output=True`
  - `timeout=timeout`
  - `cwd=tempdir` (so kicad-cli writes the report file there)
  - Do NOT use `--exit-code-violations` -- we parse the JSON instead (exit code semantics vary)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Python -> kicad-cli subprocess | External process invocation; kicad-cli may be absent, hang, or produce unexpected output |
| kicad-cli JSON report -> Python parser | Untrusted JSON from external tool; must validate structure before accessing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Denial of Service | run_erc, run_drc subprocess calls | mitigate | Timeout parameter (120s ERC, 300s DRC) prevents hanging; subprocess.run(timeout=) enforced |
| T-03-02 | Tampering | JSON report parsing | mitigate | Wrap json.loads in try/except; return error result (not crash) on malformed JSON |
| T-03-03 | Information Disclosure | kicad-cli error output | accept | stderr captured but only logged at debug level; no sensitive data in KiCad files |
| T-03-04 | Denial of Service | Tempdir cleanup | mitigate | Use finally block to ensure tempdir cleanup regardless of success/failure |
| T-03-05 | Spoofing | kicad-cli binary path | accept | Uses PATH lookup via shutil.which; assumes trusted PATH (local development tool) |
</threat_model>
