<objective>
Create a pre-mutation structural validator that catches invalid operations before they are executed, and a UUID uniqueness checker that ensures no duplicate UUIDs exist after mutation.

Purpose: The structural validator is the first gate in the validation pipeline. It checks whether an operation makes sense given the current state of the file (e.g., does the target component exist? does the target sheet exist? is the library reference valid?). This prevents wasted mutations that would inevitably fail ERC/DRC or corrupt the file. The UUID uniqueness checker ensures that every UUID in the file is unique after mutation, preventing dangling references.

Output: `structural.py` with `validate_structural()` and `validate_uuid_uniqueness()` functions, frozen result types, and comprehensive tests against real fixture files.
</objective>

**Module docstring:**
```python
"""Pre-mutation structural validation for KiCad operations.

VAL-05: Catch invalid operations before execution.
Checks that the operation is structurally sound given the current file state:
  - Target component exists (for remove, move, modify operations)
  - Target file type matches the operation type
  - UUID uniqueness is maintained after mutation
  - Library references are syntactically valid

This validator runs BEFORE the mutation is applied. It does NOT run ERC/DRC
(that happens post-mutation in the validation pipeline).

Usage:
    from kicad_agent.validation.structural import validate_structural

    result = validate_structural(operation, ir)
    if result.passed:
        # Safe to proceed with mutation
    else:
        for v in result.violations:
            print(f"BLOCKED: {v.description}")
"""
```

**Types to define:**

```python
import logging
import re
from dataclasses import dataclass
from enum import Enum
from typing import Any, Optional

from kicad_agent.ops.schema import (
    Operation,
    AddComponentOp,
    RemoveComponentOp,
    MoveComponentOp,
    ModifyPropertyOp,
)
from kicad_agent.ir.base import BaseIR

logger = logging.getLogger(__name__)


class ViolationKind(str, Enum):
    """Categories of structural violations."""
    MISSING_COMPONENT = "missing_component"
    FILE_TYPE_MISMATCH = "file_type_mismatch"
    INVALID_LIBRARY_REF = "invalid_library_ref"
    DUPLICATE_UUID = "duplicate_uuid"
    INVALID_POSITION = "invalid_position"
    EMPTY_REFERENCE = "empty_reference"


@dataclass(frozen=True)
class StructuralViolation:
    """A single structural validation violation."""
    kind: ViolationKind
    description: str
    detail: str = ""  # Additional context (e.g., which component, which UUID)


@dataclass(frozen=True)
class StructuralResult:
    """Result of a structural validation check."""
    passed: bool
    violations: tuple[StructuralViolation, ...] = ()
    operation_type: str = ""  # op_type from the operation
    target_file: str = ""     # target_file from the operation

    @property
    def error_count(self) -> int:
        return len(self.violations)
```

**Functions to implement:**

1. `validate_structural(operation: Operation, ir: BaseIR) -> StructuralResult` -- Main entry point:
   - Dispatch based on `operation.root.op_type` to type-specific validators
   - Each validator checks operation-specific constraints against the IR state
   - Collect all violations and return StructuralResult
   - `passed = True` only if zero violations

2. `_validate_add_component(op: AddComponentOp, ir: BaseIR) -> list[StructuralViolation]`:
   - Check `ir.file_type == "schematic"` (file type mismatch if not)
   - Check `library_id` matches `LIBRARY:SYMBOL` format (contains `:` with non-empty both sides)
   - Check `position.x >= 0` and `position.y >= 0` (negative coordinates are invalid)
   - Check `reference` is non-empty after stripping

3. `_validate_remove_component(op: RemoveComponentOp, ir: BaseIR) -> list[StructuralViolation]`:
   - Check `ir.file_type == "schematic"` (file type mismatch if not)
   - Check target component exists via `ir.get_component_by_ref(op.reference)` -- if IR is SchematicIR. For PcbIR, check footprints by reference.
   - If component not found: MISSING_COMPONENT violation

4. `_validate_move_component(op: MoveComponentOp, ir: BaseIR) -> list[StructuralViolation]`:
   - Same file type and component existence checks as remove
   - Check `position.x >= 0` and `position.y >= 0`

5. `_validate_modify_property(op: ModifyPropertyOp, ir: BaseIR) -> list[StructuralViolation]`:
   - Check target component exists
   - Check `property_name` is non-empty
   - Note: We do NOT validate that the property exists on the component (user may be adding a new custom property)

6. `validate_uuid_uniqueness(ir: BaseIR) -> StructuralResult` -- Check UUID uniqueness across the entire file:
   - Extract all UUIDs from `ir.raw_content` using `extract_uuids()`
   - Build a `dict[str, int]` counting each UUID value
   - Any UUID appearing more than once is a DUPLICATE_UUID violation
   - Return StructuralResult with any duplicate UUID violations
   - This function can be called post-mutation to verify the file still has unique UUIDs

For the component existence check, use duck typing rather than isinstance checks:
```python
# Try to find component by reference -- works for both SchematicIR and PcbIR
def _component_exists(ir: BaseIR, reference: str) -> bool:
    """Check if a component/footprint with the given reference exists."""
    # SchematicIR has get_component_by_ref
    if hasattr(ir, 'get_component_by_ref'):
        return ir.get_component_by_ref(reference) is not None
    # PcbIR has footprints with reference property
    if hasattr(ir, 'footprints'):
        for fp in ir.footprints:
            if hasattr(fp, 'reference') and fp.reference == reference:
                return True
    return False
```
