# GSD Phase State Machine (from Obdurate Runtime)

```swift
enum GSDPhase: String, Codable {
    case questioning
    case research
    case planning
    case review
    case execute
    case verification
    case complete
}
```

# Conversation Forking (CHAT-08, GEN-01)

User can edit a previous message and re-submit:
1. Tap message → enters edit mode
2. Edit text → tap "Re-submit"
3. Creates new conversation branch (fork from that point)
4. New branch gets new UUID, parent references old conversation
5. SwiftData stores fork relationship (ProjectBranch model)

# Spec Card Generation (GSD-02)

LLM generates structured spec from questioning responses:
- Project title
- Goal statement
- Key requirements (3-7 bullets)
- Success criteria (3-5 bullets)
- Constraints (budget, size, power)

# Roadmap Generation (GSD-03)

LLM generates visual roadmap with:
- Phases as timeline nodes (questioning → research → planning → execute → verify)
- Each phase: goal, requirements, success criteria
- Estimated duration per phase
- Dependency arrows between phases

# Completion Summary (GSD-08)

Shows when project phase ships:
- Renders (schematic SVG, PCB PNG)
- Exports (Gerber files, drill files, BOM)
- Decisions made (count + key decisions)
- Phase duration (total time from start to complete)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM → Spec/Roadmap | Unstructured LLM output parsed into structured spec/roadmap |
| User → Spec Fields | User input to editable spec fields |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-173-01 | Spoofing | LLM-generated spec | mitigate | Parse spec with Pydantic-style validation. Reject malformed specs. Regenerate on failure. |
| T-173-02 | Tampering | User-edited spec fields | mitigate | Validate spec field length (max 1000 chars), sanitize HTML (no XSS). |
| T-173-03 | Information Disclosure | Decision events | accept | Decision events stored locally — no external exposure. |
| T-173-04 | Denial of Service | Fork spam | mitigate | Cap forks at 100 per conversation. Show warning at 80. |

## Adversarial Scenario

**Scenario:** A malicious LLM response includes a crafted spec with HTML injection in the goal statement (e.g., `<script>alert('XSS')</script>`). When the spec is rendered in `SpecView`, the script executes.

**Mitigation:** Sanitize all LLM-generated text before rendering. Use `NSAttributedString` with `.init(data: options: [.documentType: NSAttributedString.DocumentType.html])` for safe rendering. Strip script tags and event handlers (onclick, onerror). Store raw LLM output in SwiftData, display sanitized version.
</threat_model>
