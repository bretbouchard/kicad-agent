  <action>
Create ProviderRouter.swift implementing task-aware routing:
```swift
struct ProviderRouter {
    let foundationModelsProvider: FoundationModelsProvider
    let mlxSwiftProvider: MLXSwiftProvider
    let cloudProviders: [String: LLMProvider]  // ProviderID → Provider

    func selectProvider(for task: TaskType, settings: ProviderSettings) -> LLMProvider {
        // Privacy mode override
        if settings.privacyMode {
            switch task {
            case .circuitGeneration, .pcbRouting:
                return mlxSwiftProvider
            default:
                return foundationModelsProvider
            }
        }

        // Check user preference per task
        if let preferredID = settings.preferredProviderForTaskType[task],
           let provider = cloudProviders[preferredID] {
            // Verify provider has API keys (non-empty config)
            if provider.isAvailable {
                return provider
            }
        }

        // Fallback to task-type defaults
        switch task {
        case .circuitGeneration:
            return mlxSwiftProvider  // Free local generation
        case .pcbRouting:
            return mlxSwiftProvider  // Gemma 4 12B V2 vision (local)
        case .boardAnalysis, .conversationHistory, .quickReply:
            return foundationModelsProvider  // Free built-in
        case .complexReasoning:
            // Try cloud provider, fallback to FoundationModels
            if let preferredID = settings.preferredProviderForTaskType[.complexReasoning],
               let provider = cloudProviders[preferredID],
               provider.isAvailable {
                return provider
            }
            return foundationModelsProvider  // MOD-11 fallback
        }
    }

    func estimateCost(for task: TaskType, inputTokens: Int, outputTokens: Int, settings: ProviderSettings) -> Decimal {
        let provider = selectProvider(for: task, settings: settings)
        return provider.estimateCost(inputTokens: inputTokens, outputTokens: outputTokens)
    }
}
```

Per MOD-02: Task-aware, cost-aware, privacy-aware routing.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Provider Router → Cloud Providers | Untrusted HTTPS (user API keys, external costs) |
| Provider Settings → SwiftData | Trusted local storage (user preferences) |
| Router → FoundationModels/MLX | Trusted local frameworks (no network) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-165-01 | S | Cloud provider API key input | mitigate | Validate key format before storing, test call on save per MOD-03, revoke on 401. |
| T-165-02 | I | Provider preferences (SwiftData) | accept | User-provided preferences, validated enum values (TaskType), no injection possible. |
| T-165-03 | D | Cost tracking (decimal overflow) | mitigate | Use Decimal for currency (arbitrary precision), validate cost < $1000 per message, warn user on runaway spend. |
| T-165-04 | E | Cloud provider unavailable | mitigate | Fallback to FoundationModels per MOD-11, show banner explaining fallback, no data loss. |
| T-165-05 | R | Privacy mode bypass | mitigate | Privacy mode is client-side only, routes to local providers, no cloud calls. Verified via code review. |
| T-165-06 | T | Cost tracking telemetry | accept | Local-only spend tracking (SwiftData), no external telemetry per pure BYOK principle. |

## Adversarial Scenario

**Attack:** Attacker steals API key from Keychain, routes all traffic through expensive cloud provider, drains user credits. **Mitigation:** (S) Keychain stores encrypted at rest, (D) Cost tracking warns on runaway spend (> $100/message), (E) Provider unavailable fallback prevents silent failure, (R) Privacy mode forces local-only, (I) Provider preferences validated enum values, (T) Local-only tracking prevents external exposure.
</threat_model>
