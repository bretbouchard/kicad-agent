  <name>Task 1: Create GitHub Actions release workflow</name>
  <files>.github/workflows/release.yml</files>
  <action>
Create `.github/workflows/release.yml` with two sequential jobs: build and publish.

Workflow trigger:
- `on.push.tags: ["v*"]`

**build job:**
- `runs-on: ubuntu-latest`
- Steps:
  1. `actions/checkout@v6` with `fetch-depth: 0` -- CRITICAL: setuptools-scm needs full git history to derive version from tag. Without this, version will be "0.0.0" (RESEARCH Pitfall 1).
  2. `actions/setup-python@v6` with `python-version: "3.11"`
  3. `run: pip install build`
  4. `run: python -m build` -- produces wheel and sdist in dist/
  5. `actions/upload-artifact@v5` with `name: dist` and `path: dist/` -- passes artifacts to publish job

**publish job:**
- `needs: build`
- `runs-on: ubuntu-latest`
- `environment: pypi` -- must match the environment configured in PyPI Trusted Publishing settings
- `permissions: id-token: write` -- required for OIDC token exchange
- Steps:
  1. `actions/download-artifact@v6` with `name: dist` and `path: dist/`
  2. `pypa/gh-action-pypi-publish@release/v1` -- uses OIDC, no API token needed

**create-github-release job:**
- `needs: publish`
- `runs-on: ubuntu-latest`
- `permissions: contents: write`
- Steps:
  1. `actions/checkout@v6` with `fetch-depth: 0`
  2. `actions/download-artifact@v6` with `name: dist` and `path: dist/`
  3. Use `softprops/action-gh-release@v2` with `files: dist/*` to create a GitHub Release with the tag name and attach the built artifacts.

Use YAML with 2-space indentation. The `.github/workflows/` directory already exists from Plan 01.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Tag push -> release workflow | Only maintainers with push access can create tags |
| OIDC token -> PyPI | GitHub issues short-lived OIDC token trusted by PyPI |
| Build artifacts -> publish job | Artifacts passed between jobs via GitHub Actions storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-05 | S | Non-maintainer pushes fake tag | mitigate | Restrict tag creation to maintainers via GitHub branch protection rules; environment protection on 'pypi' environment |
| T-18-06 | T | Compromised build artifact between jobs | accept | GitHub Actions artifact storage is internal; OIDC token is short-lived |
| T-18-07 | I | PyPI credentials exposed | mitigate | Trusted Publishing uses OIDC (no stored credentials); id-token: write permission scoped to publish job only |
| T-18-08 | D | Release workflow denial of service | accept | GitHub Actions availability is outside our control; releases can be retried |
</threat_model>
