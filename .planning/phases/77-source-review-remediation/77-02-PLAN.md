# Plan 77-02: Critical Serializer Fixes (Wave 1)

**Phase**: 77 - Source Review Remediation
**Depends on**: 76 (Native PCB Parser)
**Wave**: 1 of 5 (parallel with 77-01)
**Scope**: Fix 5 Critical/High serializer bugs that can silently corrupt files

## Context

The serializer subsystem has the highest-impact architectural issue: `serialize_pcb()` and `serialize_footprint()` route through `kiutils.to_file()` despite documented kiutils PCB corruption. The schematic serializer has a parallel regex-fix system that's less robust than the normalizer module. UUID reinjection assumes perfect 1:1 element mapping which is never guaranteed.

## Tasks

### Task 1: Fix serialize_pcb kiutils corruption path (S-BUG-001, Critical)

**File**: `src/kicad_agent/serializer/pcb_ser.py:45-53`

1. Check if there's a raw S-expression write path available (from PcbRawWriter in ops/pcb_raw_writer.py)
2. If serialize_pcb is only called with kiutils objects, add a path that writes the raw S-expression directly
3. If kiutils path must remain (for existing callers), add a clear deprecation warning
4. Ensure UUID reinjection still works with the new path
5. Add round-trip test: parse PCB → serialize → parse → compare (element count, net count, UUID count)

**Acceptance**: PCB serialization doesn't go through kiutils to_file(). Round-trip preserves all elements and UUIDs.

### Task 2: Fix schematic_ser in_bom/on_board blanket replacement (S-BUG-002, High)

**File**: `src/kicad_agent/serializer/schematic_ser.py:175-180`

1. Change Fix 6 to only replace `(in_bom yes) (on_board yes)` with `(in_bom no) (on_board no)` for elements that are embedded lib symbol references (not placed components)
2. Detect embedded lib symbols by checking for `(lib_id ...)` without a `(property "Reference" ...)` that looks like a placed component
3. Alternative: Only apply this fix inside `(lib_symbols ...)` section, not in `(symbol_instances ...)` or placed components
4. Add test: schematic with placed components preserves their in_bom/on_board status

**Acceptance**: Placed components retain `(in_bom yes) (on_board yes)` after serialization. Only lib symbol references get `(in_bom no) (on_board no)`.

### Task 3: Fix uuid_reinjector count mismatch handling (S-BUG-003, High)

**File**: `src/kicad_agent/serializer/uuid_reinjector.py:184-233`

1. Before injection, compare element count from UUID map vs element count in serialized output
2. If counts don't match, raise `ValueError` with details (expected N type_X, found M type_X) instead of silently misassigning
3. Log a clear error message showing which element types have count mismatches
4. Add test: UUID reinjection with removed element raises ValueError

**Acceptance**: UUID reinjection fails loudly on count mismatch instead of silently corrupting.

### Task 4: Unify schematic_ser with normalizer (S-BUG-004, High)

**File**: `src/kicad_agent/serializer/schematic_ser.py:89-187`

1. Replace `_fix_kiutils_output()` calls with `normalize_kicad_output()` from normalizer.py
2. Keep any schematic-specific fixes that the normalizer doesn't cover (verify by diff)
3. If the normalizer is missing fixes that schematic_ser had, add them to the normalizer instead
4. Add test: schematic with properties containing `)` characters gets correct output

**Acceptance**: Schematic serialization uses the normalizer module. No parallel regex implementations remain.

### Task 5: Fix footprint_ser non-atomic write (S-BUG-005, Low)

**File**: `src/kicad_agent/serializer/footprint_ser.py:47-51`

1. Replace `file_path.write_text(...)` with `atomic_write(file_path, ...)`
2. Match the pattern used by serialize_pcb
3. Add test: footprint serialization uses atomic write (can verify via test spy or file system check)

**Acceptance**: Footprint serialization uses atomic_write for crash safety.
