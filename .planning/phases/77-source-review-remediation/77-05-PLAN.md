# Plan 77-05: Medium/Low Bug Fixes (Wave 3)

**Phase**: 77 - Source Review Remediation
**Depends on**: All Wave 1-2 plans (77-01 through 77-04)
**Wave**: 3 of 5
**Scope**: Fix remaining Medium and Low severity bugs across all subsystems

## Context

After addressing Critical and High bugs in Waves 1-2, Wave 3 cleans up Medium and Low issues. These include missing KiCad 10 graphic types in the parser, concurrent access gaps, DFM profile gaps, dead code removal, and parser robustness improvements.

## Tasks

### Task 1: Add missing KiCad 10 graphic types to parser (P-BUG-005, High)

**File**: `src/kicad_agent/parser/pcb_native_parser.py:730-747`

1. Extend `_extract_graphic_items` type_map to include:
   - `gr_text`: text annotations (position, text content, layer, font size, rotation)
   - `gr_text_box`: text boxes (position, text, size, border)
   - `dimension`: dimension markers (position, text, style)
   - `target`: tooling targets (position, size, crosshair style)
2. Add corresponding NativeGraphicItem fields or new dataclasses
3. Update UUID extractor parent type patterns to include new types
4. Add test: parse PCB with text annotations → graphic items include gr_text

**Acceptance**: All 10 KiCad 10 graphic item types parsed and accessible.

### Task 2: Add JLCPCB DFM profile (V-BUG-003, Medium)

**File**: `src/kicad_agent/dfm/profiles.py`, `src/kicad_agent/dfm/checker.py`

1. Add `JLCPCB_4LayerProfile` with:
   - Min track width: 0.1mm
   - Min drill: 0.2mm
   - Min via pad: 0.4mm
   - Min clearance: 0.1mm
   - Max board size: 300x300mm
   - Solder mask: green (default)
   - Silkscreen: white (default)
   - Impedance control: optional add-on
2. Register in profile registry
3. Add CLI option: `kicad-agent dfm check --profile jlcpcb-4layer`
4. Add test: DFM checker produces JLCPCB-specific results

**Acceptance**: JLCPCB profile available for DFM checks matching fab requirements.

### Task 3: Fix split_plane trace crossing detection (V-BUG-002, High)

**File**: `src/kicad_agent/validation/split_plane.py`

1. Investigate why trace crossing detection returns no results
2. Fix the detection logic (likely a data path issue between NativeBoard segments and the checker)
3. Add test: split plane with traces crossing boundary → detection reports violation

**Acceptance**: Split plane trace crossing detection functional.

### Task 4: Fix pcb_netlist depth pre-scan (P-BUG-006, Critical from parser review)

**File**: `src/kicad_agent/parser/pcb_netlist.py:39-43`

1. Add depth pre-scan matching the pattern from `pcb_native_parser.py` and the fix from 77-01 Task 1
2. Also add 50MB size check before sexpdata.loads()
3. Add test: deeply nested netlist raises ValueError instead of RecursionError

**Acceptance**: Netlist parser safe from RecursionError and oversized files.

### Task 5: Improve batch_executor error handling (O-BUG-009, Medium)

**File**: `src/kicad_agent/ops/batch_executor.py:286-311`

1. Wrap individual handler dispatch in try/except within the Transaction block
2. On individual op failure, capture the error and add to results as a structured error
3. Continue with remaining ops in the batch (don't abort entire batch on single failure)
4. Return partial results with error indicators
5. Add test: batch with 3 ops, middle one fails → first and third succeed, middle has error

**Acceptance**: Batch operations handle individual failures gracefully with structured error responses.

### Task 6: Document concurrent access limitation (O-BUG-008, Medium)

**File**: `src/kicad_agent/ops/execution.py` (module docstring)

1. Add clear documentation that kicad-agent is NOT safe for concurrent access to the same file
2. Document that two processes editing the same file will corrupt it
3. Add a file-lock check at the start of execute() that warns (not blocks) if a lock file exists
4. Optionally: add a `.kicad_agent.lock` file mechanism that warns concurrent editors
5. Add test: warning emitted when lock file exists

**Acceptance**: Concurrent access limitation documented and warned.

### Task 7: Clean up dead code and fragile parsers (Low)

**Files**: `wire_router.py:117-127`, `netlist_parser.py:54-63`

1. `wire_router.py`: Remove unused `_snap_to_grid()` if grid snapping is handled by Task 4 in 77-04
2. `netlist_parser.py`: Add robustness comments to the fragile depth counting, add assertion that the regex match starts at `(net`
3. `schematic_routing/net_resolver.py`: Remove unused `_find_sheet_graph()` function
4. Add test: verify dead code removal doesn't break anything (existing tests pass)

**Acceptance**: Dead code removed. Fragile parsers documented with caveats.

### Task 8: Fix batch_executor PreAnalysisGate singleton (O-BUG-011, Low)

**File**: `src/kicad_agent/ops/batch_executor.py:46-51`

1. Change `_get_pre_analysis_gate()` to return the module-level singleton from execution.py instead of creating a new instance
2. Import from execution.py or use a shared module-level gate
3. Add test: batch_executor uses same PreAnalysisGate instance as execution.py

**Acceptance**: Single PreAnalysisGate instance shared across execution paths.
