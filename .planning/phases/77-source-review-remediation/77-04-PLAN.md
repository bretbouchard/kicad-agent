# Plan 77-04: Schematic Routing Fixes (Wave 2)

**Phase**: 77 - Source Review Remediation
**Depends on**: 77-01 (parser fixes needed for multi-unit pin resolution)
**Wave**: 2 of 5 (parallel with 77-03)
**Scope**: Fix 1 Critical, 4 High, 3 Medium bugs in schematic_routing subsystem

## Context

The schematic routing subsystem has the most operationally impactful bugs. R-BUG-001 (power unit placer inserting outside schematic block) means the entire power unit placement feature is non-functional. R-BUG-002 (multi-unit IC pin resolution) means wires may route to phantom pins for CD4066, TL072, and NE5532 — which are the most-used ICs in the analog ecosystem.

## Tasks

### Task 1: Fix power_unit_placer insertion point (R-BUG-001, Critical)

**File**: `src/kicad_agent/schematic_routing/power_unit_placer.py:643-644`

1. Replace `content.rfind(")")` with a search for the closing paren of the `(schematic ...)` form specifically
2. The correct target is the `)` that closes `(schematic ...)` inside `(kicad_sch ...)`
3. Strategy: find `(schematic ` at the top level, then find its matching `)` using paren counting or the existing `_find_matching_close_paren` utility
4. Insert power unit blocks before the `(schematic ...)` closing paren, not the `(kicad_sch ...)` closing paren
5. Add test: power_unit_placer output has blocks inside `(schematic ...)`, verifiable by paren balance check

**Acceptance**: Power units placed inside schematic block, visible to KiCad. Verify by loading generated file in KiCad.

### Task 2: Fix multi-unit IC pin resolution (R-BUG-002, High)

**Files**: `schematic_graph.py:332-409`, `pin_resolver.py:161-168`

Part A — schematic_graph:
1. In `_parse_symbol_pins()`, filter returned pins by the unit number of the placed symbol
2. Use the `(unit N)` property from the symbol instance to determine which unit is placed
3. Only return pins for that specific unit

Part B — pin_resolver:
1. Fix `_build_unit_index` heuristic: don't map unit 1 to unit 0 catch-all when pin count ≤ 20
2. For unit 2+, always look up the specific unit's sub-symbol in lib_symbols
3. Add test: CD4066BE unit 1 returns only pins 1,2,3,4,13,14 (6 pins), not all 14

**Acceptance**: Multi-unit ICs return only their placed unit's pins. No phantom pins.

### Task 3: Fix batch_executor new_segment insertion (R-BUG-003, High)

**File**: `src/kicad_agent/schematic_routing/batch_executor.py:249`

1. Same fix as R-BUG-001: find `(schematic ...)` closing paren, not `(kicad_sch ...)` closing paren
2. Extract the paren-finding logic into a shared utility function used by both power_unit_placer and batch_executor
3. Add test: new_segment wire appears inside schematic block

**Acceptance**: New wire segments inserted inside schematic block.

### Task 4: Fix wire endpoint grid snapping (R-BUG-004, High)

**File**: `src/kicad_agent/schematic_routing/wire_router.py:57-58`

1. Add grid snapping function that rounds to nearest 2.54mm (50mil) grid
2. Apply to both endpoints of generated wire fixes in `generate_fixes()`
3. Use: `round(coord / 2.54) * 2.54` for proper grid alignment
4. Add test: generated wire endpoints are on 2.54mm grid

**Acceptance**: All generated wire endpoints are on the KiCad schematic grid. No endpoint_off_grid violations.

### Task 5: Fix collision_detector netlist parser (R-BUG-005, Medium)

**File**: `src/kicad_agent/schematic_routing/collision_detector.py:76-79`

1. Replace the duplicate netlist parser with the canonical one from `netlist_parser.py`
2. Or fix the regex to handle quoted code values: `\(code\s+"?\d+"?\)` → `\(code\s+(?:"(\d+)"|\d+)\)`
3. Import from `netlist_parser.py` instead of maintaining a duplicate
4. Add test: collision_detector loads net names from KiCad 10 format netlist

**Acceptance**: Collision detector correctly identifies shorted nets in KiCad 10 format.

### Task 6: Harden ERC coordinate scaling (R-BUG-006, Medium)

**File**: `src/kicad_agent/schematic_routing/__init__.py:106-107`

1. Add a runtime check: compare ERC JSON coordinate scale with schematic coordinate scale
2. If the ratio is not 100x within tolerance, log a clear warning and skip routing (rather than silently producing 0 fixes)
3. Document the assumption in a comment with a reference to the KiCad version
4. Add test: coordinate scaling validation catches wrong scale factor

**Acceptance**: ERC coordinate scaling validated at runtime. Wrong scale produces clear warning instead of silent failure.

### Task 7: Add hierarchical sheet pin parsing (R-BUG-007, Medium)

**File**: `src/kicad_agent/schematic_routing/schematic_graph.py:441-475`

1. Parse `(sheet_pin ...)` elements in `_parse_labels()` or a new `_parse_sheet_pins()` method
2. Store sheet pins with their position and net name in the graph
3. Use sheet pins as connectivity nodes in net extraction and routing
4. Add test: hierarchical design with sheet pins has cross-sheet connectivity in the graph

**Acceptance**: Cross-sheet nets visible to the router via hierarchical sheet pins.

### Task 8: Enable L-shaped routing or provide alternative (R-BUG-008, Medium)

**File**: `src/kicad_agent/schematic_routing/wire_router.py:86-90`

1. Enable L-shaped routing by replacing the `pass` with actual implementation
2. L-shape: horizontal segment from dangling endpoint to target X, then vertical to target Y (or vice versa)
3. Add intermediate point at the corner
4. Verify corner point is on grid (from Task 4's grid snapping)
5. Add test: L-shaped wire fix for off-axis violation

**Acceptance**: L-shaped routing enabled. Off-axis violations can now be fixed.
