# Plan 77-01: Critical Parser & Validation Fixes (Wave 1)

**Phase**: 77 - Source Review Remediation
**Depends on**: 76 (Native PCB Parser)
**Wave**: 1 of 5
**Scope**: Fix 5 Critical/High bugs in parser subsystem and 1 Critical in validation

## Context

Full source review of kicad-agent (200+ Python modules, 5 subsystems) identified 38 bugs. Wave 1 addresses the most dangerous: parser safety issues (RecursionError, UUID misassignment, thread safety, dead code) and the validation pipeline temp dir leak.

Bug sources: `hardware/dumb-cartridges/aether-drive/BUGS.md` section "kicad-agent Source Review (2026-06-06)"

## Tasks

### Task 1: Fix raw_parser.py RecursionError handling (P-BUG-001, Critical)

**File**: `src/kicad_agent/parser/raw_parser.py:49-53`

1. Add depth pre-scan function `_pre_scan_depth(content: str, max_depth: int = 200) -> bool` matching the pattern from `pcb_native_parser.py:130-179`
2. Call `_pre_scan_depth()` before `sexpdata.loads()` in `parse_raw_sexp()`
3. If depth exceeds max, raise `ValueError` with clear message instead of letting sexpdata hit RecursionError
4. Handle string quoting in pre-scan (escaped quotes, multi-line strings) — copy from pcb_native_parser implementation
5. Remove the `except RecursionError` catch block (it's unsafe per CPython docs)

**Acceptance**: Deeply nested content raises ValueError cleanly instead of corrupting interpreter state.

### Task 2: Fix uuid_extractor parent type detection (P-BUG-002, Critical)

**File**: `src/kicad_agent/parser/uuid_extractor.py:96-128`

1. Replace "latest opening parent" algorithm with "nearest enclosing parent" algorithm
2. Track all opening `(type` positions and their depths in the lookback window
3. For the UUID position, find the parent with the highest depth (most deeply nested enclosing parent)
4. If depths are equal, pick the latest (closest to UUID)
5. Add test: UUID inside `(pad (net ...) (uuid ...))` inside `(footprint ...)` with an intervening `(via ...)` — should return "pad", not "via" or "footprint"

**Acceptance**: UUIDs get assigned to correct parent types. Add test with nested footprint → pad scenario.

### Task 3: Fix pcb_native_parser thread safety (P-BUG-003, High)

**File**: `src/kicad_agent/parser/pcb_native_parser.py:249-260`

1. Replace `sys.setrecursionlimit()` / `sys.setrecursionlimit()` in `parse_pcb_content()` with a `threading.Lock()`
2. Acquire lock before changing recursion limit, release in `finally`
3. Alternative: Use `threading.local()` to store per-thread original recursion limits
4. Add test: two threads parsing different PCBs simultaneously don't crash

**Acceptance**: Concurrent PCB parsing works without thread-safety crashes.

### Task 4: Remove dead code in _extract_nets (P-BUG-004, High)

**File**: `src/kicad_agent/parser/pcb_native_parser.py:308-322`

1. Remove the dead loop at lines 315-322 (the `_find_all_symbols(root, "net")` call that produces result list only to discard it)
2. Verify the second loop at line 325+ is the actual extraction logic and is unaffected
3. Run existing tests to confirm no regression

**Acceptance**: Dead code removed, existing tests pass.

### Task 5: Fix validation pipeline missing shutil import (V-BUG-001, Critical)

**File**: `src/kicad_agent/validation/pipeline.py`

1. Find where `shutil` is used (likely `shutil.rmtree()` or `shutil.copytree()`) without import
2. Add `import shutil` at module level
3. Verify temp dirs are properly cleaned up after ERC/DRC pipeline runs
4. Add test: pipeline run doesn't leak temp directories

**Acceptance**: No temp dir leaks during validation pipeline execution.
