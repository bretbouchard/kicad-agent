# Plan 77-03: Critical/High Ops/Execution Pipeline Fixes (Wave 2)

**Phase**: 77 - Source Review Remediation
**Depends on**: 77-01 (parser fixes), 77-02 (serializer fixes)
**Wave**: 2 of 5
**Scope**: Fix 7 Critical/High/Medium bugs in the ops/execution pipeline

## Context

The execution pipeline has 3 Critical cache corruption bugs where stale parse results are cached after mutations. The `convert_kicad6_to_10` operation silently undoes its own work because the executor overwrites the converted file. Project file operations lack Transaction wrapping. The PersistentUndoStack has a manifest drift bug.

## Tasks

### Task 1: Fix stale schematic cache after mutation (O-BUG-001, Critical)

**File**: `src/kicad_agent/ops/execution.py:282-330`

1. After handler mutates the IR (line 309) and executor serializes (line 314), invalidate the cache entry BEFORE re-caching
2. Re-read the file from disk to get fresh `raw_content` for the new parse result
3. Alternatively: create a new `ParseResult` from the serialized file content instead of reusing the stale one
4. Add test: mutate schematic → check cache has fresh raw_content matching file on disk

**Acceptance**: Cache always contains parse results matching the current file state.

### Task 2: Fix stale PCB cache after raw_written mutation (O-BUG-002, Critical)

**File**: `src/kicad_agent/ops/execution.py:391-436`

1. When `ir.raw_written` is True, invalidate cache and do NOT re-cache the stale parse_result
2. The next operation on this file will trigger a fresh parse
3. Also invalidate the uuid_map since raw writes may change UUIDs
4. Add test: PCB operation with raw_written=True → cache miss on next operation → fresh parse

**Acceptance**: Raw-written PCBs don't pollute cache with stale data.

### Task 3: Fix convert_kicad6_to_10 overwrite (O-BUG-003, Critical)

**File**: `src/kicad_agent/ops/handlers/schematic.py:230-236` + `ops/execution.py:312-317`

1. Add `convert_kicad6_to_10` to `SELF_SERIALIZING_OPS` set in execution.py (so executor skips its own serialization)
2. Verify the handler's direct file write is the final output
3. After handler writes converted file, re-parse it to get fresh parse_result for cache
4. Add test: convert_kicad6_to_10 on a KiCad 6 file produces KiCad 10 format that persists

**Acceptance**: `convert_kicad6_to_10` operation produces lasting conversion. File format is KiCad 10 after operation.

### Task 4: Add Transaction wrapping for project file execution (O-BUG-004, High)

**File**: `src/kicad_agent/ops/execution.py:477-509`

1. Wrap the handler call in `execute_project()` with `Transaction(file_path)`
2. On handler failure, Transaction __exit__ rolls back the file
3. Match the pattern used by `execute_schematic()` and `execute_pcb()`
4. Add test: project file handler failure rolls back the file

**Acceptance**: Project file mutations are transactional. Failure triggers rollback.

### Task 5: Fix PersistentUndoStack.clear() manifest drift (O-BUG-005, High)

**File**: `src/kicad_agent/ops/persistent_undo.py:319-327`

1. Call `_save_manifest()` at the end of `clear()` method
2. Verify manifest references empty state after clear
3. Add test: clear → check manifest → restart → verify consistent state

**Acceptance**: Manifest is consistent with disk state after clear().

### Task 6: Fix pre_pcb_gate multi-sheet support (O-BUG-006, Medium)

**File**: `src/kicad_agent/ops/validation_gates.py:195-213`

1. Change `pre_pcb_gate()` to iterate ALL `sch_files`, not just `sch_files[0]`
2. Run ERC on each schematic file
3. Aggregate violations across all sheets
4. Fail if ANY sheet has unclean ERC
5. Add test: multi-sheet design with error in sub-sheet → pre_pcb_gate fails

**Acceptance**: Pre-PCB gate validates all schematic sheets, not just the root.

### Task 7: Fix review_schematic handler routing (O-BUG-007, Medium)

**File**: `src/kicad_agent/ops/handlers/query.py:31-46`

1. Move `review_schematic` from `_QUERY_HANDLERS` to `_SCHEMATIC_QUERY_HANDLERS` in handlers/schematic_query.py
2. Or add routing logic in executor to dispatch review_schematic to schematic query path based on target file extension
3. Add test: review_schematic with .kicad_sch target works without error

**Acceptance**: `review_schematic` operation works on schematic files.
