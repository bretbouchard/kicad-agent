## Why this exists
Gap A3 from GAP-ANALYSIS-CURRENT.md. fix_net_short, fix_pin_type_mismatches, fix_shorted_nets, strip_shorts all return placeholder messages instead of performing real fixes. The Python equivalents exist (volta/ops/) and work; they need Swift parity.

## Background
Phase 235 was marked Complete on 2026-07-14 but did not actually implement these ops. The current implementations:
- FixNetShortGenOp: returns "Net short fix requires topology analysis"
- FixPinTypeMismatchesGenOp: returns "Pin type fix requires library symbol editing"
- FixShortedNetsGenOp: delegates to BreakWireShortsGenOp (this one is partly real)
- StripShortsGenOp: delegates to BreakWireShortsGenOp (this one is partly real)

The partly-real ones (fix_shorted_nets, strip_shorts) need verification. The string-returning ones (fix_net_short, fix_pin_type_mismatches) need full implementation.

## Scope (in)
- fix_net_short: real implementation using SchematicNets.findShorts + remove_wire for each short
- fix_pin_type_mismatches: real implementation using SymbolParser to update the .kicad_sym pin type
- fix_shorted_nets: verify existing BreakWireShorts implementation is correct; fix if not
- strip_shorts: verify; fix if not
- All four ops journaled as transactions

## Scope (out)
