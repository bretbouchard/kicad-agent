## Goal

Establish Fastlane as the single build/test/sign/ship pipeline for the entire v6.0 Mac+iPhone app. Every CI workflow invokes Fastlane lanes — no bespoke xcodebuild shell scripts, no manual Xcode archives, no App Store Connect web UI clicks. Unified `build_daemon` lane orchestrates the PyInstaller binary alongside Swift builds.

## Requirements Addressed

- **TEST-13**: All build/archive/sign/test/ship via Fastlane
- **TEST-14**: Code signing via Fastlane match
- **TEST-15**: TestFlight via Fastlane pilot
- **TEST-16**: App Store submission via Fastlane deliver
- **TEST-17**: Pre-submission validation via Fastlane precheck
- **TEST-18**: PyInstaller binary via Fastlane `build_daemon` lane

## Stupid-Proof Verification

**User-stupid (guardrails):**
- Code signing failures (most common ship-blocker) eliminated — `match` syncs certs automatically
- App Store rejection risks caught in CI via `precheck`, never at submission
- Beta builds always come from main branch (no debug builds shipped)
- All submissions reproducible from git SHA — no "works on my machine"

**Magic-stupid (zero friction):**
- One command ships TestFlight: `fastlane beta`
- One command submits App Store: `fastlane release`
- Code signing invisible — no provisioning profile juggling
- Screenshots auto-generated for all device sizes + languages via `snapshot`

**Fastfile lanes (initial):**
```ruby
default_platform(:mac)

platform :mac do
  desc "Build the macOS app (debug)"
  lane :build do
    build_mac_app(
      project: "KiCadAgent.xcodeproj",
      scheme: "KiCadAgent-macOS"
    )
  end
  
  desc "Build + run tests + check coverage"
  lane :test do
    scan(
      project: "KiCadAgent.xcodeproj",
      scheme: "KiCadAgent-macOS",
      code_coverage: true,
      output_types: "junit,xcresult",
      clean: true
    )
  end
  
  desc "Build PyInstaller daemon binary"
  lane :build_daemon do
    Dir.chdir("../daemon") do
      sh("pyinstaller --noconfirm kicad-agent-daemon.spec")
    end
  end
  
  desc "Build everything (daemon + app)"
  lane :build_all do
    build_daemon
    build
  end
  
  desc "Ship to TestFlight"
  lane :beta do
    build_all
    match(type: "appstore", platform: "macos")
    pilot(
      app_identifier: "com.bretbouchard.kicad-agent",
      skip_waiting_for_build_processing: true,
      notify_external_testers: true
    )
  end
  
  desc "Submit to App Store"
  lane :release do
    precheck
    deliver(
      app_identifier: "com.bretbouchard.kicad-agent",
      submit_for_review: true,
      automatic_release: true,
      force: true  # skip HTML report verification
    )
  end
  
  desc "Generate screenshots via snapshot"
  lane :screenshots do
    snapshot(
      project: "KiCadAgent.xcodeproj",
      scheme: "KiCadAgent-macOS-UITests",
      devices: ["Mac"]
    )
  end
end

platform :ios do
  desc "Build iPhone companion"
  lane :build_iphone do
    build_ios_app(
      project: "KiCadAgent.xcodeproj",
      scheme: "KiCadAgent-iOS"
    )
  end
  
  desc "Ship iPhone to TestFlight"
  lane :beta_iphone do
    build_iphone
    match(type: "appstore", platform: "ios")
    pilot(
      app_identifier: "com.bretbouchard.kicad-agent.iphone"
    )
  end
end
```

### Task 2: Code Signing via match

**Matchfile:**
```ruby
git_url("git@github.com:bretbouchard/certificates.git")

])

```

**Setup steps:**
1. Create private GitHub repo `bretbouchard/certificates` (or use existing)
2. Run `fastlane match appstore` to generate certs + profiles (first-time only)
3. Run `fastlane match development` for dev signing
4. Add Apple ID + Team ID to `.env`:
   ```
   APPLE_ID=bret@example.com
   TEAM_ID=ABC123DEF
   ITC_TEAM_ID=987654
   ```
5. Add `MATCH_PASSWORD` to GitHub Secrets (CI uses this to decrypt cert repo)

**Fastlane match nuke (recovery option):**
- `fastlane match nuke appstore` — revoke all certs, recreate (emergency only)
- Document in README when to use

**Acceptance Criteria:**
- [ ] `match appstore` succeeds on clean machine with repo access
- [ ] `match development` succeeds on all developer Macs
- [ ] CI workflow uses `MATCH_PASSWORD` from secrets to decrypt
- [ ] No `.p12` or `.mobileprovision` files tracked in app repo
- [ ] Match repo encrypted at rest (Fastlane handles via openssl)

```yaml
# .github/workflows/testflight.yml
name: TestFlight Beta
on:
  push:
    branches: [main]
jobs:
  beta:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v4
      - name: Setup Fastlane
        run: brew install fastlane
      - name: Cache Pods
        uses: actions/cache@v3
        with:
          path: Pods
          key: ${{ runner.os }}-pods-${{ hashFiles('**/Podfile.lock') }}
      - name: Decrypt secrets
        run: |
          echo "${{ secrets.MATCH_REPO_DEPLOY_KEY }}" > /tmp/match_key
          chmod 600 /tmp/match_key
          export GIT_SSH_COMMAND="ssh -i /tmp/match_key"
      - name: Fastlane beta
        env:
          APPLE_ID: ${{ secrets.APPLE_ID }}
          TEAM_ID: ${{ secrets.TEAM_ID }}
          FASTLANE_PASSWORD: ${{ secrets.FASTLANE_PASSWORD }}
          MATCH_PASSWORD: ${{ secrets.MATCH_PASSWORD }}
        run: fastlane beta
```

## Threat Model

### Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI Runner → Match Repo | Semi-trusted (CI has deploy key, repo is encrypted) |
| CI → App Store Connect | Trusted (API key scoped to single app) |
| CI → TestFlight | Semi-trusted (internal testers only pre-release) |
| Local Dev → Match Repo | Trusted (developers have deploy keys) |

### STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-203-01 | Spoofing | Match repo compromise | mitigate | Encrypt at rest via openssl, multi-party access (2+ deployers), branch protection requires review, signed commits enforced |
| T-203-02 | Tampering | CI secret leak | mitigate | Scoped API keys (App Store Connect only, no GitHub repo perms), rotate annually via `match nuke`, GitHub Secrets with 2-person approval, audit log review |
| T-203-03 | Repudiation | Build provenance | mitigate | Git SHA embedded in app version, Fastlane logs commit hash, all builds reproducible from source |
| T-203-04 | Information Disclosure | TestFlight build leak | mitigate | Internal testers only pre-release, TestFlight CSV roster pinned per pilot, external testers require approval, changelog sanitized |
| T-203-05 | Denial of Service | Malicious PR triggers release | mitigate | Required reviews + signed commits for main branch, workflow_dispatch only (no auto-release on push), `precheck` catches rejection risks |
| T-203-06 | Elevation of Privilege | PyInstaller arbitrary code | mitigate | `build_daemon` lane verifies .spec file changes, no unchecked subprocess exec, PyInstaller binary path validated, code signature verification |
| T-203-07 | Elevation of Privilege | Fastlane credential compromise | mitigate | Least-privilege API keys (App Store Connect only), no GitHub repo write access, rotate on employee offboard, audit log monitoring |

### Adversarial Scenario

**Name:** "CI credential theft via malicious PR"

**Scenario:** Attacker submits malicious PR that:
- Modifies `.github/workflows/testflight.yml` to exfiltrate `MATCH_PASSWORD`
- Adds Fastlane lane to upload PyInstaller binary to external server
- Alters `deliver` lane to submit to TestFlight with malicious metadata

**Attack vector:** If PR merges to main, CI workflow runs with elevated credentials, exfiltrates secrets, ships malicious build.

**Mitigation:**
1. Branch protection: Main branch requires 2 reviewers + signed commits (SPOOFING)
2. Workflow audit: PR modifies `.github/workflows/` → auto-block, required admin review (EOP)
3. Secret scoping: `MATCH_PASSWORD` only available to `testflight.yml`, not other workflows (TAMPERING)
4. Binary validation: `build_daemon` lane verifies PyInstaller binary signature before upload (EOP)
5. Pre-check gate: `precheck` runs before `deliver`, catches metadata tampering (REPUDIATION)

All mitigation layers must fail before attacker succeeds. Defense-in-depth prevents single-point compromise.

## Risks & Mitigations

| Risk | Mitigation |
|---|---|
| Match repo access lost (locked out of signing) | Multiple team members have deploy keys; backup passphrase in 1Password |
| Fastlane version drift | Pin Fastlane version in Gemfile (Bundler) |
| App Store API rate limits | Use App Store Connect API key (not Apple ID login) for CI |
| PyInstaller + Fastlane mismatch | `build_daemon` lane verifies binary exists before Swift build |
| Snapshot test flakiness | Frozen time fixtures (TEST-03 augmentation), retry with backoff |
| TestFlight processing delays (Apple side) | `skip_waiting_for_build_processing: true` — testers notified when ready |
