<context>
## Why this exists
Gap A4 + B8 from GAP-ANALYSIS-CURRENT.md. Phase 233 shipped SwiftSVGRenderer (150 LOC) but the preview views still render mock data. The App Store description claims inline previews. This gap makes that claim false.

## Background
SchematicPreviewView currently has hard-coded SchematicIR.mock data. PCBPreviewView similarly. SwiftSVGRenderer is real and works.

## Scope (in)
- Wire SchematicPreviewView to parse a real .kicad_sch and render via SwiftSVGRenderer
- Wire PCBPreviewView to either shell out to kicad-cli pcb render for a real PNG, or write a Swift PCB renderer (preferred, same path as Phase 233)
- Add a preview unavailable empty state for parse failures
- Ensure the preview updates on file change (debounced file watcher)

## Scope (out)
- Real-time 3D preview (separate feature)
- Pan/zoom beyond what NSScrollView gives for free
</context>

<tasks>

### Task 1: SchematicPreviewView real IR
Files: macos-app/Sources/KiCadAgent/Views/InlineRendering/SchematicPreviewView.swift

Accept a URL to a .kicad_sch instead of hard-coded SchematicIR.mock. Parse via SchematicParser (existing). Pass to SwiftSVGRenderer to get SVG. Wrap SVG in a SwiftUI Image (NSImage from SVG bytes). Add empty state for parse errors.

### Task 2: PCBPreviewView real PCB
Files: macos-app/Sources/KiCadAgent/Views/InlineRendering/PCBPreviewView.swift

Option A: shell out to kicad-cli pcb render --output path --format png if KiCad is installed.
Option B: extend SwiftSVGRenderer with a PCB to SVG path (preferred).
Detect KiCad presence; use whichever option is available. Show a KiCad 10 not installed message if Option A is needed and KiCad missing.

### Task 3: File watcher
Files: macos-app/Sources/KiCadAgent/Views/InlineRendering/PreviewFileWatcher.swift (new)

Watch the schematic / PCB file with a 250ms debounce. On change, reload the preview. Use DispatchSource.makeFileSystemObjectSource (no external dep).

### Task 4: Wire into LiquidGlassShell
Files: macos-app/Sources/KiCadAgent/Views/LiquidGlassShell.swift

When a chat message's renderArtifact is a schematic or PCB URL, pass that URL into the preview view. Add a View in KiCad button (shells out to open -a KiCad path).

### Task 5: Tests
Snapshot tests: same input to same SVG output (regression). Parse-failure: malformed .kicad_sch to Preview unavailable message shown.
</tasks>
