  <action>Create .github/workflows/ci.yml with swift-testing and coverage enforcement:
```yaml
name: CI
on: [push, pull_request]

jobs:
  test:
    runs-on: macos-26
    steps:
      - uses: actions/checkout@v4
      - name: Run tests with coverage
        run: swift test --enable-code-coverage
      - name: Generate coverage report
        run: xcrun llvm-cov report --instr-profile=.build/debug/codecov/*.profdata --ignore-filename-regex=Tests/.* .build/debug/VoltaApp*.swift
      - name: Enforce 100% coverage
        run: |
          COVERAGE=$(xcrun llvm-cov report --instr-profile=.build/debug/codecov/*.profdata .build/debug/VoltaApp*.swift | grep "Total" | awk '{print $NF}' | sed 's/%//')
          if (( $(echo "$COVERAGE < 100" | bc -l) )); then
            echo "Coverage $COVERAGE% is below 100%"
            exit 1
          fi
```
This enforces TEST-02 requirement (build fails if coverage below 100%). Uses macOS 26 for FoundationModels availability.</action>

| T-191-01 | T | CI coverage gate | mitigate | Use GitHub Actions hardened runner, pin workflow actions to SHAs, validate coverage calculation logic to prevent bypass via percentage manipulation |
| T-191-02 | I | Package.swift dependencies | mitigate | Pin swift-testing to exact version, verify checksums, use SPM audit mode to detect supply chain attacks |

| T-191-04 | E | Coverage report artifacts | mitigate | Encrypt coverage reports stored as artifacts, sign with GitHub Actions key, verify integrity before parsing threshold values |
| T-191-05 | R | Test infrastructure rollback | accept | Swift Testing is Apple-first-party, minimal rollback risk. Version pinned in Package.swift. |

## Adversarial Scenario

An attacker attempts to bypass the 100% coverage gate by modifying the CI workflow to parse coverage output incorrectly (e.g., always return 100%). Mitigation: pin workflow action versions, require branch protection for CI workflow edits, add coverage report validation step that uses an independent calculation method (xcrun llvm-cov export JSON format parsed with jq). The dual-path validation makes bypass significantly harder.
