```yaml
name: Mutation Testing
on: [pull_request]

jobs:
  mutation:
    runs-on: macos-26
    steps:
      - uses: actions/checkout@v4
      
      - name: Install Mull
        run: brew install mull
      
      - name: Generate coverage info
        run: swift test --enable-code-coverage
      
      - name: Run mutation tests
        run: |
          mull-cxx \
            -test-framework=SwiftTesting \
            --coverage-info=.build/debug/codecov/*.profdata \
            --mutate-killed-threshold=90 \
            --mutate-score-threshold=90 \
            --reporters=IDE,JSON \
            --output=mutation-results.json
      
      - name: Enforce 90% mutation score
        run: |
          SCORE=$(jq -r '.mutation_score' mutation-results.json)
          if (( $(echo "$SCORE < 90" | bc -l) )); then
            echo "Mutation score $SCORE% is below 90%"
            jq '.killed_mutants' mutation-results.json
            exit 1
          fi
      
      - name: Upload mutation results
        uses: actions/upload-artifact@v4
        with:
          name: mutation-results
          path: mutation-results.json
```
This enforces TEST-05 requirement (>90% mutation score required to merge).</action>

## Adversarial Scenario

An attacker submits malicious code that passes tests but introduces a vulnerability when mutated (e.g., authentication check that passes unit tests but fails when `==` → `!=`). Mitigation: Mutation testing fuzzes ALL comparison operators, so the vulnerable auth check would be mutated to `!=` and tests would fail (killing the mutant). The 90% threshold ensures high-quality tests catch such mutations before merge. Score trend tracking detects gradual degradation (attacks that sneak in via multiple small PRs).
