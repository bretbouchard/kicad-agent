## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| iCloud sync | External Apple service stores private database — encrypted transit/at-rest |
| Conflict resolution | User chooses winner for LWW — social engineering attack surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-177-01 | T (Tampering) | CloudKit sync records | mitigate | ckRecordChangeTag validates server-side integrity. Reject if tag mismatch. |
| T-177-02 | I (Information disclosure) | Private database contents | accept | CloudKit encrypts at rest/transit. Conversations may contain sensitive design intent but no PII. Acceptable for v1. |
| T-177-03 | D (Denial of Service) | Search query without limit | mitigate | Pagination cap 500 results. Query timeout 10s. Prevents OOM on huge datasets. |
| T-177-04 | E (Elevation) | LWW conflict prompt UI | mitigate | Prompt options limited to (Keep Mine/Keep Theirs/Merge). No arbitrary value injection. Validate user selection. |
| T-177-05 | R (Repudiation) | OpJournal conflict log | mitigate | Conflict resolution auto-logged with timestamp + actor. Cannot erase log. |

## Adversarial Scenario

**Name:** CloudKit schema drift causes silent data loss

**Attack:** Phone app v1.0 updates to v1.1 with new schema field, Mac stays on v1.0. CloudKit auto-migrates Phone schema, wipes local Mac data to force re-sync from server (which has new field Mac doesn't understand). User loses unsynced changes.

**Mitigation:** Explicit VersionedSchema.v6_0_0 with `migrationPlan: nil` (no auto-migrate). On schema mismatch, CloudKit sync fails with `.partialError` showing "schema incompatible". App catches error, shows alert: "Cloud sync paused until both devices update to v1.1". Test `testSchemaMismatchAlertsUser` verifies this path per Pitfall 4.

**Detection:** Two-device migration test (Phone v1.0, Mac v1.1) verifies alert shown, not silent wipe.
