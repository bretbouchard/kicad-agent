    **microstrip_z0 function (IPC-2141 surface microstrip):**
    ```python
    def microstrip_z0(w: float, h: float, t: float, er: float) -> float:
        eff_er = ((er + 1) / 2
                  + (er - 1) / 2 * math.pow(1 + 12 * h / w, -0.5))
        z0 = (87 / math.sqrt(eff_er)
              * math.log(5.98 * h / (0.8 * w + t)))
        return z0
    ```
    Verified numerically: w=0.47mm, h=0.2mm, t=0.035mm, er=4.5 gives Z0 ~50 ohm.

    **stripline_z0 function (IPC-2141 symmetric stripline):**

    ```python
    def stripline_z0(w: float, h: float, t: float, er: float) -> float:
        z0 = (60 / math.sqrt(er)
              * math.log(1.9 * (2 * h + t) / (0.8 * w + t)))
        return z0
    ```

    Create `src/kicad_agent/routing/length_matching.py` with the following components (per RESEARCH.md RQ4 and D-03):

    **LengthMatchResult frozen dataclass:**
    ```python
    @dataclass(frozen=True)
    class LengthMatchResult:
        path: tuple[tuple[float, float], ...]
        target_delta_mm: float
        achieved_delta_mm: float
        num_bumps: int
        valid: bool
    ```

    **_sawtooth_extra_length helper:**
    ```python
    def _sawtooth_extra_length(amplitude: float, half_pitch: float) -> float:
        leg = math.hypot(half_pitch, amplitude)
        return 2 * leg - 2 * half_pitch
    ```
    Each sawtooth bump replaces a straight segment of length `2*half_pitch` with two diagonal legs forming a triangle.

    **_generate_sawtooth_bumps helper:**
    Uses the same interpolation framework as accordion but generates triangle shapes (3 points per bump instead of 4 for U-shape). Imports `_interpolate_path` and `_direction_at` from `kicad_agent.routing.geometry` (created in Plan 01; this module must handle the case where geometry.py does not yet exist by importing at function level -- the actual import will succeed when both plans are complete).

    For each bump position:
    1. Find start point (at bp - half_pitch along path)
    2. Find center point (at bp along path)
    3. Find end point (at bp + half_pitch along path)
    4. Compute perpendicular direction at center
    5. Peak = center + perpendicular * amplitude
    6. Insert 3 points: start, peak, end

    **add_sawtooth_matching main function:**
    ```python
    def add_sawtooth_matching(
        path: tuple[tuple[float, float], ...],
        target_delta_mm: float,
        spacing_mm: float = 1.0,
        max_detour_ratio: float = 3.0,
    ) -> LengthMatchResult:
    ```
    Follows the same measure-and-refine pattern as `_add_serpentining` in diff_pair.py (per RESEARCH.md Pattern 4):
    - Calculate bump_pitch, half_pitch, max_amplitude
    - Determine num_bumps from usable path length
    - Run up to 10 refinement iterations: generate bumps, measure actual delta, adjust amplitude proportionally
    - Cap amplitude at half_pitch * max_detour_ratio (per RESEARCH.md Pitfall 4)
    - Return LengthMatchResult with valid=True if achieved_delta is close to target_delta

    Uses `_path_length` from `kicad_agent.routing.geometry`.

    - Comparison: sawtooth adds less length per bump than accordion at same amplitude/pitch (per RESEARCH.md: sawtooth 0.281mm vs accordion 0.691mm)
