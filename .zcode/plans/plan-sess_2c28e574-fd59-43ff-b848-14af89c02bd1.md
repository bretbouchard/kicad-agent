## Phase 218: Native ERC/DRC Engine — Complete kicad-cli Replacement

### Goal
Replace all kicad-cli ERC/DRC dependency with a pure-Python engine. App Store sandbox-compatible. Zero GPL risk.

### What exists (don't rebuild)
- 50 DFM checks in `dfm/extended_checks.py` (geometry-based, standalone)
- Schematic graph with pin types + union-find net resolution
- Native PCB parser (segments, pads, vias, zones, net_classes, Edge.Cuts)
- shapely geometry layer (STRtree spatial index, board outline extraction)
- 6 existing validation modules (structural, symbol mismatch, spatial, split plane, silkscreen, grid)

### The full build: 18 checks in 4 tiers

---

#### Tier 1: Core ERC (electrical correctness) — 4 checks

**Check 1: Pin-type conflict detection**
- New file: `src/kicad_agent/validation/native_erc.py`
- Encode KiCad's 11×11 pin electrical type compatibility matrix (publicly documented, fixed table)
- Group pins by resolved net (union-find already exists in topology_builder.py)
- For each net with 2+ pins: check every pair against the matrix
- Pin types: input, output, bidirectional, passive, power_input, power_output, open_collector, open_emitter, tri_state, free, unspecified

**Check 2: Power net validation**
- In `native_erc.py`
- Fix: schematic_graph.py skips power symbols (#PWR refs at line 551). Add a separate pass that parses power symbols and records their power_output pins
- For each net classified as POWER (via net_classifier.py): verify at least one pin is power_output
- If only power_input pins exist: warning "power net not driven"

**Check 3: No-connect validation**
- In `native_erc.py`
- Iterate all pins. Pin is error if: (not connected AND not flagged NC) or (flagged NC but actually connected)
- Position matching already exists (no_connect parsing at schematic_graph.py:681, is_connected at :236)

**Check 4: Hierarchical label/sheet pin validation**
- In `native_erc.py`
- Check that hierarchical labels in sub-sheets match labels in parent sheet
- Sub-sheet pin names must match parent's hierarchical sheet pin names
- Leverages existing check_sheet_pin_labels pattern

---

#### Tier 2: Core DRC (manufacturing) — 4 checks

**Check 5: Copper spacing rules**
- In `native_drc.py`
- Build buffered shapely geometries from segments (LineString + buffer(width/2)), pads (shape polygons), vias (circles)
- Group by layer. Pairwise distance check with same-net exemption
- Use existing STRtree spatial index from spatial/query.py for O(n log n) performance
- Template: CopperPourSpacingCheck in extended_checks.py

**Check 6: Board edge clearance** (already done, wire it up)
- BoardEdgeClearanceCheck in extended_checks.py is complete
- Just needs to be exposed via the native_drc interface

**Check 7: Netclass width + spacing enforcement**
- In `native_drc.py`
- Build {net_name: netclass} lookup from parsed net_class blocks
- For each segment: assert width >= netclass.track_width
- For each same-layer different-net pair: assert distance >= netclass.clearance

**Check 8: Global minimum track width**
- In `native_drc.py`
- Check all segments against a global minimum (configurable, default 0.127mm for JLCPCB)

---

#### Tier 3: Advanced DRC — 4 checks

**Check 9: Courtyard overlap**
- Fix the existing CourtyardOverlapCheck in extended_checks.py to use proper F.CrtYd/B.CrtYd geometry
- Fix placement_gate.py:601-640 to use courtyard graphics instead of pad envelopes
- shapely intersection of courtyard polygons between adjacent footprints

**Check 10: Hole-to-hole clearance**
- Extract via drill_diameter + pad drill from native parser
- Pairwise Euclidean distance check on drill centers
- Minimum: typically 0.3mm between drill edges

**Check 11: Solder mask bridge**
- shapely buffer of pad shapes by solder mask expansion
- Check for slivers (narrow copper-free strips) between adjacent pads
- Template: SolderMaskCheck in checks.py (already exists, needs geometry upgrade)

**Check 12: Annular ring**
- For each via/pad: check (pad_diameter - drill_diameter) / 2 >= minimum
- Already exists as ANNULAR_RING_01 + VIA_ANNULAR — wire into native_drc

---

#### Tier 4: The last 5% — 6 checks

**Check 13: Net-tie handling**
- New: detect footprints with `net_tie` attribute or pads sharing a net within the same footprint
- Suppress pad-to-pad short errors within net-tie footprints
- Apply special clearance rules to net-tie pads

**Check 14: Thermal relief spoke counting**
- Upgrade the existing heuristic check
- For each pad-on-zone: count actual copper connections by checking zone polygon edges intersected by trace geometries
- Warn if fewer than 2 spokes (solid pour), warn if >4 (excessive)

**Check 15: Matched-length net groups**
- New: parse length-match groups from net classes / .kicad_dru constraints
- Compute true routed length per net (sum of segment lengths including diagonal — upgrade from Manhattan approximation)
- Check skew within tolerance for all nets in the group
- Template: route_quality.py _check_diff_pairs (extend from 2-member to N-member)

**Check 16: True differential pair checks**
- Replace naive net-name string matching with actual diff-pair membership (from net class diff_pair flag)
- Check within-pair gap consistency along the coupled length
- Check uncoupled length budget
- Compute differential impedance from geometry + stackup

**Check 17: Teardrop detection**
- Detect teardrop polygon shapes attached to pad/via junctions
- Check that teardrops exist on critical vias (high-speed / power)
- Replace the has_teardrop boolean-flag check with geometry recognition

**Check 18: Custom DRC rule expression evaluator**
- New file: `src/kicad_agent/validation/rule_engine.py`
- Build a mini-interpreter for KiCad's DRC expression DSL:
  - Parse: `A.NetClass == 'HV'`, `A.Width < 0.2mm`, `isCoupledDiffPair()`, `insideArea('zone')`
  - Evaluate against net/segment/zone attributes
  - Support boolean operators (and, or, not), comparison (<, >, <=, >=, ==, !=)
  - Support units (mm, mil, in)
- This is the biggest single piece — ~300-400 lines of Python

#### Daemon handler: kicad.native_check

New daemon handler that replaces `kicad.post_check`'s kicad-cli dependency:
- `kicad.native_check` — runs all native checks, returns same result shape as kicad.post_check
- Falls back to kicad.post_check (kicad-cli) in Developer ID builds where available
- App Store builds use native_check exclusively

#### Validation panel wiring

Update ValidationPanel.swift to call `kicad.native_check` instead of `kicad.post_check`. Results display the same way.
