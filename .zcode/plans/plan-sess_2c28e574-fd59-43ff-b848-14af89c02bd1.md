- **Base model**: Gemma 4 12B 4-bit MLX quantized (~7 GB, from `mlx-community/gemma-4-12b-it-4bit` on HuggingFace)

### The Plan: 6 Tasks

#### Task 1: Add mlx-swift-extras dependency

**Why:** Without it, `stream()` throws "llmLoopRequiresPhase165Router." This package provides the autoregressive generation loop + LoRA adapter loading.

**Changes:**
- `Package.swift`: add `mlx-swift-extras` as a dependency, add `MLXLM` product to the target
- `project.yml`: add `mlx-swift-extras` as an SPM package for the Xcode target (re-enable MLXLocalProvider.swift)
- Fix the explicit module build issue (Cmlx, _NumericsShims) — may need `SWIFT_ENABLE_EXPLICIT_MODULES: NO` at the package level

**Risk:** The explicit module build issue that excluded MLX may persist. If so, keep MLXLocalProvider excluded from Xcode but add the Swift code path for when it's available. The download still works (saves to disk), just generation waits for a fix.

#### Task 2: ModelDownloader class

**New file:** `macos-app/Sources/KiCadAgent/Models/ModelDownloader.swift`

Downloads from HuggingFace via URLSession (same pattern as `HFHubModelCatalog`). Downloads:
1. Base model files (config.json + safetensors shards) from `mlx-community/gemma-4-12b-it-4bit`
2. Adapter files (adapters.safetensors + adapter_config.json) from a hosted location (S3, HF, or GitHub release)

Features:
- Async streaming download with progress (`AsyncStream<Double>` for 0.0–1.0)
- SHA-256 verification per file
- Resumable (skip files that already exist + match checksum)
- VRAM check before download starts (3 GB minimum, warns if <16 GB for 12B model)
- Cancel support
- Downloads to `~/Library/Application Support/VoltaPCB/models/gemma-4-12b-it-4bit/` + `~/Library/Application Support/VoltaPCB/models/volta-adapter-v1/`

---
